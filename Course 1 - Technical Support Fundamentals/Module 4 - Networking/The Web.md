🔥 **What the web is and how websites are built**
– **Websites** are essentially text documents formatted with **HTML (HyperText Markup Language)** that web browsers interpret.  
– **Web pages** commonly include multimedia: **text, images, audio, and embedded video (e.g., YouTube)**.  
– Browsers render HTML into the visual pages users navigate.

🌐 **URL structure and domain names**
– **URL (Uniform Resource Locator)** is a web address, analogous to a home address; example: **www.reddit.com**.  
– The **www** prefix stands for **World Wide Web**; the middle portion (**reddit.com**) is the **domain name**.  
– The final portion (**.com**, **.net**, **.org**, **.edu**) is the **top-level domain (TLD)**; TLDs commonly indicate site type (for example, **.edu** is primarily used for educational institutions and is typically restricted).  
– **Domain names are registered through registrars and coordinated under ICANN**; once a domain is registered it is unique and unavailable to others until released.

🧭 **IP addresses and DNS (Domain Name System)**
– Computers and servers are identified by **IP addresses**; network communication uses IPs rather than human-readable names.  
– **DNS** is the critical protocol that **maps domain names to IP addresses**, functioning as the internet’s directory.  
– Example: typing **172.217.6.46** into a browser can take you to **Google’s homepage** because DNS maps that IP to Google’s domain.  
– Every website visit triggers a **DNS lookup** to resolve the domain into an IP the computer can use.

🛠️ **Diagnostics and IT implications**
– If a site is reachable by its **IP address but not by its domain name**, that strongly indicates a **DNS configuration problem** on the network.  
– **IP addresses appear in server logs** and are often used to identify request sources for troubleshooting, access control, and auditing.  
– Many networked systems and infrastructure components require explicit **IP configuration** to function properly.  
– **DNS is a large, complex system** with operational and security implications; careful configuration and monitoring are important for reliable service.
