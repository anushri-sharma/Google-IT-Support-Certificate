🔥 **Userspace — how humans interact with the OS**
  - **Userspace** is the part of an operating system focused on human interaction: creating files/folders, opening applications, deleting items, etc.
  - Interaction methods expose functionality provided by the kernel to users and applications.

💻 **Two primary interaction models: CLI (shell) and GUI**
  - **CLI / shell**: text commands typed by the user that a shell program interprets and sends to the OS to execute.
  - **GUI**: a visual, mouse-driven interface where users click/drag to perform actions and see operations represented graphically.

1. CLI Shell

⌨️ **Shells (CLI) — purpose and form**
  - A **shell** is a program that accepts text commands, interprets them, and requests the kernel perform operations.
  - Historically, all interactions were typed; GUIs are more recent layers above shells.
  - Some shells have graphical front-ends, but the underlying idea remains command interpretation.

2. GUI
   
🖱️ **Graphical User Interface (GUI) — ubiquity and recognition**
  - GUIs are widely used daily (desktop OSes, mobile phones, tablets, ATMs, airport kiosks).
  - Users often recognize devices or OSes by **GUI design** (windows, menus, icons).
  - GUIs present actions visually, making many tasks more intuitive for average users.

⚙️ **Variety among shells and the chosen example**
  - Shells differ in **features** and **performance characteristics**, analogous to different operating systems.
  - The common shell used here is **Bash (Bourne-Again SHell)**.

🔧 **Why learning the shell matters — practical implications**
  - **Power users** (above-average computer users) rely on shells for efficiency and advanced tasks.
  - In Linux and many IT contexts, **knowing commands is essential** because remote systems are often accessed without a GUI.
  - Certain tasks and certification/test scenarios can only be completed via CLI.

🚀 **Scale, automation, and efficiency — key advantages of CLI**
  - Commands can be executed once to affect **many machines**, avoiding repetitive GUI actions per device.
  - Shell usage enables automation, scripting, and bulk-management in advanced IT roles.
  - For remote administration and large-scale management, CLI is typically faster, scriptable, and more reproducible than GUI-based workflows.
