🔧 **Punch cards and early machine code (1950s)**  
  – **Punch cards** stored programs as bits that the CPU read and executed as low-level instructions.  
  – Typical machine-step sequence: **grab input data from a memory location → perform arithmetic → write output to a memory location**.  
  – Storing programs on punch cards was **tedious and error-prone**, requiring physical stacks of cards and manual handling.

🧩 **Assembly language: human-readable machine instructions**  
  – **Assembly language** introduced mnemonic, human-readable instructions that assemble into machine code.  
  – Example instruction pattern: **take integer from register 1, take integer from register 2, add them, output to register 4** (shows register-based, low-level operations).  
  – Assembly is a **thin veil over machine code**: it still reads and behaves robotically and is **specific to a CPU or CPU family**, so programs written in assembly are not portable across different CPU architectures.

⚙️ **Compiled programming languages and compilers**  
  – **Compiled languages** allow programmers to write in human-readable code which a **compiler** translates into machine instructions.  
  – **Admiral Grace Hopper** developed early compiler concepts and tools that helped make higher-level compilation practical.  
  – **Compilers** were a foundational advance that enabled more abstract programming and helped drive modern computing forward.  
  – The rise of many compilers and languages led to **hundreds of programming languages** created to further abstract CPU-specific details into simpler commands.

🧠 **Interpreted languages and scripts**  
  – **Interpreted languages** are not compiled ahead of time; a file of code (a **script**) is executed by an **interpreter** that translates instructions on the fly at runtime.  
  – Interpreters convert script statements into CPU actions just in time to run them, rather than producing standalone machine-code binaries beforehand.

🔁 **Scripting and automation for IT support**  
  – **Scripting** enables automating repeated tasks by having the computer perform them on behalf of the user.  
  – This allows an IT specialist to **solve a problem once (via a script)** and then reuse that solution instead of repeating manual steps.  
  – Learning a scripting language is presented as a practical skill for IT support roles.

📌 **Role and purpose of programming languages**  
  – Programming languages (compiled or interpreted) exist to **create programs that perform tasks** by expressing instructions at varying levels of abstraction.  
  – The evolution from punch cards → assembly → compiled and interpreted languages reflects a continual push to make programming **more readable, portable, and automatable**.
