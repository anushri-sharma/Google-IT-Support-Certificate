# Types of Software

[Types of Software](https://www.youtube.com/watch?v=kTy3oUV-ScI&t=183s)

🔥 **Copyright protects original software work**
  - **Copyright** applies to written software and generally protects creators' rights to use and distribute their work, subject to country-specific legal caveats.
  - **Developers choose distribution**: they can impose commercial licenses or release software under other terms.

🍳 **Commercial licensing vs open source**
  - Commercial software commonly requires paying for a **license** to use it.
  - **Open source** lets others **share, modify, and distribute** software for free under the project's license terms.
  - **Implication:** choice affects cost, redistribution rights, and how software can be modified or integrated.

💡 **Notable open-source examples and impact**
  - **Linux kernel**: used in **Android OS**, enterprise servers, and personal computers; runs on **hundreds of millions of devices**.
  - Other examples: **LibreOffice**, **GIMP**, **Firefox**.
  - **Implication:** large projects can advance rapidly through community contributions and shared development.

🔧 **Open-source development model**
  - Many open-source projects are built and advanced by **volunteer contributors** who work in their free time.
  - **Implication:** project health depends on community engagement, maintainers, and contribution policies.

📋 **License management in IT environments**
  - In IT environments, verify licensing because some software may require **multiple paid licenses**, while others are **free/open source**.
  - **Actionable rule:** **check the license agreement before installing** any software to understand usage, distribution, and cost constraints.

🗂️ **Two functional categories of software**
  - **Application software**: created to fulfill specific user needs (examples: **text editors**, **web browsers**, **graphic editors**).

  - **System software**: maintains core system operation (examples: **operating system tools**, **utilities**).

🔩 **Firmware as a type of system software**
  - **Firmware** is software **permanently stored** on hardware components.
  
  <img width="1280" height="429" alt="image" src="https://github.com/user-attachments/assets/e12932f0-a287-46bc-a045-bf41cc303306" />

  
  - Example: **BIOS**, which initializes hardware and loads the operating system; permanence matters for startup reliability.

🔢 **Software versions and numbering**
  - **Versions** indicate feature sets and changes in specific software iterations.
  - Commonly follow sequential numeric patterns such as **1.2.5** and **1.3.4**; **1.3.4** is newer than **1.2.5** because of a larger sequence number.
  - Developers may use different versioning standards, but the majority follow a sequence-based approach; consult project-specific versioning docs for exact meaning.

🔁 **Practical learning implication**
  - Most software operates under similar concepts and interfaces; **learning one program transfers skills to others**, reducing the learning curve across different tools.
