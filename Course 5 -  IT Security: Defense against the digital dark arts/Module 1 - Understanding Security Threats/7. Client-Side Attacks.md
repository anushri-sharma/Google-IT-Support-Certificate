
# [Client-Side Attacks](https://www.youtube.com/watch?v=4hIArmwPz8w)

🔥 **Scope: attacks beyond direct targeting and common injection vector**
  - Web applications experience attacks that target software and users indirectly, not only direct victim-targeting methods.
  - **Injection attacks** are a widespread class of exploits on the web where an attacker inserts malicious code or data into an application's inputs.

💡 **Analogy and core mechanism**
  - Analogy: injecting a strawberry‑banana milkshake into a car's gas tank illustrates how an unexpected input can break a system.
  - Defensive principle from the analogy: implement a mechanism that **only accepts the correct type of input** to prevent harmful injections.

🛡️ **Primary mitigations**
  - **Input validation** — enforce strict checks on incoming data types, formats, ranges, and allowed characters.
  - **Sanitizing data** — remove or neutralize unsafe content before it is processed or rendered.
  - These measures are described as part of **good software development principles** to mitigate injection attacks.

🧩 **Cross-site scripting (XSS) — attacker targets users**
  - **Cross-site scripting (XSS)** is an injection attack where malicious scripts are embedded in web content and executed in a user's browser.
  - Typical impact: **session hijacking** by stealing **cookies** or other client-side session artifacts, allowing the attacker to impersonate the user on a site.
  - Attack vector example: an attacker embeds a script in a webpage that runs when a user views the page, causing cookie exfiltration or other client-side actions.

🧨 **SQL injection (SQLi) — attacker targets the website/database**
  - **SQL injection (SQLi)** targets applications that use SQL databases by injecting SQL commands through inputs.
  - Potential consequences: attackers can **delete**, **copy**, or otherwise manipulate the website's stored data and execute arbitrary database commands.
  - Distinction: unlike XSS, which targets individual users' browsers/sessions, SQLi compromises the server-side data layer and can affect the entire site.
