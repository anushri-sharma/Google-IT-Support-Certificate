### Definition and Basic Concept of Denial-of-Service (DoS) Attacks  
- A **Denial-of-Service (DoS) attack** aims to **prevent legitimate users from accessing a service** by overwhelming the network or server resources.  
- Example: If a website can only serve 10 users at a time, an attacker occupying all 10 slots denies access to real users.  
- This concept applies on a much larger scale to major websites like Google or Facebook, where attackers consume resources to block genuine user access.  
- **Core Insight:** DoS attacks target resource exhaustion to disrupt normal service availability.

### Examples of DoS Attacks: Ping of Death and Ping Flood  
- **Ping of Death (PoD):**  
  - Sends a **malformed ping packet** that exceeds the size limit defined by Internet protocols.  
  - Causes a **buffer overflow**, which can crash the system or enable malicious code execution.  
- **Ping Flood:**  
  - Sends a large number of **ICMP echo requests (ping packets)** to overwhelm the target.  
  - The victim attempts to respond with ICMP echo replies, but if overwhelmed, the system crashes or becomes unresponsive.  
- **Key Point:** Both attacks exploit the ICMP protocol to exhaust system resources.

### SYN Flood Attack and TCP Connection Process  
- **TCP Connection Establishment Process:**  
  1. Client sends a **SYN** packet to initiate a connection.  
  2. Server responds with a **SYN-ACK** packet.  
  3. Client sends an **ACK** packet to complete the handshake.  
- **SYN Flood Attack Mechanism:**  
  - Attacker sends many **SYN packets** but never sends the final **ACK**.  
  - Server keeps the connection half-open, holding resources for each attempt.  
  - This leads to **resource exhaustion** and legitimate users cannot connect.  
- **Terminology:** This attack is called a **half-open attack** because connections are never fully established.  
- **Impact:** Server resources are tied up indefinitely, leading to denial of service.

### Transition from DoS to DDoS: Multiple Machines Involved  
- Traditional DoS attacks use **a single machine** to overwhelm the target.  
- **Distributed Denial-of-Service (DDoS) Attack:**  
  - Utilizes **multiple compromised machines** to launch a coordinated attack.  
  - Results in a much higher volume and speed of attack traffic.  
  - These machines are often part of a **botnet**, a network of devices controlled by attackers.  
- **Significance:** DDoS attacks are far more difficult to defend against due to their distributed nature and scale.

### Case Study: October 2016 DDoS Attack on DNS Provider DYN  
- **Incident Summary:**  
  - The DNS service provider **DYN** was targeted by a massive DDoS attack involving:  
    - Fake DNS lookup requests  
    - SYN flood attacks  
  - These attacks were carried out by botnets controlling many devices.  
- **Impact:**  
  - DYN managed DNS services for major websites such as **Reddit, GitHub, Twitter,** and others.  
  - When DYN's system was overwhelmed and went down, all its customers' services became inaccessible.  
  - Resulted in widespread disruption to internet access for millions of users.  
- **Key Lesson:** The incident highlights the critical dependency of web services on third-party DNS providers and the devastating potential of DDoS attacks to cripple internet infrastructure.

### Summary Table: DoS Attack Types and Mechanisms

| Attack Type      | Mechanism                                      | Targeted Protocol/Resource       | Key Effect                              |
|------------------|------------------------------------------------|---------------------------------|---------------------------------------|
| Ping of Death    | Sends oversized ping causing buffer overflow   | ICMP (Ping)                     | System crash, potential code execution |
| Ping Flood       | Sends large volume of ICMP echo requests        | ICMP (Ping)                     | Resource exhaustion, system overwhelmed|
| SYN Flood        | Sends many SYN packets, no ACK response         | TCP Connection Handshake        | Half-open connections, resource exhaustion |
| DDoS             | Uses multiple systems (botnets) for attacks     | Various (DNS, TCP, ICMP, etc.)  | Large-scale service disruption          |

---

### Key Insights  
- **DoS and DDoS attacks exploit system resource limitations to deny legitimate users access to services.**  
- **SYN floods abuse the TCP handshake process by leaving connections half-open, draining server resources.**  
- **DDoS attacks magnify threat severity via multiple attacking machines, often coordinated through botnets.**  
- **Critical internet infrastructure like DNS providers are prime targets, with attacks on them causing cascading service outages.**

---

### Glossary  
| Term               | Definition                                                                                   |
|--------------------|----------------------------------------------------------------------------------------------|
| DoS (Denial of Service) | Attack that makes a service unavailable to legitimate users by exhausting resources.         |
| DDoS (Distributed Denial of Service) | DoS attack using multiple machines to amplify attack volume and impact.               |
| ICMP (Internet Control Message Protocol) | Protocol used for sending control messages like ping requests and replies.           |
| SYN Flood          | DoS attack exploiting the TCP handshake by sending many SYN packets without completing handshake. |
| Botnet             | Collection of compromised computers controlled by an attacker to launch coordinated attacks.    |
| DNS (Domain Name System) | System translating domain names to IP addresses, essential for internet navigation.            |
