1. Regulatory Compliance
PCI DSS
The Payment Card Industry Data Security Standard (PCI DSS) is a set of security
standards designed to ensure that all companies that accept, process, store, or
transmit credit card information maintain a secure environment.

HIPAA is for healthcare data.

ISO (specifically ISO 27001) is a general information security management framework.

2. Risk Assessment
Threat modeling
The first step is identifying what you are trying to protect and who might
want to attack it. Threat modeling helps you understand the potential "adversaries"
and the methods they might use before you start testing for specific technical holes.

```

3. Verification Testing
Attempting to break into a system or network for the purpose of verifying the systems in place.
Penetration testing (or "pen testing") is an authorized, simulated attack. Unlike a vulnerability
scan, which just lists potential holes, a pen test attempts to actually exploit them to see how
much damage could be done.
```

4. Storage Risks
CD drives

USB sticks
Any unmanaged, removable media is a high risk because it is easily lost, stolen,
or used to bypass network-based security controls. While an encrypted drive is safer,
it is still a physical risk compared to a centralized, monitored file share.

5. & 6. Credentials and Identity
5. 2-factor authentication
While strong encryption and passwords are important, 2-factor authentication (2FA) is
the single most effective way to secure authentication because it requires a second "factor"
(like a code on your phone) that an attacker is unlikely to have.

```

6. Phishing emails
Phishing remains a top risk because it targets the "human element." It is far easier
for an attacker to trick an employee into typing their password into a fake website
than it is to hack a complex encryption algorithm.

```

7. & 8. Third-Party Risk
7. Trusting the third party's security
When you use a third-party service (like a cloud provider), you are essentially extending
your security perimeter to their data centers. You are at risk if their internal security practices are weak.

9. Security Culture
Asking security-related questions

Checking website URLs when authenticating (to spot phishing)

Locking your screen (to prevent unauthorized physical access)

Note: Shaming people is counterproductive; a healthy security culture encourages employees to
report mistakes without fear so they can be fixed quickly.

10. Incident Response
containment
The standard incident response lifecycle is:

 Detection → Scope/Analysis → Containment → Eradication → Recovery.

Once you know how big the fire is, the very next step is to "put a box around it"
(containment) to stop it from spreading to other systems.
