# Notes

 
This segment introduces the shift in focus from **network hardware hardening** to **network software hardening techniques** in the context of IT support security measures. It emphasizes the importance of understanding how to implement software-based network security solutions, such as **firewalls, proxies, and VPNs**, which are crucial for protecting organizational networks and their traffic.

- Network software hardening complements hardware security measures.
- Key components include firewalls, proxies, and VPNs.
- These tools are essential in securing network traffic and access.


**Firewalls** are highlighted as a critical tool for network security. They can be deployed in two main forms:

- **Network-based firewalls:** Dedicated infrastructure devices that regulate traffic flow for an entire network.
- **Host-based firewalls:** Software running on individual client systems, protecting that single host.

<img width="1275" height="561" alt="image" src="https://github.com/user-attachments/assets/a5000650-5eab-4ca2-83dc-00f5b31461a0" />

**Best practice:** Deploy both network-based and host-based firewalls for comprehensive security coverage.

- Host-based firewalls are especially important for **mobile devices** (e.g., laptops) used in untrusted environments like airport Wi-Fi hotspots.
- They also help protect against internal threats that network-based firewalls might miss.
- Modern operating systems typically include built-in host-based firewalls.
- Network-based firewalls are common both in corporate environments and consumer devices (e.g., home routers).

 
This part discusses **VPNs (Virtual Private Networks)** and their role in securing remote access:

- VPNs provide **secure access to internal resources** for mobile or roaming users.
- Two primary VPN use cases:
  - **Site-to-site VPN:** Connects two geographically separate offices into a unified network, allowing seamless resource sharing.
  - **Remote access VPN:** Enables individual users to securely connect to internal resources when outside the office.

<img width="1278" height="267" alt="image" src="https://github.com/user-attachments/assets/01664a4e-aad4-4890-8188-b24f437b9526" />

Key points about site-to-site VPNs:

- Encrypts all traffic between remote networks through a secure VPN tunnel.
- Users in either office can access services in the other without additional configuration.
- Creates a seamless, unified network experience.

Remote access VPNs typically use the same infrastructure to allow individual clients secure connectivity.

The segment explains the use of **proxies** for traffic protection and remote access as alternatives or complements to VPNs:

- **Standard web proxies** can be configured on client devices to route web traffic through a controlled proxy server.
- Benefits of using proxies include:
  - **Logging web requests** for monitoring and forensic investigation.
  - **Traffic analysis** to detect suspicious or malicious activities.
  - Blocking access to malicious, dangerous, or policy-violating content.

- **Reverse proxies** provide secure remote access to internal web-based services without requiring a VPN.

<img width="1178" height="718" alt="image" src="https://github.com/user-attachments/assets/1d836587-0251-4ab1-8ed8-99304c373396" />

As an IT support specialist, responsibilities may include:

- Configuring and maintaining reverse proxy services.
- Using reverse proxies at the network edge to intercept and relay connection requests from outside clients to internal services.

Additional security measures for reverse proxies include:

- Requiring **client TLS certificates** combined with **username and password authentication**.
- Implementing **Access Control Lists (ACLs)** to restrict access further.

**Popular proxy software supporting reverse proxy configurations** includes:

### Key Insights and Concepts

- **Network software hardening** involves securing traffic and access through software tools: firewalls, VPNs, and proxies.
- **Firewalls** come in two main types: network-based (for entire networks) and host-based (for individual devices).
- Combining both firewall types enhances protection against external and internal threats.
- **VPNs** are critical for:
  - Securely connecting separate office locations (site-to-site).
  - Providing secure remote access for individual users.
- **Proxies** serve multiple roles:
  - Traffic logging and monitoring.
  - Content filtering according to company policies.
  - Providing secure access to internal services via reverse proxy without VPN.
- Reverse proxies act as intermediaries between external clients and internal services, adding layers of authentication and access control.
- Popular proxy solutions such as HAProxy, Nginx, and Apache are versatile tools in network software hardening.
