# Notes

 
**Introduction to Packet Sniffing and Packet Capture**  
- Packet sniffing (or packet capture) is the process of intercepting and analyzing network packets in their entirety.  
- It is a critical tool for IT support specialists to troubleshoot network issues.  
- Numerous tools simplify packet capture, but understanding the underlying concepts is essential before usage.

  
**Default Network Interface Behavior**  
- By default, network interfaces and the OS networking stack only process packets addressed specifically to their MAC address.  
- Packets not addressed to the interface are simply dropped and ignored.  
- This default behavior is restrictive when the goal is to monitor all network traffic on a segment.

  
**Promiscuous Mode for Network Interfaces**  
- Promiscuous mode is a special Ethernet interface mode that allows the interface to accept **all packets it can see**, regardless of destination address.  
- This mode is essential for capturing and analyzing all network traffic on a segment.  
- Enabling promiscuous mode requires **administrative or root privileges**.  
- Many packet capture tools automate enabling promiscuous mode.

  
**Access Limitations to Network Traffic on Switched Networks**  
- On a switched network, a machine connected to a switch port only sees traffic sent from or destined to itself.  
- Promiscuous mode does **not** overcome this limitation because the switch selectively forwards frames only to intended ports.  
- To capture traffic between other hosts on a switch, your device must have visibility over that traffic.

 
**Methods to Obtain Full Traffic Visibility on Switches**  
- **Insertion Point Method:** Place your capture device between the switch uplink port and upstream device to see all packets flowing in/out of the network segment.  
- **Port Mirroring (SPAN):** Managed enterprise switches can be configured to mirror traffic from one or more ports or VLANs to a designated monitoring port.  
  - This enables convenient and secure access to all switch traffic without physically rerouting connections.  
- **Hub Insertion:** Using a hub instead of a switch replicates all traffic to all ports, allowing capture of all packets.  
  - However, hubs reduce network throughput and increase collision domains, making them less ideal.

**Wireless Packet Capture Differences**  
- Wireless interfaces in **promiscuous mode** receive packets destined for other clients on the same associated wireless network.  
- To capture **all wireless traffic** in the vicinity, regardless of network association, wireless interfaces must be placed into **monitor mode**.  
- Monitor mode allows scanning across multiple wireless channels and captures all packets from all APs and clients within range.  
- This mode does **not** require the capturing device to be connected or associated with any wireless network.

**Details on Wireless Monitor Mode and Tools**  
- Enabling monitor mode is similar in complexity to promiscuous mode and can often be handled automatically by capture tools.  
- Capturing wireless traffic requires proximity to the target APs and clients to receive their signals.  
- Popular open-source wireless capture tools include:  
  - **aircrack-ng**  
  - **Kismet**  
- When wireless networks use encryption, packet capture is still possible, but **payload decryption requires knowing the network password**.  
- Without the password, only encrypted packets can be captured but not decoded.

### Important Notes and Conclusions  
- **Promiscuous mode alone is insufficient for monitoring all traffic on switched networks.** It only helps if the interface physically receives the packets.  
- **Port mirroring is a preferred, secure, and manageable way** to capture traffic across switch ports in enterprise environments.  
- **Wireless traffic capture requires specialized modes (monitor mode) and tools** to see all traffic in the air, not just traffic on the connected network.  
- **Encryption limits the ability to decode wireless packet payloads**, underscoring the importance of having network credentials for meaningful wireless traffic analysis.  
- Root or administrative privileges are consistently required to enable packet capturing modes and perform effective network traffic analysis.
