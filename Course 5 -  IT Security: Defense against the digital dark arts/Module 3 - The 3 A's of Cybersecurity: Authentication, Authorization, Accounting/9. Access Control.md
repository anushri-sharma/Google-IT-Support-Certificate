# Notes

**OAuth Overview and Functionality**  
- OAuth is an **open standard for access delegation**, enabling users to grant third-party websites or applications access to their information **without sharing account credentials**.  
- This access delegation is achieved by prompting the user to explicitly approve which specific information or access rights the third party is requesting.  
- Once approved, the identity provider issues an **access token** to the third party, which can then use this token to access the user's data or services on their behalf.  
- OAuth is widely used by major internet companies (e.g., Google, Microsoft, Facebook) to allow third-party apps to interact with their APIs securely.

**OAuth Token Usage and Example Scenario**  
- Tokens specify a **scope** defining what resources or services can be accessed.  
- Example: A meme creation website uses OAuth to send memes via the user’s email account instead of sending emails from an unfamiliar address.  
- The website requests permission from the user’s email provider to send emails, and upon approval, receives an access token scoped only for email access, **not other services like cloud storage or calendar**.  
- This ensures **least privilege access**, limiting the third party strictly to the approved functionality.

**Security Considerations and Phishing Risks with OAuth**  
- Users must carefully review OAuth permission requests to avoid granting excessive or unnecessary access.  
- OAuth tokens can be exploited in phishing attacks because attackers do not need to steal passwords; they only need to trick users into authorizing malicious apps.  
- Example: In early 2017, an **OAuth-based worm attack** used phishing emails mimicking legitimate Google Doc sharing requests.  
- Victims were prompted to authorize access to email, documents, and contacts via a malicious service disguised as “Google Apps.”  
- The attacker then used the victim’s email account to propagate the attack further by sending infected emails to their contacts.  
- This highlights the **importance of verifying the legitimacy of OAuth authorization prompts** before granting access.

**Distinction Between OAuth and OpenID**  
- OAuth is an **authorization system** that controls access to resources.  
- OpenID is an **authentication system** that verifies user identity.  
- They are often used together to provide both authentication and authorization.  
- **OpenID Connect** is an authentication layer built on OAuth, designed to improve integration and security between authentication and authorization processes.

**Introduction to TACACS+ as an AAA System**  
- TACACS+ is a **complete Authentication, Authorization, and Accounting (AAA) system**.  
- It handles both authentication (verifying user identity) and authorization (controlling what users are allowed to do).  
- After authentication, TACACS+ determines whether a user can execute certain commands or access specific devices.  
- This enables **granular control over user permissions** in network environments.

**Example of Role-Based Authorization Using TACACS+**  
- Network administrators are given **admin access** to configure and manage switches, routers, and other infrastructure.  
- Support teams are granted **read-only access** sufficient to perform troubleshooting without the ability to change configurations.  
- Other users have **no access** to networking infrastructure, preventing unauthorized connections or modifications.  
- Some AAA systems allow **further refinement** of authorization down to specific commands or actions, enhancing security and operational flexibility.

**Radius and Network Access Authorization**  
- Radius also supports authorization by controlling **network access permissions**.  
- Example: Some users may be allowed WiFi and VPN access, while others are restricted.  
- When users authenticate via Radius, the server responds with configuration details specifying which **network services** the user is authorized to access.  
- This provides **policy-based control** over network resource allocation and access levels.

### Summary Table: AAA Systems and Their Functions

| Protocol  | Functions Covered                | Key Features                                         | Use Case Example                                    |
|-----------|--------------------------------|-----------------------------------------------------|----------------------------------------------------|
| OAuth     | Authorization only              | Delegates limited access via tokens                  | Allowing third party apps to send emails on behalf of the user |
| OpenID    | Authentication only             | Verifies user identity                               | Logging into websites with a single identity        |
| OpenID Connect | Authentication + Authorization layer on OAuth | Combines identity verification with OAuth-based permissions | Secure single sign-on and access delegation         |
| TACACS+   | Authentication + Authorization + Accounting | Granular command-level authorization                | Managing network device access among different user roles |
| Radius    | Authentication + Authorization | Controls network access services                      | Granting WiFi/VPN access based on user roles         |

### Key Insights  
- **OAuth tokens are scoped and limited**, but users must be vigilant to avoid phishing that abuses OAuth’s authorization flow.  
- **OAuth is distinct from OpenID**, with the former focused on access delegation and the latter on identity verification, though they complement each other.  
- AAA systems like **TACACS+ and Radius provide sophisticated access control** beyond simple authentication, enabling role-based and service-specific user permissions.  
- Network security relies heavily on **proper configuration of authorization policies** to ensure users have only the necessary access to perform their duties.  

IT highlights the importance of understanding the differences between authentication and authorization systems, the security considerations when using OAuth, and the role of AAA protocols in managing secure access to network resources.
