
# [Hashing](https://www.youtube.com/watch?v=UGtRfbHZ9qc&t=5s) 

🔑 **Definition and core behavior of a hash function**  
  – **Hash function**: maps *arbitrary-length* input to a **fixed-size output** called a **hash** or **digest**.  
  – **Output size** is typically specified in **bits** and often included in the function name (e.g., **SHA-256** → **256-bit** digest).  
  – Same input always yields the **same** digest; different inputs should produce **different** digests (uniquely identify data in practice).

🧩 **Desired properties of cryptographic hash functions**  
  – **Deterministic**: identical input → identical hash every time.  
  – **Efficient**: **quick to compute** for any input size.  
  – **One-way / Preimage resistance**: infeasible to recover the original input from its hash.  
  – **Avalanche effect**: a **small change** in input produces a **large, uncorrelated** change in output.  
  – **Collision resistance**: infeasible to find two different inputs that produce the same digest (absolute prevention is impossible due to fixed output size, so the practical requirement is computational infeasibility).

🔒 **How hashing differs from encryption**  
  – Both can produce unintelligible outputs from plaintext, but **encryption is reversible with a key**, whereas **cryptographic hashing is one-directional** and cannot be reversed to recover plaintext.  
  – Hashes are used to *verify* or *identify* data rather than to conceal and later recover it.

⚙️ **Primary applications**  
  – **Software/data structures**: hash tables for fast lookups.  
  – **Storage/processing**: detecting duplicates, deduplication, speeding searches.  
  – **Security and integrity**: authentication, message integrity checks, fingerprinting, data-corruption detection, and digital signatures.

🧠 **Implementation notes and relationship to ciphers**  
  – Many hash algorithms **operate on blocks of data** and several popular hashes are built from or inspired by **modified block cipher** constructions.  
  – Digest **bit-length** influences collision probability: shorter digests increase collision likelihood, so cryptographic use favors longer outputs.

🧪 **Illustrative examples**  
  – Demo (imaginary hash): input **"Hello World"** → **E49AOOFF**; input **"hello world"** → **FF1832AE** — shows **determinism** and the **avalanche effect**.  
  – Real-world demo: **md5sum** can be used to compute a digest for demonstration; note **MD5 is considered cryptographically broken** for collision resistance and is unsuitable for security-critical uses.
