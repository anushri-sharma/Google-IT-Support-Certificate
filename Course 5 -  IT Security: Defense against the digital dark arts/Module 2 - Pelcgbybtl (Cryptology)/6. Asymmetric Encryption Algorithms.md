
# [Asymmetric Encryption Algorithms](https://www.youtube.com/watch?v=JR6PD2NOeEA&t=4s)

🔐 **RSA (Rivest–Shamir–Adleman)**  
  – Invented by **Ron Rivest, Adi Shamir, and Leonard Adleman**; patented in **1983**, released to public domain by RSA Security in **2000**.  
  – Specifies **key generation, key distribution, encryption, and decryption** operations for a public-key system.  
  – Key generation depends on choosing **two unique, random, very large prime numbers**; security rests on the difficulty of factoring their product.  
  – Mathematical details are complex and beyond the scope here, but the prime-factor basis is the core security assumption.

✍️ **DSA (Digital Signature Algorithm)**  
  – Standardized in **1991** and included in the US **Federal Information Processing Standard (FIPS)**; used for **signing and verifying** data rather than general encryption.  
  – Specification covers **key generation** and **signature/verification** operations.  
  – Security requires a **fresh, random seed value per signature** (a per-signature nonce); if that value is leaked or predictable, an attacker can **recover the private key**.  
  – Real-world example: **Sony PlayStation 3 (2010)** — reuse/predictability of the randomized value enabled the hacker group **Fail0verflow** to recover Sony’s signing private key, allowing them to sign and run custom software on a locked console and enabling widespread game piracy and associated sales losses.

🔁 **Diffie–Hellman (DH) key exchange**  
  – Named for its co-inventors; designed **solely for establishing a shared secret** over an unsecured channel (commonly used to derive symmetric keys).  
  – Basic protocol steps (using participants Suzanne and Darryl):  
    – Agree on a **shared starting number**: a session-specific, large random integer that **does not need to be secret**.  
    – Each participant selects a **secret large random number** (kept private).  
    – Each combines the shared starting number with their secret to produce a mixed value, then **exchange** those mixed values.  
    – Each participant combines their own secret with the received mixed value to compute a **shared secret** that matches on both sides while exposing insufficient information for an eavesdropper to recover it.  
  – Applications and notes: often used to establish a symmetric session key; there are efforts/adaptations to use DH-like mechanisms for encryption and within **Public Key Infrastructure (PKI)**.


🧭 **Elliptic Curve Cryptography (ECC)**  
  – Public-key cryptography based on the **algebraic structure of elliptic curves over finite fields**.  
  – Elliptic curve example equation: **y² = x³ + a x + b**.  
  – Key geometric/algebraic properties used in crypto:  
    – **Horizontal symmetry** of the curve (points reflect across the x-axis).  
    – Any non-vertical line intersects the curve in **at most three points**, enabling group operations used for key agreement and signatures.  
  – Practical advantage: **smaller key sizes** for equivalent security compared to factoring-based systems; example: **256-bit ECC ≈ 3072-bit RSA**, which reduces storage and transmission overhead for keys.  
  – Variants: **ECDH** (Elliptic Curve Diffie–Hellman) and **ECDSA** (Elliptic Curve Digital Signature Algorithm).  
  – Standards and policy: **NIST recommends** EC algorithms; the **NSA permits ECC** to protect up to **top secret** information with **384-bit EC keys**.  
  – Caveat: the NSA has expressed concern about **potential quantum computing** threats to ECC as quantum technology advances.
