
# [Hashing Algorithms](https://www.youtube.com/watch?v=DD8_CyUYCJg&t=1s)

🔥 **MD5: historical cryptographic hash (now insecure)**  
  - **Published: 1992.**  
  - **Block size: 512 bits; Digest: 128 bits.**  
  - **Design flaw discovered: 1996**, leading to recommendations to migrate to stronger hashes.  
  - **Collision vulnerability demonstrated: 2004**, enabling creation of two different files with identical MD5 digests.  
  - **Security exploit: 2008**, researchers used an MD5 collision to create a fake SSL certificate that validated due to the collision.  
  - **Operational guidance: deprecated for cryptographic use by ~2010** because collisions enable forgery and impersonation.  
  - **Real-world abuse: 2012 Flame malware** used an MD5-based forgery of a Microsoft digital certificate to sign malware, making it appear legitimate.

💡 **SHA-1: design, uses, and eventual practical breakage**  
  - **Designed by NSA; published: 1995.**  
  - **Block size: 512 bits; Digest: 160 bits.**  
  - **Widespread use:** TLS/SSL, PGP/SSH, IPSec, and as the object-identification hash in **Git** (data-integrity / revision identification).  
  - **Standards guidance:** NIST and other organizations **recommended migrating from SHA-1 to SHA-2 (or SHA-3) by 2010** due to emerging weaknesses.  
  - **Theoretical and partial attacks in the 2000s** showed weaknesses but required very large compute resources for full collisions.  
  - **Estimated early cost for a collision using older methods: ~** **$2.77 million** in cloud CPU resources (prior-generation cost estimates).  
  - **2015: improved attack techniques** demonstrated feasible non-full-collision attacks and reduced complexity; full-collision estimates dropped to **~$75,000–$120,000** using combined CPU/GPU cloud resources.  
  - **2017: first practical full SHA-1 collision published**, producing two distinct PDF files with the same SHA-1 hash.  
  - **Compute effort for 2017 collision (reported):** equivalent to **6,500 CPU-years** and **110 GPU‑years** of nonstop computation.  
  - **Implication:** SHA-1 is practically breakable for collision attacks given modern GPU/cloud resources; it is unsuitable where collision resistance is required.

🔢 **Collision economics and technical trend**  
  - **Improved algorithms + GPU/cloud acceleration** dramatically lowered cost and time to produce collisions compared to early theoretical attacks.  
  - **Practical attackability timeline:** from theoretical (2000s) → demonstrable methods (2015) → full real-world collisions (2017).  
  - **Consequences:** signing, certificate issuance, code-revision identification, and any reliance on collision resistance are at risk with weakened hashes.

🧾 **MIC vs MAC: integrity vs authentication distinction**  
  - **MIC (Message Integrity Check):** essentially a plain hash/checksum of message contents used to detect accidental corruption.  
  - **MAC (Message Authentication Code):** uses a secret key to provide both integrity and authentication.  
  - **Security difference:** a **MIC provides no authentication**—an attacker can alter the message, recompute the hash, and replace the MIC; a **MAC prevents that** because the attacker lacks the secret key.  
  - **Use case guidance:** use MICs/checksums for detecting accidental errors; use MACs (or digital signatures) to protect against tampering and forgery.
