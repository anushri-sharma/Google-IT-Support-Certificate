

# [Windows: View User and Group Information](https://www.youtube.com/watch?v=zrJpnkxVKyA&t=6s)

🔥 **Open Computer Management (Local) to view user and group info**
  - **Search "Computer Management"** in Windows and open the app; the top of the sidebar shows **Computer Management (Local)** indicating management of a single machine.
  - **In enterprise environments** use a **Windows domain** to manage multiple machines, users, and files centrally; domain admins can view accounts/computers from any machine in the domain.

🍳 **Computer Management main sections (quick rundown)**
  - **System Tools** contains:
    - **Task Scheduler** – schedule programs/tasks (example: auto-shutdown at 11:00 PM).
    - This lets you schedule programs and tasks to run at certain times, like automatically shutting off the computer at 11:00 PM every night.
    - **Event Viewer** – our system stores its system logs.
    - **Shared Folders** – This shows the folders that different users on the machine share with each other.
    - shows folders shared among users; access depends on folder permissions.
    - **Local Users and Groups** – where user and group management is performed.
    - **Performance** – monitors system resources like **CPU** and **RAM**.
    - **Device Manager** – manage hardware (network cards, sound cards, monitors, etc.).
  - **Storage** Under the storage menu, we have a sub menu for disk management.
  - contains **Disk Management** for disks/partitions.
  - **Services and Applications** the services and applications menu shows us the programs and services that we have available on the system.
  - lists services (can **enable/disable services** like DNS).

💡 **Why use Computer Management**
  - **Contains essential admin settings** not all exposed in the Settings app.
  - **More efficient for power users** than navigating the default Settings application.

🧭 **Local Users and Groups → Users view**
  - Lists built-in Windows accounts such as **Guest** and **Administrator**.
  - **Administrator account**:
    - **Disabled by default** on many systems.
    - Has **unfettered access**; remaining logged into it continuously is a security risk.
  - Example account inspection (user "Cindy"): double-click a user to view tabs and options.

🔐 **User: General tab options and implications**
  - **User must change password at next login**
    - Admins can force this (useful if an account’s password was compromised).
  - **User cannot change password** – prevents the user from modifying their password.
  - **Password never expires** – disables expiration policy for that account.
  - **Account is disabled**
    - Disabling makes the account **inactive** (cannot be used to log in).
    - Use to temporarily remove access without deleting the account.
  - **Account is locked out**
    - Locked accounts **cannot log in** (e.g., after failed attempts or as an admin response to malicious activity).

🧩 **User: Member Of tab**
  - Shows **group memberships** for the user.
  - Being a member of **Administrators** allows use of admin privileges without logging into the built-in administrator account.

🔑 **UAC (User Account Control) explained**
  - **Prevents unauthorized changes** to the system by requiring administrator approval for privileged actions.
  - When a change needs admin approval, an administrator **enters credentials** or confirms to proceed.
  - Allows day-to-day use with a standard user session while elevating only when needed.

📁 **Shared folders nuance**
  - Not all files are universally visible to other users.
  - If a user places files in a **shared folder**, anyone with **access to that shared folder** can view them; folder permissions determine visibility.

🧾 **User: Profile tab**
  - Configure **profile settings** such as the **home folder** location.
  - **More relevant in domains** where centralized home folders and roaming profiles are common.

👥 **Groups view**
  - Lists available groups and shows **their members**.
  - Use this view to audit group membership and privilege distribution.

⚙️ **Administrative actions and examples**
  - **Force password change** when compromise is suspected.
  - **Disable or lock accounts** to prevent access (e.g., disgruntled or malicious users).
  - **Add/remove users from groups** to grant or revoke privileges.
