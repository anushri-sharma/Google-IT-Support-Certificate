🔐 Root user, superuser concept, and distinction from root directory  
  – **Root User** is the initial account created during Linux installation and holds **unrestricted/superuser privileges**.  
  – **UID 0** identifies root; the system recognizes users by UID, not username.  
  – The **root account** is distinct from the filesystem **root directory (/**)**.  
  – Anyone granted root powers (e.g., via sudo) can act as a **superuser**, but the canonical root account is unique.

🧾 Protected files and command examples (how to access root-only files)  
  – Files like **/etc/sudoers** are readable only by root; attempting to read them without privilege returns **permission denied**.  
  – Run a single command as root with **sudo**, e.g., **sudo cat /etc/sudoers** (runs that command with elevated privileges).

👥 Group membership and /etc/group structure  
  – Group memberships and group info live in **/etc/group**.  
  – Each line has four colon-separated fields: **group name : password field : GID : member list**.  
  – The password field commonly shows **x**, indicating the encrypted group password is stored elsewhere (in **/etc/gshadow**), not that the root password is the group password.  
  – The system uses the **GID** (group ID) internally when applying group-based permissions.

🧭 User accounts and /etc/passwd structure  
  – User account records are in **/etc/passwd**; many entries represent system/service accounts, not human users.  
  – Important fields include: **username : password placeholder : UID : GID : GECOS : home directory : shell**.  
  – The password field usually contains **x**, meaning the encrypted password is stored in **/etc/shadow**.  
  – **Root has UID 0**; other UIDs map to other user accounts or system processes.

⚠️ Security implications and recommended practice  
  – Remaining logged in as **root** is risky because a single mistake can modify or delete critical files or services.  
  – Prefer **sudo** for executing occasional privileged commands and limit the time spent in a root shell when using **su**.  
  – You can inspect who has sudo-capable membership by checking **/etc/group** (look for the **sudo** group membership list).

🔧 Corrections and authoritative file locations (clarifying common confusions)  
  – Correct file names are **/etc/group** and **/etc/passwd** (not “/etc/groupfile” or “/etc/password”).  
  – Encrypted user passwords are stored in **/etc/shadow**; encrypted group passwords are in **/etc/gshadow**, not directly in **/etc/passwd** or **/etc/group**.
