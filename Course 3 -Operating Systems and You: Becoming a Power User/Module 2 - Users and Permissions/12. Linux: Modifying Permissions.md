🔧 Change permissions with **chmod** (change mode)  
  - **Permission sets:** owner (**u**), group (**g**), others (**o**).  
  - Use **symbolic format** (letters) to target sets and use **+** to add or **-** to remove permissions.

➕➖ Symbolic format examples and usage  
  -  - **chmod u+x my_cool_file** — add **execute** (**x**) for the owner (**u**).  
  -  - **chmod u-x my_cool_file** — remove **execute** for the owner.  
  -  - **chmod u+rx my_cool_file** — add **read** and **execute** for the owner.  
  -  - **chmod ugo+r my_cool_file** — add **read** for owner, group, and others.  
  -  - Symbolic tokens: **r** (read), **w** (write), **x** (execute); target tokens: **u**, **g**, **o**, or combinations.


This is saying we want to **add read and execute permissions for the owner of my_cool_file.**

Let's **chmod u+x, my_cool_file.**

This command is saying that we want to change the permission of my_cool_file by giving executable or x axis to the owner or u.

This format of using r, w, x and ugo to denote permissions and users in chmod is known as symbolic format.

We can also change permissions numerically, which is much faster and simpler.


🔢 Numeric (octal) format — faster for setting all sets at once  
  -  - Numeric values: **r = 4**, **w = 2**, **x = 1**.  
  -  - Add values for each permission set and place as three digits: **[owner][group][others]**.  
  -  - Example: **chmod 754 my_cool_file**  
  -  -  - **7** = 4+2+1 → owner can **read, write, execute**.  
  -  -  - **5** = 4+1 → group can **read, execute**.  
  -  -  - **4** = 4 → others can **read**.  
  -  - Numeric form is generally quicker when updating all three sets at once.

👥 Change owner and group of a file  
  -  - **chown Devan my_cool_file** — change the file owner to **Devan**.  
  -  - **chgrp best_group my_cool_file** — change the file’s group to **best_group**.  
  -  - **Privilege note:** changing the owner with **chown** typically **requires superuser (root) privileges**; **chgrp** can be used by the file owner to change the group to one they belong to.
