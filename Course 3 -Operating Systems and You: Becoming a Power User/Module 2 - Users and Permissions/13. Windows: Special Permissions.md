🔥 **Simple permissions vs. special (specific) permissions**  
  – **Simple permissions** are presets that map to multiple **special permissions** (fine‑grained NTFS permissions).  
  – Example: **Read** on a file actually enables **List folder / Read data**, **Read attributes**, **Read extended attributes**, **Read permissions**, and **Synchronize**.

🍳 **Viewing and editing special permissions in the GUI**  
  – Open the file/folder’s **Advanced** permissions dialog and select a username to see the full list of **special permissions** (individual ACEs).  
  – You can modify those special permissions directly instead of using the basic (simple) permission checkboxes.

💡 **Viewing special permissions in the CLI with icacls**  
  – Use **icacls ** (e.g., **icacls C:\Windows\Temp**) to display the DACL and the shorthand special permission flags.  
  – You can redirect output to a file for later inspection: **icacls C:\Windows\Temp > icacls.txt**.

#### CREATOR OWNER is a special user that represents the owner of whichever file the decal applies to.

In this directory, and all subdirectories, whoever owns a file or folder has full control of it.

🧭 **Understanding common icacls shorthand and flags**  
  – **WD** = WriteData (create files / write data).  
  – **AD** = AppendData (create folders / append data).  
  – **S** = Synchronize.  
  – **IO** (displayed on an ACE) = **Inherit Only** (ACE is inherited to children but does not apply to the current container).  
  – Correct term: **DACL** = Discretionary Access Control List; ACE = Access Control Entry.

The user's group includes all user accounts on the local machine.

🗂️ **Use case: C:\Windows\Temp — allow users to create but not delete others’ files**  
  – Goal: let **everyone** create files/folders in C:\Windows\Temp but prevent deleting other users’ files.  
  – Do not grant **Modify** or **Full control** to Everyone because those include delete rights.

🔐 **How C:\Windows\Temp ACLs accomplish the goal**  
  – **Administrators** and **SYSTEM / computer account** are granted **Full control** on the folder and its contents.  
  – **Everyone (Users group)** is granted specific special permissions: **WD**, **AD**, and **S** — sufficient to create files/folders but not to delete them.  
  – A separate ACE for **CREATOR OWNER** gives **the owner of each file/folder full control** over that object, allowing users to delete or fully manage only the items they own.

🧪 **Observed behavior when creating files/folders in Temp**  
  – Newly created items inherit the parent folder’s ACEs.  
  – The creator becomes the **owner** and therefore receives the **CREATOR OWNER** ACE privileges (full control for that object).  
  – Practical result: users can create items, cannot delete other users’ items, and can delete or manage their own items.

⚠️ **Implications and complexity**  
  – **Special permissions + NTFS ACLs** enable precise, custom permission models (e.g., create-only areas, owner-controlled files).  
  – They are more complex to design and audit than basic permissions; careful use of inheritance flags (e.g., **IO**, **OI**, **CI**) and identities like **CREATOR OWNER** is required to avoid unintended access.
