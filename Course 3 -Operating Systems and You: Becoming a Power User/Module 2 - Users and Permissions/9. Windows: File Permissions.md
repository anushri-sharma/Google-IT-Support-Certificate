## 🔥 Core concept: files and directories use Access Control Lists (ACLs) to control access
  - **Access control lists (ACLs)** assign who can access a file/folder and what they can do.
  - Two ACL types: **Discretionary ACL (DACL)** controls allowed/denied access; **System ACL (SACL)** configures auditing so Windows logs access events.
  - Every file/folder has an **owner** and one or more DACL entries.

## 🧭 Permission categories and their effects
  - **Read**
    - Lets you see that a file exists and **read its contents**.
    - For directories, allows reading the files and subfolders inside.
  - **Read and Execute**
    - Includes **Read** plus the ability to **run executables**.
    - Selecting **Read and Execute** will also select **Read** automatically.
  - **List Folder Contents**
    - On a directory this is an **alias for Read and Execute**; checking one checks the other.
  - **Write**
    - Lets you **modify a file** or **create/write files and subdirectories** inside a directory.
    - Notable edge case: **Write can be granted without Read**, so a user may be able to overwrite or create files without seeing their contents.
  - **Modify**
    - An umbrella permission that **includes Read, Execute, and Write**.
  - **Full Control**
    - **Includes all Modify permissions** and additionally allows **taking ownership** and **changing ACLs** for the object.

## 💻 Command-line: icacls to view/modify ACLs
  - Use the **icacls** utility (not "ICACLs"); it displays and lets you change ACLs for files/folders.
  - Example: **icacls %USERPROFILE%\Desktop** shows DACL entries for the Desktop folder.
  - The icacls output uses **letters to represent permission sets** (e.g., **F** = Full Control); the help output explains each abbreviation.
  - Common shorthand seen: **F** (Full), **M** (Modify), **RX** (Read & Execute), **R** (Read), **W** (Write).

## 🧬 Inheritance flags and behavior
  - NTFS permissions can be **inherited** from parent to child objects.
  - Common inheritance markers in icacls:
    - **OI = Object Inherit** (applies to files/objects created inside the folder)
    - **CI = Container Inherit** (applies to subdirectories created inside the folder)
  - If a folder’s DACL includes OI/CI, **new files and subfolders inherit that DACL** by default.

## ⚠️ Practical implications and edge cases
  - Granting **Full Control** permits changing ownership and ACLs—this effectively lets the user remove restrictions for others.
  - **Write without Read** enables modification without visibility—can be surprising and a security consideration.
  - **SACL entries** do not change access; they only cause Windows to **log access attempts/events** for auditing purposes.
