
# [Windows: Adding and Removing Users](https://www.youtube.com/watch?v=WtYRG0LDKcY&t=1s)

### 🖥️ GUI: Add a local user via Computer Management  
  – **Location:** Computer Management > **Local Users and Groups** > right‑click > **New User**.  
  – **Fields to set:** **User name**, **Full name**, and **Password**.  
  – **Security practice:** set a default password and check **User must change password at next log on** so the user picks a secret password on first login.  
  – **Delete:** right‑click user > **Delete**.  
  – **Warning on deletion:** **User names are unique** only in label; deleting an account removes its SID and resources linkage—recreating the same user name will not restore access to the original account’s resources.

### 💻 CLI: Add, require password change, list accounts  
  – **Command family:** use **net user** for simple CLI tasks; native PowerShell **New‑LocalUser** exists but requires scripting to supply a secure password (see supplemental reading).  
  – **Create with prompted password:** **net user andrea * /add**  
  – **List accounts:** **Get‑LocalUser** (PowerShell) to confirm creation.

### 🔁 Patterns and best practices in CLI usage  
  – **Pattern recognition:** many CLI actions follow verb patterns (create/add, set/modify, delete/remove) and similar parameter structures—observing these patterns helps discover and recall commands.  
  – **Choose method by need:** use **net user** for quick tasks; use **New‑LocalUser/Remove‑LocalUser** and related PowerShell cmdlets for scripted or more controlled automation (requires handling secure strings for passwords).
