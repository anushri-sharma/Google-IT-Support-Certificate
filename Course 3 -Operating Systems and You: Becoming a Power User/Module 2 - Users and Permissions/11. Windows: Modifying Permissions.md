
# [Windows: Modifying Permissions](https://www.youtube.com/watch?v=lP3-fWvsYqc&t=2s)

🔧 GUI: add a user and set permissions  
  - Right‑click the folder → **Properties** → **Security** → **Edit** → **Add**.  
  - Enter the username (e.g., **Devan**) → **Check Names** → **OK**.  
  - Select the user and check the **Allow** boxes for desired permissions (example: **Modify** to let Devan add photos).

⚠️ Deny vs Allow and precedence  
  - **Deny** explicitly refuses a permission for that user and **generally takes precedence** over Allow entries.  
  - Example implication: if Devan belongs to a group that is Allowed access, an explicit **Deny** on Devan’s account will still block him.  

🖥️ CLI tool: icacls overview and quoting differences  
  - Use **icacls** to modify ACLs from the command line; it was designed for cmd.exe and uses characters (parentheses) that PowerShell may interpret.  
  - In **PowerShell**, wrap parameters containing parentheses or spaces in **single quotes** to prevent interpretation (example below).

📌 Key icacls flags and meaning  
  - **(OI)(CI)** — inheritance flags: **OI = Object Inherit** (files), **CI = Container Inherit** (subfolders).  
  - Permission letters: **R = Read**, **M = Modify** (used for granting modify access).  
  - Common operations: **/grant**, **/remove**, and running **icacls ** to list ACLs.  

🧾 Concrete icacls examples (PowerShell and cmd forms)  
  - Grant **Modify** to a user (PowerShell):  
    - icacls 'C:\Path\Vacation Pictures' /grant 'Devan:(OI)(CI)M'  
  - Grant **Read** to **Everyone** (PowerShell):  
    - icacls 'C:\Path\Vacation Pictures' /grant 'Everyone:(OI)(CI)R'  
  - Grant **Read** to **Authenticated Users** (PowerShell):  
    - icacls 'C:\Path\Vacation Pictures' /grant 'Authenticated Users:(OI)(CI)R'  
  - Remove all entries for **Everyone** (either shell):  
    - icacls 'C:\Path\Vacation Pictures' /remove Everyone  
  - Verify ACLs (either shell):  
    - icacls 'C:\Path\Vacation Pictures'  
  - Notes on quoting: in cmd.exe replace single quotes with double quotes for paths/ACE strings where needed.  

👥 Groups — who is included and when to use them  
  - **Everyone** includes all local user accounts and **Guest** accounts (guest accounts are disabled by default).  
  - **Authenticated Users** includes users who logged on with credentials and **excludes Guest**.  
  - Use **Everyone** to allow literally anyone who can use the PC (including guest). Use **Authenticated Users** to exclude guest accounts.

♻️ Inheritance behavior and effects  
  - Adding an ACE with **(OI)(CI)** makes the permission inherited by newly created files and subfolders inside the folder.  
  - The transcript’s example showed newly created items inheriting the folder’s permissions.  

📝 Practical sequence demonstrated in the example  
  - GUI: added **Devan** and gave **Modify** via Properties → Security.  
  - CLI: used **icacls** to add **Everyone** read access, then switched to **Authenticated Users** to exclude guests, then removed **Everyone**, and verified the ACLs.
