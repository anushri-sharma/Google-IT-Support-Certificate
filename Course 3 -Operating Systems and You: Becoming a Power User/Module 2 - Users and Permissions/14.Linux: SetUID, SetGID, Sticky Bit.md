# [Linux: SetUID, SetGID, Sticky Bit](https://www.youtube.com/watch?v=1mGIKro2Y44&t=15s)


🔐 SetUID (set user ID)  
  – **Purpose:** allow a program to run with the **file owner’s** privileges so normal users can perform specific tasks that require elevated rights.  
  – **Example:** **/usr/bin/passwd** runs with root privileges to update **/etc/shadow**, which is owned by root.  
  – **Indicator:** an **s/S** appears where the owner execute bit would be (**lowercase s** if owner execute is set, **uppercase S** if not).  
  – **How to enable:** symbolic: **chmod u+s ** (or **chmod +s **); numeric: **prepend 4**, e.g., **chmod 4755 **.  
  – **Implication:** grants elevated privileges to the running process; setuid binaries must be minimized and securely implemented to avoid privilege escalation.

If you need to change a file owned by root, you'd have to use sudo.

🧩 SetGID (set group ID)  
  – **Purpose:** run a program with the **file group’s** privileges or cause new files in a directory to inherit the directory’s group.  
  – **Example:** a program with setgid set and group owner **tty** runs as group **tty** when executed.  
  – **Indicator:** an **s/S** appears in the group execute position (**lowercase s** if group execute is set, **uppercase S** if not).  
  – **How to enable:** symbolic: **chmod g+s **; numeric: **prepend 2**, e.g., **chmod 2755 **.  
  – **Implication:** useful for collaborative/group workflows but can introduce privilege risks similar to setuid if misused.

📌 Sticky bit  
  – **Purpose:** on directories, prevents users from deleting or renaming files they do not own; others can still create and modify their own files.  
  – **Example:** **/tmp** is typically sticky; users can add files there but generally cannot delete files owned by other users.  
  – **Indicator:** a **t/T** appears in the others execute position (**lowercase t** if execute is set, **uppercase T** if not).  
  – **How to enable:** symbolic: **chmod +t ** (or **chmod o+t **); numeric: **prepend 1**, e.g., **chmod 1777 ** for a world-writable sticky directory.  
  – **Implication:** enables shared writable directories while protecting users’ files from deletion by others.

⚠️ Practical and security notes  
  – **Common use case:** allow limited privileged actions (e.g., password changes) without granting full root access.  
  – **Numeric combinations:** special bits combine by adding digits (4=setuid, 2=setgid, 1=sticky); e.g., **chmod 6755** sets setuid+setgid plus the listed permissions.  
  – **Security caution:** setuid/setgid binaries are high-risk attack surfaces; minimize their number, apply least privilege, and ensure code is secure and updated.  
  – **Operational note:** these bits are not changed frequently day-to-day but are fundamental for controlled sharing and delegated privileges.
