
# [Windows: Passwords](https://www.youtube.com/watch?v=W5cnH0-x5vM&t=6s)

🔐 Password purpose and rules  
  - **Passwords secure user accounts** so only the individual user can access their account; administrators should **not** know other users’ passwords.  
  - **If you manage someone’s account, have the user enter the password or force a change** rather than learning or setting their password yourself.

🖥️ GUI: force password change or set a password  
  - To force a change: **Computer Management → Local Users and Groups → right‑click user → Properties → check “User must change password at next login” → Apply/OK**.  
  - To reset a forgotten password: right‑click user → **Set Password**.  
  - **Caveat:** resetting someone’s password can cause **loss of access to encrypted or stored credentials** (examples: EFS/DPAPI‑protected files, saved network/RDP credentials, credential manager entries); only reset when necessary and notify the user.

🛠️ CLI/PowerShell: net user and PowerShell alternatives  
  - Legacy convenient option: **net user**.  
  - Safer interactive entry: use **net user * ** to have the command prompt you for the password rather than putting it on the command line.  
  - **Why use the asterisk prompt:** prevents **shoulder‑surfing** and avoids putting passwords into **command history or centralized logs** that may capture CLI activity.  
  - Privacy caveat: if you type a password on behalf of another user, **you will know their password**—avoid this; instead require a next‑logon change.  
  - To require the user to change the password on their next logon, set the equivalent **“must change password at next login”** flag from GUI or via the appropriate CLI/PowerShell parameter or script (consult **net user /?** or use scripted PowerShell).  
  - Native PowerShell cmdlets exist (e.g., **Set-LocalUser** / **New-LocalUser**), but they require creating a **SecureString** and minimal scripting, for example: **Set-LocalUser -Name 'username' -Password (ConvertTo-SecureString 'P@ssw0rd' -AsPlainText -Force)** (using plain text in scripts is insecure; prefer prompting).

⚠️ Security best practices and operational implications  
  - **Do not learn or retain other users’ passwords.**  
  - Prefer forcing a password change by the user rather than assigning a password you know.  
  - **Never embed passwords in CLI commands** because they can be captured in logs or audit trails.  
  - When resetting passwords, **warn users about possible loss of access to encrypted or stored credentials** and coordinate recovery if needed.
