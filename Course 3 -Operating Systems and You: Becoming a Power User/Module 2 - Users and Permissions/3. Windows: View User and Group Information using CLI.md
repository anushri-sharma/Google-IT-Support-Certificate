
# [Windows: View User and Group Information using CLI](https://www.youtube.com/watch?v=MA21rf5RTR8&t=6s)

### 🔎 Problem & faster approach  
  - **Manual GUI method**: open Computer Management → Local Users and Groups → inspect each user; slow for multiple machines.  
  - **CLI alternative**: use PowerShell local account cmdlets to audit users and groups across machines much faster.

### 💻 List local users — Get-LocalUser  
  - Run **Get-LocalUser** to display all local user accounts on the machine.  
  - Output includes **usernames, default Windows accounts**, and an **Enabled status** so you can tell if an account (e.g., the local Administrator) is enabled or disabled.

### 🧩 List local groups — Get-LocalGroup  
  - Run **Get-LocalGroup** to list all groups defined on the local machine.  
  - Most groups returned are **built-in** system groups and typically **should not be changed.**

<img width="1280" height="528" alt="image" src="https://github.com/user-attachments/assets/83307e8b-aebc-4dfe-a40e-6f514a610239" />

### 🔑 Check group membership — Get-LocalGroupMember (Administrators example)  
  - Use **Get-LocalGroupMember -Group Administrators** to see who has administrative rights on the machine.  
  - The **Administrators** group controls who can make any change on the system; auditing this group tells you who effectively has full control.  
  - Example outcome: shows the built-in **Administrator** account and your user account as members; no additional members means the admin set looks appropriate.

### ⚙️ Requirements & environment note  
  - These local account PowerShell cmdlets require **PowerShell 5.1 or newer**.  
  - When an organization uses **Active Directory**, user accounts are often managed centrally and may not appear as **local accounts**; these cmdlets **target local accounts/groups only.**
