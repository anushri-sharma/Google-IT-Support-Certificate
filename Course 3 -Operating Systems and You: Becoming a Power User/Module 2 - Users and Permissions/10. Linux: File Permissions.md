
# [Linux: File Permissions](https://www.youtube.com/watch?v=T80-RexUPg4&t=3s)

🔥 **Core concept: Linux permissions control who can read, modify, or run files and directories.**
  
  - **Read (r)**: allows viewing the contents of a file or listing a directory.  
  - **Write (w)**: allows modifying a file or changing directory contents.  
  - **Execute (x)**: allows running a file as a program or entering a directory.

🍳 **How to view permissions: use the long listing with `ls -l`.**
  
  - **`ls -l`** displays a 10-character permission string, owner, group, and other metadata for each file.

<img width="1278" height="176" alt="image" src="https://github.com/user-attachments/assets/d7a68fb3-dfb3-4d45-bb29-17d64b201bd7" />

💡 **Structure of the 10-character permission string (example: `-rwxrw-r--`).**
  
  - The **first character** is the **file type**: **`-`** = regular file, **`d`** = directory (and other letters for special types).  
  - The remaining **9 characters** are permission bits, grouped into three trios: **owner**, **group**, **others**.

🔍 **Meaning of each trio (left-to-right): owner, group, others.**
  
  - First trio = permissions for the **user who owns** the file.  
  - Second trio = permissions for the **group** the file belongs to.  
  - Third trio = permissions for **all other users**.

📌 **Interpreting letters and dashes in the trios.**
  
  - A letter (**r, w, x**) means that permission is **enabled**; a dash (`-`) means it is **disabled**.  
  - Analogy to binary: a set bit is enabled, an unset bit (dash) is disabled.

🧾 **Concrete example explained (`-rwxrw-r--`).**
  
  - **Owner** (first trio `rwx`): can **read, write, and execute** the file (in the example owner is **Cindy**).  
  - **Group** (second trio `rw-`): group **"cool"** can **read and write** but **not execute**.  
  - **Others** (third trio `r--`): everyone else can **only read** the file.

⚙️ **Implications and practical note.**
  
  - Permissions allow fine-grained control based on role (**owner**, **group**, **others**) rather than just a global setting.  
  - Reading permission strings becomes intuitive with practice; reviewing examples helps build speed.
