
# [Windows: Reading Process Information](https://www.youtube.com/watch?v=irg3HYJ_Muo&t=5s)

🔥 **Processes = programs in motion (concept)**  
  – **Programs on disk** become **processes** when the OS loads and runs them (example: a browser’s code on the hard drive becomes a running application when started).  
  – Processes represent **running, responding, working applications** you interact with (launching, halting).  
  – Understanding processes reveals how the OS manages execution and is useful for **IT support and troubleshooting**.

🍳 **Windows Task Manager (taskmgr.exe) — quick access and view**  
  – Open with **Ctrl+Shift+Esc** or via the **Start menu**.  
  – The **Processes tab** lists processes the **current user** is running and **some system-level processes visible to that user**.  
  – Columns show **application/image name**, **user who launched it**, and resource usage such as **CPU** and **memory**.

💥 **Ending a process via Task Manager**  
  – Select a process row and click **End task** (lower right) to terminate it (example demo: launched **notepad.exe** from the command line then ended it in Task Manager).  
  – Terminating a process kills that instance of the program immediately; this is a primary method to stop unresponsive applications.

🔎 **Finding process IDs (PID) in Task Manager**  
  – Use the **Details tab** in Task Manager to display additional information including the **PID** for each process.  
  – The PID is required for command-line termination or more granular management.

💻 **Command-line alternatives: Command Prompt and PowerShell**  
  – From Command Prompt use **tasklist** to show running processes.  
  – To stop processes by PID from the command line, use **taskkill** (commonly used with a **/PID** argument).

🧭 **Multiple methods, choose by need**  
  – Task Manager provides a graphical, quick overview and a simple kill button.  
  – Command Prompt/PowerShell provide scripting, automation, and PID-based control useful for advanced troubleshooting or remote management.


⚠️ **Visibility and permission nuances**  
  – Task Manager’s Processes tab is scoped to the current user; **not all system processes are shown** there.  
  – Some system or elevated processes may require administrative privileges to view or terminate (use Details tab or elevate privileges as needed).
