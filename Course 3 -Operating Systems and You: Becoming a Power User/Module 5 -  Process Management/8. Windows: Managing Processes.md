
# [Windows: Managing Processes](https://www.youtube.com/watch?v=1VBtjcCOrCU&t=12s)


🔥 **Process Explorer overview and purpose**  
  – **Process Explorer** is a Microsoft utility for inspecting and managing running processes, aimed at IT support and system administrators.  
  – It is **not built into Windows**; it must be **downloaded from Microsoft's website**.  

<img width="1276" height="400" alt="image" src="https://github.com/user-attachments/assets/b1668057-04d7-4d2c-8d5e-068f6a67a513" />

💡 **Primary UI: two-pane view**  
  – The **top pane** lists currently active processes.  
  – The **bottom pane** shows files/handles a selected process is using, useful to identify which process has a file open or to inspect what a process is doing.  

🔎 **Searching for processes**  
  – Invoke search with **Ctrl+F** or the binocular button.  
  – Example: a search can return **C:\Windows\System32\notepad.exe** and related entries such as **notepad.exe.mui** (**MUI = Multilingual User Interface**, a language-support package).

🌳 **Process tree / parent-child relationships**  
  – Processes appear nested to indicate **parent → child** relationships (example: **notepad.exe** shown under **command.exe**).  
  – Parentage matters for understanding origin and for some management actions (see Kill Process Tree).  

<img width="1279" height="717" alt="image" src="https://github.com/user-attachments/assets/c6c0255d-2d82-4a03-9135-6b0e6ef70b5e" />

🗡️ **Kill actions**  
  – **Kill Process** (**Q process**) terminates the selected process only.  
  – **Kill Process Tree** (**Q process tree**) terminates the selected process and **all of its descendants** (child processes), with no selective exceptions.  

<img width="1280" height="717" alt="image" src="https://github.com/user-attachments/assets/2f0644b7-b6e7-43cc-8b0c-327f62e1e7b6" />

🔁 **Restart behavior and re-parenting**  
  – **Restart** stops and then starts the process.  
  - Let's restart with the notepad.exe process.
  - We started from command.exe.
  - Interesting, after the restart, notepad.exe doesn't appear as a child of command.exe anymore.
  – A restarted process may be **re-parented** to the process that performed the restart (example: after restarting notepad.exe from Process Explorer, it becomes a child of **procexp.exe**).  
  – Re-parenting occurs because the restarting process (Process Explorer) is the new creator/launcher.  


- if you search for notepad.exe again, we can see it's been restarted as a child of the procexp.exe process.

- This is the process name for Process Explorer.

- This makes sense since Process Explorer was the process in charge of starting it again after we terminated it.


⏸️ **Suspend / Resume**  
  – **Suspend** pauses a process instead of killing it; the **CPU column** will show **“suspended”**.  
  – While suspended, the process **does not consume the CPU/time resources** it used when active.  
  – **Resume** continues the suspended process from where it left off.  

- Instead of killing a process, you can use this option to suspend it and potentially continue it at a later time.

- If we right-click suspend the process, we'll see that in the CPU column, the Process Explorer output, the word suspended appears.

🔧 **Practical uses and implications**  
  – Use the bottom pane to **find which process holds or uses a file** (helps diagnose file-locking issues).  
  – Suspend can be used for temporary resource reduction or troubleshooting without losing process state.  
  – Kill Process Tree must be used with caution because it will stop all descendant processes.  

📚 **Further information**  
  – Process Explorer provides additional monitoring details and advanced features; official **documentation is available on Microsoft’s website**.
