
# [Linux: Reading Process Information](https://www.youtube.com/watch?v=DSOERdAHlwM&t=4s)

🔥 **Basic command: ps -x**  
  – **ps -x** shows a snapshot of the current processes for the invoking user.  
  – Output can be dense; inspect columns to interpret each process entry.  
  – Snapshot is static (point-in-time), not a live/streaming view.


**We'll be using the ps command - to view the processes running on our system and Linux.**

🔎 **Common columns explained**  
  – **PID** — **process ID**, unique identifier assigned at launch.  
  – **TTY** — terminal associated with the process (if any).  
  – **STAT** — process status code: **R** = running / runnable, **T** = stopped (suspended), **S** = interruptible sleep (waiting for an event).  
  – **TIME** — total **CPU time** consumed by the process.  
  – **CMD** / **COMMAND** — name or command line used to run the process.  
  – Additional STAT codes exist; consult the man pages for full meanings.

🧰 **Expanded view: ps -ef**  
  – **-e** = show **all processes** (including other users).  
  – **-f** = **full** format with more fields/verbosity.  
  – Common columns in **ps -ef**: **UID** (user), **PID**, **PPID** (parent PID), **C**, **STIME**, **TTY**, **TIME**, **CMD**.  
  – **C** represents recent **CPU utilization** (not number of child processes).  
  – **PPID** shows which process spawned the entry; **STIME** is the process **start time**.


Run this command, ps-ef.

The **e flag is used to get all processes**, even the ones run by other users.

The **-f flag is for full**, which shows you full details about a process.

🔍 **Filtering process output**  
  – Use shell filtering, e.g., **ps -ef | grep Chrome** to show lines containing “Chrome”.  
  – Grep helps find matching processes but will also match the grep command itself unless handled (standard grep caveats).

<img width="1270" height="120" alt="image" src="https://github.com/user-attachments/assets/559fff80-c22a-4a3f-96b5-f988b4348a00" />


📁 **Processes exposed via /proc**  
  – Every running process has a directory under **/proc/** named by its PID (e.g., **/proc/1805/**).  
  – Files like **/proc//status** provide detailed internal state information beyond ps fields.  
  – **/proc** is comprehensive for inspection but verbose and less convenient for routine troubleshooting.

💡 **Practical recommendation**  
  – Use **ps -ef** for most troubleshooting and quick overviews.  
  – Dive into **/proc/** when you need low-level, per-process details not shown by ps.
