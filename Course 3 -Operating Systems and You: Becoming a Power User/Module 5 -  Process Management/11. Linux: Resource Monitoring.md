
 # [Linux: Resource Monitoring](https://www.youtube.com/watch?v=pLSN1lq7vPE&t=4s)

 🔥 **top**: real-time view of processes and system resource use  
  - Shows the **top processes using the most resources**, plus a quick snapshot of **total tasks** (running/idle), **CPU usage**, **memory usage**, and more.  
  - Pay attention to the **percentage CPU** and **percentage MEM** columns to see **per-process** CPU and memory consumption.  
  - Exit the utility with **q**.

🍳 Investigating slow systems (common cause: resource overuse)  
  - If a machine is slow, overuse of **CPU** or **memory** is a common cause; use **top** to identify high-usage processes.  
  - **Investigate** what a heavy process is doing before acting; you may **terminate** a process to immediately free resources if appropriate.

⏱️ **uptime**: time, uptime, users, and load averages  
  - Reports current time, how long the system has been up, number of logged-on users, and the **load average**.  
  - Example sample output: current time **16:43**, uptime **5 hours 8 minutes**, **1** user.  
  - **Load averages** represent average CPU load over **1, 5, and 15 minute** intervals and are useful for observing system behavior over time.  
  - Load averages are a nuanced metric; consult supplemental reading for detailed interpretation.

🧭 **lsof**: find open files and the processes using them  
  - Lists open files and the processes that hold them, useful for tracking file locks and resource ownership.  
  - Practical example: when ejecting a USB yields the error **"device or resource busy"**, run **lsof** to find which process is still accessing files on the device.

🖥️ Monitoring hardware separately from processes  
  - You can monitor CPU and memory metrics independently of per-process views using various commands and tools.  
  - Per-machine metrics are less useful at scale; centralized monitoring across a **fleet of machines** is important for broader operational visibility (implementation not covered here).

🎯 Operational relevance for IT support specialists  
  - Reading process information and managing processes is essential for troubleshooting performance and resource issues in support roles.
