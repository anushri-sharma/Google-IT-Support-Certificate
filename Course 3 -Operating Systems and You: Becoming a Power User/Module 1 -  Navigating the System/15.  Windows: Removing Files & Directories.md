
# [Windows: Removing Files & Directories](https://www.youtube.com/watch?v=lDhg9HJH7r0&t=2s)

🔥 **GUI deletion vs PowerShell deletion**  
  – **GUI:** right-click → **Delete** sends items to the **Recycle Bin**, allowing restore via **Restore**; emptying the bin makes recovery impossible.  
  – **PowerShell:** use **RM** or **remove** (alias for Remove-Item); this **does not** use the Recycle Bin, so removals are immediate and permanent.

🛠️ **Basic remove usage and immediate effect**  
  – Example: removing **text1.txt** from the home directory removes the file immediately (no recycle).  
  – Because removals bypass the Recycle Bin, accidental deletes are irreversible through normal GUI restore.

🔐 **Permissions and system-file protections**  
  – Attempts to remove files can fail with a permission error if the file is marked as a **system file** or the user lacks file-system permissions.  
  – File-system restrictions are enforced so only authorized users can delete protected files; to delete a file owned by another user or requiring elevated rights, you must use an **administrator account**.

⚠️ **Using -Force to override protections and prompts**  
  – If PowerShell warns you about an important or protected file, re-running the command with **-Force** will remove it despite the caution.  
  – **Implication:** **-Force** can permanently delete critical system files; use only when you understand the consequences.

📂 **Removing directories and recursion behavior**  
  – Attempting to remove a directory that contains files without **-Recurse** triggers an interactive confirmation prompt asking to confirm deletion of the directory and all contents.  
  – You can cancel the prompt and re-run the command with **-Recurse** to indicate explicit intent to delete the directory tree; with **-Recurse**, removal proceeds without the interactive confirmation.

🧭 **Practical precautions and consequences**  
  – Remove operations are destructive: **no automatic recovery** via the Recycle Bin.  
  – Rely on file permissions and interactive prompts as safeguards, but use **-Force** and **-Recurse** only when you intentionally accept permanent deletion.
