
# [Linux: Input, Output and Pipeline](https://www.youtube.com/watch?v=TlsPv2MRhZI&t=4s)

🔥 **Three standard streams: stdin (0), stdout (1), stderr (2)**  
  - **stdin** is input to commands (file descriptor **0**).  
  - **stdout** is normal output from commands (file descriptor **1**).  
  - **stderr** is where error messages go (file descriptor **2**).  

🍳 **Redirect stdout: overwrite with >**  
  - Example: `echo woof > dog.txt`  
  - Effect: **overwrites** any existing **dog.txt** with the text **woof**.  
  - Implication: use with care to avoid data loss.

🍳 **Append stdout: >>**  
  - Example: `echo woof >> dog.txt`  
  - Effect: **appends** the output to the end of **dog.txt** instead of overwriting.  

<img width="1280" height="418" alt="image" src="https://github.com/user-attachments/assets/8bb864c6-d397-4f0d-a92e-a48e3fff0c73" />


💡 **Redirect stdin from a file: <**  
  - Example: `command < file_input` is functionally the same as `cat file_input | command` for many uses.  
  - Use case: feed a file as a command’s input instead of typing on the keyboard.

🛑 **Redirect stderr: 2>**  
  - Example: `ls /dir/fake_dir 2> error_output.txt`  
  - Effect: captures **only error messages** into **error_output.txt** (stdout still goes to the terminal).  
  - Tip: the leading **2** denotes the **stderr** file descriptor.

<img width="1280" height="201" alt="image" src="https://github.com/user-attachments/assets/dd375d1c-bbf3-4e05-bb67-bac6dda3fd6c" />


🗑️ **Discard unwanted output: /dev/null**  
  - Example (discard errors): `ls /dir/fake_dir 2>/dev/null`  
  - Effect: sends errors to the OS “black hole” so they don’t appear or get saved.  
  - Note: `/dev/null` is the Unix/Linux equivalent of Windows’ null sink.

🔗 **Pipes: send stdout into another command using |**  
  - Example: `ls -la /etc | grep Bluetooth`  
  - Effect: **pipes stdout of the first command into stdin of the second**, enabling quick filtering/searching without opening directories manually.  
  - Use case: chain commands to transform/filter data streams.

🧭 **Common redirection/combination nuances and alternatives**  
  - **Redirect both stdout and stderr to the same file:** `command > file 2>&1` (portable) or `command &> file` (bash/zsh shorthand).  
  - **Append both streams:** `command >> file 2>&1`.  
  - **Redirect only stdout but keep stderr visible:** `command > out.txt` (stderr stays on terminal).  
  - Remember file descriptor numbers: **0=stdin, 1=stdout, 2=stderr** when composing complex redirects.

🔎 **Practical implications**  
  - Redirectors let you capture, log, filter, or discard different kinds of output precisely.  
  - They are essential for scripting, debugging, and automating command output handling.
