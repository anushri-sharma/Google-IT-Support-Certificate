## 🖥️ Two Windows command-line interfaces  
  – **cmd.exe** (Command Prompt): legacy shell similar to MS‑DOS.  
  – **PowerShell** (**powershell.exe**): modern shell that **supports most cmd.exe commands and many more**, chosen here for exercises.

## ⚙️ PowerShell aliases and familiarity  
  – Many commands are **aliases** for common shell commands; **"An alias is sort of like a nickname for a command."**  
  – Implication: familiar commands (e.g., **ls**) work in PowerShell even if they originated in other shells.

## 📂 Listing directories with ls (list directory)  
  – Use **ls** (or "list directory") plus a **path parameter** to view contents; the path is a parameter, not part of the command itself.  
  – Example target: the **root of the C drive** (where Windows is installed; often the only drive).  
  – Output varies by machine — you may see few or many directories depending on use.

## 🧩 Parent vs. child directories (hierarchy terminology)  
  – **Parent directory**: container folder (e.g., C:\ root).  
  – **Child directory**: items nested inside a parent (example: folder **dogs** containing child **Corgi**).  
  – Implication: common OS term for hierarchical relationships when navigating filesystems.

## 📁 Common child directories in C:\ and their roles  
  – **Program Files (x86)**: contains most installed applications and programs (32‑bit apps).  
  – **Users**: contains user profile / home directories; each Windows user gets their own folder.  
  – **Windows**: contains the Windows operating system files.

## 📚 Using Get-Help to learn commands  
  – **Get-Help ls**: shows a brief summary and parameters for **ls**.  
  – **Get-Help ls -Full**: shows detailed descriptions of each parameter and usage examples.  
  – Implication: use Get-Help to discover capabilities and correct syntax for PowerShell commands.

## 🔒 Revealing hidden and system files with -Force  
  – Add the **-Force** parameter to **ls** to list hidden and system files that default ls omits.  
  – Examples revealed when using **-Force**: **Recycle Bin** (where moved files are stored instead of immediate deletion), **ProgramData** (stores data used by programs installed in Program Files).  
  – Implication: **-Force** exposes important system directories and items normally hidden from casual directory listings.
