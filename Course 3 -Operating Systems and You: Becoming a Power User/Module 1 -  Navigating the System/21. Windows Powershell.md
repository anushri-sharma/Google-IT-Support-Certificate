
# [Windows Powershell](https://www.youtube.com/watch?v=z2Fse7xxstE&t=1s)

🔥 PowerShell is a **complex, powerful, and robust** command language; common Linux-like **aliases** are available but **real PowerShell commands** are needed for advanced features.  

💡 Use **Get-Alias** to see what an alias actually runs.  
  – **Get-Alias ls** reveals **ls** maps to **Get-ChildItem**.  
  – **Get-ChildItem** "gets or lists the children" (files and subdirectories) of a given item.

🔁 Example equivalence: **Get-ChildItem C:\** produces the same output as **ls C:\**.  
  – Demonstrates that aliases are convenience wrappers around longer PowerShell cmdlets.

⌨️ PowerShell cmdlet naming tradeoff: **long, descriptive names** improve readability but require more typing; **aliases** speed interactive work.  
  – Aliases are appropriate for quicker interactive use; full cmdlets provide clarity and access to cmdlet-specific features.

<img width="1280" height="285" alt="image" src="https://github.com/user-attachments/assets/e79e2299-2dd0-4666-9d14-e28788da18bc" />


🧭 Three ways to run commands on Windows:  
  – **PowerShell cmdlets** (native PowerShell commands like **Get-ChildItem**).  
  – **Aliases** (shorthand names like **ls** that map to cmdlets).  
  – **cmd.exe commands** (legacy MS‑DOS commands available for backward compatibility).

⚠️ Differences between PowerShell and cmd.exe commands:  
  – **cmd.exe commands** are older and generally **less powerful** than PowerShell cmdlets.  
  – Example: **dir** (a cmd.exe command) also maps to **Get-ChildItem** in PowerShell via compatibility/aliasing.

❓ Help systems differ by command type:  
  – Use **Get-Help** for **PowerShell cmdlets** and aliases (example: **Get-Help ls**).  
  – Use **/ ?** (slash question mark) for **cmd.exe** commands (example: **dir /?**).  
  – Mixing them fails: **ls /?** returns nothing because the underlying cmdlet (**Get-ChildItem**) does not understand the **/ ?** parameter.
