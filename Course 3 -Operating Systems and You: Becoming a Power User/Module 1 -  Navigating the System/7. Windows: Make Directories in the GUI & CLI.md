 🖱️ Create a folder in the GUI  
  – **Right-click → New → Folder** to make a new directory visually.

🔧 Create a folder in the CLI (PowerShell)  
  – Use **mkdir** (short for **make directory**) to create folders from the command line.  
  – Example: **mkdir my_cool_folder** creates a folder named **my_cool_folder**.

<img width="1280" height="421" alt="image" src="https://github.com/user-attachments/assets/59d39caa-16a1-4f11-af4c-3f7b74370d3e" />


⚠️ Space characters in folder names cause errors if unquoted  
  – Running **mkdir my cool folder** fails because the shell treats **cool** and **folder** as separate arguments or parameters, not part of the single name.

<img width="1274" height="223" alt="image" src="https://github.com/user-attachments/assets/2c7278a0-45d5-4ff6-abd7-4c41e35cba1d" />


💡 Use quotes to include spaces in a name  
  – Put the entire name in quotes so the shell treats it as **one argument**.  
  – Example: **mkdir 'my cool folder'** creates a folder with spaces in its name.

<img width="1278" height="108" alt="image" src="https://github.com/user-attachments/assets/801400a8-344a-4a0c-a44f-ed5e67635c28" />


⛓️ Escape spaces with the shell’s escape character (PowerShell example)  
  – In **PowerShell**, the **escape character** is the **back tick**. Place it before a space to include that space in the filename.  
  – Example: **mkdir my` cool` folder** (back tick before each space) makes a single folder named **my cool folder**.  
  – Escaping means the next character is treated **literally** instead of as syntax.

<img width="1272" height="133" alt="image" src="https://github.com/user-attachments/assets/f404ed34-4ca7-4a87-9007-aec961ec312a" />


🌐 Escape-character note across environments  
  – **PowerShell** uses the **back tick** as its escape character.  
  – **Other shells and languages** may use a different escape character (for example, **backslash \\** in many Unix shells), so use the appropriate escape method for the environment.
