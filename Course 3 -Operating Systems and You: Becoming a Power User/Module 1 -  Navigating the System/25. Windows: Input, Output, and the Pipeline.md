🔥 Creating and inspecting simple output files with PowerShell  
  - Use `echo` (an alias for **Write-Output**) to produce text to the standard output; example: `echo woof > dog.txt` creates or overwrites **dog.txt** with "woof".  
  - `ls` or `cat` can be used to confirm the file exists and view its contents.

🌊 Concept of I/O streams (stdin, stdout, stderr)  
  - Every process and PowerShell command can accept input and produce output via **three streams**: **standard in**, **standard out**, and **standard error**.  
  - Keyboard input goes to **standard in**; the process writes normal results to **standard out**, which the CLI displays.  
  - Error messages are written to **standard error**, separate from normal output.

➡️ Redirecting standard output with > and >>  
  - The **`>`** operator redirects **standard out** to a file; it **overwrites** the file if it exists or creates a new one if not.  
  - The **`>>`** operator **appends** standard out to the end of an existing file; example: `echo woof >> dog.txt` adds another "woof".

| Pipe operator: passing output to another command  
  - if we wanted to send the output of one command to the input of another command.
  - The pipe `|` passes the **standard out** of one command into the **standard in** of the next, enabling composition of simple tools into complex workflows.  
  - Example pipeline: `**cat words.txt | select-string st**` filters lines from `words.txt` that contain the substring **st**.  
  - You can combine piping and redirection: `**cat words.txt | select-string st > st_words.txt**` writes the filtered result to **st_words.txt**.

⚠️ Standard error and stream numbering  
  - Output streams are numbered for redirection: **1** = **standard out**, **2** = **standard error**.  
  - To capture error messages separately, redirect stream 2: `rm secure_file 2>errors.txt` writes the error output to **errors.txt** instead of displaying it.

🕳️ Discarding unwanted error messages with $null  
  - `$null` is a special PowerShell variable representing **nothing**; it's used as a sink/black hole for redirection.  
  - To suppress error messages entirely: `rm secure_file 2>$null` redirects standard error to **$null**, removing error output from the console.

🔧 Practical implications, edge cases and tips  
  - Using `>` will destroy existing file contents; use `>>` to preserve and append instead.  
  - Errors from restricted/protected operations (e.g., permission denied) go to **standard error**, so they must be redirected with the stream number to capture or suppress them.  
  - The redirection syntax can target any numbered stream; you must specify the stream number when redirecting non-default streams.
