
# [Windows: Display File Contents](https://www.youtube.com/watch?v=MWfpLI4DKGM&t=1s)

🔥 Double-click in Windows GUI to view files  
  – **Default application for text files is Notepad**; change it via **Right-click → Properties → Open With** to use another editor (e.g., WordPad).  
  – Course focus: **text and configuration files**, not images or media.

💡 PowerShell quick view: cat  
  – **cat** is an alias for **Get-Content** (the name comes from **concatenate**).  
  – Running **cat ** **dumps the entire file to the shell** (prints everything until done).

📄 Paging long files: more  
  – Use **more** to view one screen at a time (e.g., **Get-Content  | more** or **more **).  
  – **More launches a separate pager program**, so navigation uses that program’s keys.  
  – Key controls:  
    – **Enter** advances **one line**.  
    – **Space** advances **one page** (page size depends on terminal window).  
    – **Q** quits the pager and returns to the shell.

🔎 View just the start: head  
  – Use **Get-Content -Head  ** (alias: **cat -Head  **) to show the **first n lines**.  
  – If no number is specified in examples here, the typical default shown is **10 lines**.

<img width="1274" height="402" alt="image" src="https://github.com/user-attachments/assets/48fd58c6-83c6-4c7b-b753-133a3464cf45" />


🔚 View just the end: tail  
  – Use **Get-Content -Tail  ** (alias: **cat -Tail  **) to show the **last n lines**.  
  – Default behavior discussed: **last 10 lines** if a count is not specified.

<img width="1279" height="391" alt="image" src="https://github.com/user-attachments/assets/828d31f1-6d8b-4451-ac99-2ad635d9da73" />


🛠 Practical implication for workflows  
  – Use **cat/Get-Content** for full dumps, **more** to read interactively, and **-Head/-Tail** to quickly inspect file starts/ends without opening the whole file.
