🔧 **Installing software from source on Linux**  
  – **Installation method varies by programming language and build tools**, because languages are compiled or packaged differently.  
  – You can install directly from a source archive, but the exact steps depend on the software.

🗂 **Typical contents of a source archive (example: “flappy app package”)**  
  – **README**: a standard file with information and instructions; it is intended to be read before proceeding.  
  – **setup script**: a script that runs the installation tasks.  
  – **source code**: the actual program files that the setup script typically builds or deploys.

📝 **What a setup script does (examples and mechanics)**  
  – **Compile** source code into **machine instructions** (when needed).  
  – **Copy compiled binary** to system locations such as **/bin**.  
  – **Create user folders** or application directories under **/home/** or other paths.  
  – These are illustrative tasks; a real setup script can run any sequence of operations the developer programs.

🧩 **Who decides what installation does and why it varies**  
  – **Developers define the required tasks and dependencies**; installation behavior is determined by the script or build instructions they provide.  
  – Tasks may include creating files, updating directories, compiling, or other setup steps specific to the software.  
  – If you understand the programming languages and build instructions, you can read the setup script to see exactly what will run.

💡 **Key implication**  
  – **Installation behavior is not standardized across source packages**: expect variability and inspect the **README** and **setup script** to know what will happen on your system.
