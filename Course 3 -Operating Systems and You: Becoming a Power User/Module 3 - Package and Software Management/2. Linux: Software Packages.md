
# [Linux: Software Packages](https://www.youtube.com/watch?v=wXXbfmigm5w&t=8s)

🔧 Package types vary by Linux distribution  
  – **Red Hat** families use **.rpm** (RPM / Red Hat Package Manager) packages.  
  – **Debian/Ubuntu** families use **.deb** (Debian package) files.  
  – **Implication:** package file type and tooling change between distros; you must use the appropriate package manager for that distro.

📦 Debian packages (.deb) and standalone downloads  
  – Developers may distribute software as standalone **.deb** files on websites; these are installed locally (not via distro repos).  
  – **Tool:** use **dpkg** (Debian package command) to work with local **.deb** files.  
  – **Example:** the Atom editor distributed as a **.deb** file can be installed directly with **dpkg**.  

➕ Install a Debian package  
  – Command: **dpkg -i .deb** (the **-i** flag = install).  
  – **Practical note:** installing local **.deb** often requires **root/sudo**.  
  – **Edge case:** **dpkg** does not resolve or fetch dependencies automatically; if dependencies are missing, installation can fail. Use **apt** or **apt-get** (e.g., **apt -f install**) to fix missing dependencies after using **dpkg**.

➖ Remove a Debian package  
  – Command: **dpkg -r ** (the **-r** flag = remove).  
  – **Nuance:** **dpkg -r** removes the package but may leave configuration files; use **dpkg --purge** (or **dpkg -P**) to remove configuration files as well.

🔎 Search installed packages with grep and piping  
  – Combine commands: **dpkg -l | grep ** to filter the package list for a specific string.  
  – **Pipe behavior:** the pipe sends the standard output of **dpkg -l** into the standard input of **grep**.
