
# [Windows: Package Manager](https://www.youtube.com/watch?v=wLZshRt1obM&t=3s)


🔥 **Typical Windows install/update/remove workflow and its limitations**  
  – Normal steps: **search web → download installer → run installer**; updates often via each program’s own mechanism or **Windows Update**.  
  – Removal usually via **Add/Remove Programs** or a program-specific uninstaller.  
  – **Windows Installer** can handle some dependency management but does **not** provide a central package catalog, consistent automatic updates, or easy automated enterprise deployment.

💡 **Chocolatey — a Windows package manager overview**  
  – **Chocolatey** is a **third-party** package manager for Windows (not written by Microsoft).  
  – Enables installing Windows applications from the **command line**.  
  – Built on existing Windows tech, primarily **PowerShell**.
  - **Chocolatey is built on some existing Windows technologies like PowerShell and lets you install any package or software that exists in the public Chocolatey repository.**
  - You can add any software that might be missing to the public repository.
  - You can even create your own private repository if you need to package something like an internal company application.
  - **Configuration management tools like SCCM and Puppet even integrate with Chocolatey.**


🔗 **Repository and packaging options**  
  – Uses the **public Chocolatey repository** to install many packages.  
  – You can **add missing software** to the public repo or create a **private repository** for internal/company applications.  
  – Supports packaging anything that can be installed on Windows as a Chocolatey package.

🏢 **Enterprise integration and automation**  
  – Integrates with configuration-management/deployment tools such as **SCCM** and **Puppet**.  
  – Facilitates automated, centralized software deployment and updates across multiple machines.

🛠️ **Ways to use Chocolatey**  
  – Install and run the **Chocolatey CLI** directly from **PowerShell**.  
  – Or use PowerShell’s **PackageManagement** (OneGet) feature and set the package **source** to the Chocolatey repository to use standard PackageManagement cmdlets.

🔍 **Finding packages (example)**  
  – Use **Find-Package** with dependency inclusion, e.g.: **Find-Package -Name sysinternals -IncludeDependencies**  
  – The **-IncludeDependencies** flag locates the package and its required dependencies.

⬇️ **Installing a package (example)**  
  – Use **Install-Package -Name sysinternals** to install the package and its dependencies via PackageManagement or Chocolatey-backed provider.  
  – The command may prompt for confirmation depending on your environment; it installs both the package and referenced dependencies.
