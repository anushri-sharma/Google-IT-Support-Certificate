🔥 **Use PuTTY's SCP client (pscp.exe) to copy files between Windows and Linux**  
  - **pscp.exe** is included in the **PuTTY** package and implements the **SCP** protocol, functioning similarly to the Linux scp command.  
  - Example usage pattern: **pscp.exe C:\path\file.txt user@linuxhost:/remote/path** (copies a local file to a remote Linux workstation).  
  - Transfers with **pscp** are straightforward for single-machine copies but can be time-consuming when pushing files to many machines.

💡 **Windows built-in Shared Folders: purpose and workflow**  
  - **Shared Folders** let you expose a folder to specified users or groups on the network so they can access files placed there.  
  - GUI steps: right-click the folder → **Share with** → **Specific people** → add users or groups (or **Everyone**).  
  - Adding **Everyone** is convenient but **reduces security** because it grants access to all network users.

🍳 **Accessing a shared folder from another Windows computer**  
  - Map it: open **This PC** → **Computer** tab → **Map network drive** to assign a drive letter to the shared folder.  
  - Finally, on another computer, you can visit it directly from the run box by typing in and enter **\ComputerName\ShareName(folder name that you mapped it to.)** to open the share immediately.

🧰 **Command-line sharing with net share**  
  - The **net share** command can create shares and set permissions from an elevated (administrator) PowerShell or CMD prompt.  
  - Example to share a folder named **ShareMe** with full access to everyone (correct syntax): **net share ShareMe=C:\ShareMe /GRANT:Everyone,FULL**  
  - Run **net share** with no arguments to **list currently shared folders** on the computer.

🔐 **Permissions and administrative requirements**  
  - Creating shares with **net share** or setting network permissions typically requires **administrator privileges**.  
  - Granting **Everyone** full access is insecure; prefer adding specific users or groups and apply least-privilege permissions.

🔄 **When to use each method (practical implication)**  
  - Use **pscp/ SCP** for ad-hoc, single-machine transfers, or copying directly to Unix/Linux targets.  
  - Use **Windows Shared Folders** (GUI or **net share**) for repeated access by multiple Windows clients, or when centralizing files for a group.
