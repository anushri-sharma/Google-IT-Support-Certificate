## 🔥 PuTTY (Windows SSH client and installer)
  
  - **PuTTY** is a free, open-source client for remote connections supporting **SSH** and other protocols.
  
  - Download options: full installer (**MSI**) or individual executables (e.g., **putty.exe**) depending on whether you want the whole package or just a specific tool.
  
  - The GUI shows **Host Name**, **Port**, and **Connection Type**; defaults are **Port 22** and **SSH**.
  
  - To start an SSH session from the GUI: enter the **host name or IP address** and click **Open**.

## 💻 PuTTY command-line usage and plink
  
  - You can run PuTTY from PowerShell/CMD: use **putty -ssh user@host -P port** (example pattern).
  
  - **plink** (PuTTY link) is included with PuTTY and provides command-line SSH connections: use **plink user@host -P port**.

## 🔗 SSH use case and implications
  
  - SSH is useful for connecting from **Windows to remote Linux** systems to get a shell session.
  
  - Default **SSH port is 22**; specify different ports when the server uses a nonstandard port.

  
## 🖥️ Remote Desktop Protocol (RDP) overview
  
  - **RDP** provides a graphical interface to remote Windows machines when **incoming RDP is enabled** on the remote host.
  
  - Cross-platform clients exist (examples: **RealVNC** for Linux, **Microsoft Remote Desktop** for macOS).

  
## ⚙️ Enabling Remote Desktop on Windows
  
  - Steps to enable: open **Start**, right-click **This PC** → **Properties** → **Remote settings** → choose an option under **Remote Desktop** to allow connections.
  
  - The remote machine must also have the connecting user included in the **allowed users** list.

## 🪟 Using the Windows RDP client (mstsc.exe)
  
  - The client program is **mstsc.exe** (Microsoft Terminal Services Client).
  
  - Launch methods: type **mstsc** in the Run box, search **Remote Desktop Connection** in Start, or run from the command line.
  
  - Command-line options exist for additional control; example: **mstsc /admin** to connect with **administrative credentials**.

  
## 🔒 Security and administrative considerations
  
  - Allowing remote connections exposes attack surface; only permit **trusted users** and configure access control.
  
  - In organizational environments these settings are typically managed by the **system administrator**.
