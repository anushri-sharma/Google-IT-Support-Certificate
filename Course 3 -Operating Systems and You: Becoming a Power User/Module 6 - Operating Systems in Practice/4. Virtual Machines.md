
 
 # [Virtual Machines](https://www.youtube.com/watch?v=JTO3drQGZjc&t=10s)


 🔥 **Definition: virtual instance = a single virtual machine**
  
  – **Virtual instance** refers to one VM running on host hardware via virtualization software.

💾 **Tool used: VirtualBox (open-source)**
  
  – **VirtualBox** is the virtualization software used; a download link is provided in the lesson reading.  
  – Host machine in the demo: **Windows**. Guest OS in the demo: **Ubuntu** (image pre-downloaded).

🛠️ **Create a new VM (step-by-step)**
  
  – Click **New** to create a VM.  
  – Give the VM a **name** and set **type/version** of OS (defaults were accepted).  
  – Allocate **RAM** (demo used **1 GB**).  
  – Allocate **hard-drive** space (demo used **10 GB**).  
  – Click **Create** to finish VM creation.

▶️ **Start the VM and install the OS**
  
  – Click **Start** to boot the VM.  
  – Prompt appears to **select installation media** (ISO) — analogous to booting from a USB image.  
  – Select the pre-downloaded **Ubuntu ISO** and proceed with the OS installation.

⚙️ **Modify VM hardware after creation**
  
  – Right-click the VM and choose **Settings** to change allocated resources.  
  – Example: increasing RAM is done via **Settings** (no physical hardware change needed).  
  – Other hardware and VM settings are available in **Settings** (specific settings not detailed).

🗑️ **Remove a VM**
  
  – Right-click the VM and select **Remove**.  
  – Removal options: **remove only from list** or **delete all files** (including the VM install).  
  – Deleting a VM removes only virtual files — no physical hardware disposal required.

💡 **Key implications and differences vs physical machines**
  
  – Resource changes (e.g., RAM) are configurable in software without buying or installing physical components.  
  – VM disposal is simplified to deleting files rather than recycling hardware.

⚠️ **Caveats and omitted details**
  
  – The lesson did not cover all VirtualBox menu items or the specifics of advanced VM settings.  
  – The demo used default values; real workloads may require larger RAM/disk allocations.
