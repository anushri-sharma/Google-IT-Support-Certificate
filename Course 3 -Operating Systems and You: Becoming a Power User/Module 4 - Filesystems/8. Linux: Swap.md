
# [Linux: Swap](https://www.youtube.com/watch?v=v6iAUZY1RtI&t=9s)


## The dedicated area of the hard drive used for virtual memory is known as swap space.

let's mix swap space.

We can create swap space by using the new disk partitioning tools

In our case, since we just have a USB drive which doesn't need swap, we're
just going to partition the rest of it as swap to show you how this works.

In practice, you would create swap partitions for your main storage devices like hard drives and SSDs.

Okay, let's mix swap space.

First go back into the **parted tool and select /dev/sdb where USB** is.

<img width="1273" height="436" alt="image" src="https://github.com/user-attachments/assets/0ed3ca13-7a54-4b1b-a82b-b390e3397109" />

We're going to partition it again, this time to make a swap partition and then we'll format the linux-swap file system on it.

<img width="1273" height="606" alt="image" src="https://github.com/user-attachments/assets/af4724dd-d4c9-4d62-a20f-2f34cffe1b43" />

So make part primary linux-swap 5Gib 100%.

You'll notice that the endpoint of the drive says 100%.

Which indicates that we should use the rest of the free space on our drive.

We're not done yet.

Swap isn't actually a file system.

So this command won't be enough.

If you think about it, it makes a lot of sense, since pages go into swap and not file data.

Anyways to complete this process, we need to specify that we want to make it swap space with the **make swap command**.


<img width="1272" height="285" alt="image" src="https://github.com/user-attachments/assets/6cb9f937-0db6-4783-812a-dd0d7c3af19f" />


Let's **quit out** of parted and run this command on our new swap partition.

So **sudo make swap dev** and our **new swap partition is on dev/sdb2.**


<img width="1279" height="161" alt="image" src="https://github.com/user-attachments/assets/0e8308f9-fcca-46e9-b0a9-b60adbe50df0" />

Finally, there's **one more commander run to enable swap on the device swapon.**


<img width="1270" height="131" alt="image" src="https://github.com/user-attachments/assets/3aba53d8-1cc9-450f-ac52-16d9678a11f1" />

So **sudo swapon dev/sdb2.**

<img width="1264" height="116" alt="image" src="https://github.com/user-attachments/assets/79e4101f-d01a-4a93-844d-1b6299924778" />

If we want to automatically mount swap space every time the computer boots up, just add a swap entry to the / etc/fstab file
