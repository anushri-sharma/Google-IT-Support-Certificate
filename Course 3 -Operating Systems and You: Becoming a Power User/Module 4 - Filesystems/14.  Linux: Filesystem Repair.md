
# [Linux: Filesystem Repair](https://www.youtube.com/watch?v=RrsqpAWc4gY&t=10s)

To try and repair a file system manually in Linux, you can also use the fsck or file system check command.

Just make sure the file system isn't mounted.

<img width="1274" height="169" alt="image" src="https://github.com/user-attachments/assets/de93d652-dc43-4136-9368-2ba3323cb0e1" />

**fsck on a mounted partition, there's a high chance that it will damage the file system.**

File system repair isn't always a guaranteed fixed, but it can help in most cases.

Just be nice to your hardware and it'll be nice to you, in most cases.

Another thing to call out is that on some versions of Linux, fsck actually runs on your

computer when you boot it to check for any issues and attempt to auto repair the file system.
