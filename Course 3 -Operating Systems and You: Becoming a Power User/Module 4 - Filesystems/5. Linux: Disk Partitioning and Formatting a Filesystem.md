## 🔍 List connected disks with parted  
  - Run **sudo parted -l** (note the space) to list disks and current partitioning.  
  - Output fields include **partition table**, **number**, **start**, **end**, **size**, **filesystem**, **name**, **flags**.  
  - Partition numbers map to device nodes (**/dev/sda1**, **/dev/sda2**, …).  
  - Example from demo: **/dev/sda = 128GB**, **/dev/sdb ≈ 8GB**; **/dev/sdb** initially had no partitions.

## ⚠️ Confirm the correct disk before modifying  
  - Always double-check the target device (e.g., **/dev/sdb**) using **sudo parted -l** or other tools; modifying the wrong disk can cause data loss.  
  - Parted is powerful; mistakes can produce significant damage.

## 🧭 Set a partition table label (create GPT or MBR)  
  - Inside parted, use **mklabel gpt** to create a **GPT** partition table.  
  - If you see **"unrecognized disk label"**, create a label before partitioning.

## ➗ Create partitions with mkpart  
  - Syntax: **mkpart [partition-type] [filesystem-type] [start] [end]**.  
  - For **GPT**, the **partition-type** is effectively non-essential; using **primary** is standard.  
  - Demo example: **mkpart primary ext4 1MiB 5GiB** → creates a partition from **1 MiB** to **5 GiB** (~**5 GiB**).  
  - Note: the speaker said they would make two partitions but the demo created one partition and left the rest unpartitioned for later use.

## 📏 Use binary units (MiB/GiB) for precise sizing  
  - Prefer **MiB/KiB/GiB** (mebibyte/kibibyte/gibibyte) when partitioning: **1 KiB = 1024 bytes** (vs **1 kB = 1000 bytes**).  
  - Using binary units avoids OS measurement confusion and potential wasted space.

## 🧹 Format the partition (create a filesystem)  
  - After partitioning, run **sudo mkfs -t ext4 /dev/sdb1** to format the new partition with **ext4**.  
  - Ensure you specify the correct partition device (e.g., **/dev/sdb1**, not the whole disk).

## 📂 Mount the filesystem to use it  
  - A formatted filesystem must be **mounted** to a directory to read/write files.  
  - Typical next steps (not shown in demo): create a mount point and mount, e.g., **sudo mkdir -p /mnt/usb** then **sudo mount /dev/sdb1 /mnt/usb**.  
  - To make mounts persistent, add an entry to **/etc/fstab** (exercise caution and verify UUIDs).

## ⚒️ Additional notes and partition metadata explained  
  - **Start/End** fields show partition offsets on disk; **size** shows partition length.  
  - **Filesystem** column shows detected filesystem (if any); empty if unformatted.  
  - **Flags** column lists attributes like boot or esp if set.  
  - Parted handles both **MBR** and **GPT** partitioning schemes.


There's **one last step** to get a usable disk in Linux.

We have to **mount the file system** to a directory so that we can access it from the shell.
