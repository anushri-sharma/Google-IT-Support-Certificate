🔥 **Virtual memory**: a system that maps **virtual addresses to physical RAM**, letting each program access memory without tracking where other programs store data.  
  - This mapping simplifies program design and isolation because programs do not need to know physical memory layout.

💾 **Pages and paging**: memory is managed in blocks called **pages**; when a page is unused it can be **evicted** (copied) to disk to free RAM.  
  - Evicted pages are stored on disk so the system can present more usable memory than physically installed.

⚡ **Performance trade-off**: **RAM is much faster than disk**, so accessing an evicted page causes the OS to read it from disk and restore it to memory, which is slower.  
  - The OS favors keeping the most frequently accessed pages in RAM and moves infrequently used pages to disk.

🧠 **Windows implementation**: Windows uses a **memory manager** to map virtual to physical memory and to perform paging.  
  - Pages saved to disk are placed in a hidden file on the root of a volume named **pagefile.sys** (transcript mispronounced this as "page file.sis").  
  - The memory manager copies pages to the page file and reads them back when needed.

🛠️ **Controlling Windows paging files**: Windows auto-creates page files but allows manual changes via System Properties.  
  - Path: Open Control Panel → System and Security → System → Advanced system settings → Advanced tab → Performance Settings → Advanced tab → **Virtual memory** → **Change**.  
  - From there you can **override defaults**, set explicit paging file sizes, and **add paging files on other drives**.

📏 **Sizing guidance**: Windows generally manages page file size well automatically.  
  - Example guideline: on **64-bit Windows 7**, the **minimum paging file size** is recommended to be **1× the amount of RAM**.  
  - Unless you have a specific reason, it is usually best to **let Windows automatically manage** the paging file.

⚠️ **Implications and practical notes**: using a page file enables systems to run workloads that exceed physical RAM but incurs slower disk access when paging occurs.  
  - You can change size, number, and location of page files, but changes affect performance and available virtual memory; adjust only for known needs.
