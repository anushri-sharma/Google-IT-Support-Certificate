
# [Hosts Files](https://www.youtube.com/watch?v=QHtO9uWS0d4&t=2s)

🔥 **What a hosts file is**  
  - A **flat text file** that maps a network **IP address** to a **hostname** on a per-line basis (IP followed by the name).  
  - Example line format: **1.2.3.4 webserver** — on that computer you can use "webserver" instead of "1.2.3.4".  
  - The hosts file is read by the operating system’s networking stack, so an entry applies system-wide (all applications).

🍳 **How entries affect usage**  
  - Entries translate names for any network operation on that machine (e.g., typing a hostname in a web browser or running ping yields the specified IP).  
  - Hosts lookup is performed before a DNS query on most major operating systems, so a hosts entry can **override DNS** for that device.

💡 **Loopback addresses and typical defaults**  
  - A **loopback address** sends traffic to the same machine; it does not leave the node and bypasses network infrastructure.  
  - Common, almost-universal hosts entries: **127.0.0.1 localhost** (IPv4) and **::1 localhost** (IPv6).  
  - These entries are why loopback functionality is available by default on modern devices.

🛠️ **Why hosts files still exist and practical uses**  
  - Present on **all modern OSes**, including phones and tablets, despite DNS ubiquity.  
  - Useful for local testing, forcing a single computer to resolve a domain to a specific IP, and certain legacy software that requires explicit hosts entries.  
  - Common troubleshooting tool in IT support to isolate name-resolution or routing issues.

⚠️ **Security risks and misuse**  
  - Hosts files are a frequent target for malware and viruses that modify entries to **redirect or block** user traffic (e.g., hijacking domains to malicious IPs).  
  - Because hosts entries override DNS locally, malicious changes can be stealthy and effective at redirecting specific machines.

🔁 **Trade-offs and operational considerations**  
  - Hosts files are effective for single-machine overrides but are **not scalable** for network-wide name management compared to DNS.  
  - They remain valuable for targeted debugging, development, or temporary overrides, but require careful management and security controls to avoid misuse.
