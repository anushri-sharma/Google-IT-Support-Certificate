
 # [What is The Cloud?](https://www.youtube.com/watch?v=Qi_Z4paS5UA&t=10s)

 🔷 **Cloud = a computing model, not a single technology**  
  - Definition: **cloud computing** = provisioning **shared computing resources** so many users get what they need when they need it.  
  - Character: conceptual and intentionally broad; the term is used for many related services and architectures.  
  - Implication: businesses can offer and consume services from each other using remotely pooled resources.

🔧 **Core technology: hardware virtualization**  
  - **Hardware virtualization** abstracts a physical machine (host) from a logical machine (guest).  
  - A **hypervisor** is software that runs and manages virtual machines, presenting each guest with a virtual platform indistinguishable from real hardware.  
  - Result: one physical host can run many independent **virtual machines (VMs)**, each with its own OS and runtime.

🧩 **How the cloud builds on virtualization**  
  - Large clusters of interconnected hosts can collectively host many VMs and share resources across them.  
  - Shared pooling enables better utilization of CPU, RAM, storage and network than isolated physical servers.

🧾 **Practical example showing resource efficiency**  
  - Requirements: an email server (**8 GB RAM, Windows**), a name server (minimal compute, **Linux**), a financial database (quiet most of month but needs **32 GB RAM, special Linux**, plus a backup server).  
  - Physical-model result: you'd buy four physical machines totaling **80 GB RAM**, while **only ~40 GB** might be used at peak—significant wasted capacity and cost.  
  - Cloud-model result: place each service in VMs on a pooled cluster, allocate RAM on demand, and pay a hosting provider for VM instances—lower cost and less idle hardware.

⚙️ **Operational and service benefits of cloud providers**  
  - Providers offer ready-made services (example: **backup**, **load balancing**) so customers need not build those systems themselves.  
  - Hardware failures can be masked: providers can move VMs to other hosts without customer-visible interruption.  
  - Provisioning speed: new servers/services can be deployed in moments via web interfaces rather than waiting for physical hardware provisioning.  
  - Implication: faster time-to-service, built-in resilience, and offloaded operational tasks.

🏢 **Cloud deployment models and their use cases**  
  - **Public cloud**: large clusters run by a third-party provider and shared among many customers.  
  - **Private cloud**: same virtualization and pooling concepts but dedicated to a single organization, typically hosted on-premises.  
  - **Hybrid cloud**: not a separate technology—combines private and public clouds (e.g., keep sensitive workloads on a private cloud and less sensitive ones on the public cloud).

🔎 **Nuances, edge cases, and operational considerations**  
  - Virtualization permits different operating systems to run on the same physical host, resolving many OS-compatibility constraints from the physical-only model.  
  - Bursty workloads (e.g., end-of-month billing) can be met by temporarily allocating more resources without buying permanent hardware.  
  - Trade-offs include dependence on provider capabilities and the rationale for private/cloud splits driven by security, compliance or performance requirements.
