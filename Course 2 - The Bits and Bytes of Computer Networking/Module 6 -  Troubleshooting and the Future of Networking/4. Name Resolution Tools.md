
# [Name Resolution Tools](https://www.youtube.com/watch?v=dpCkDvCiI1k&t=5s)

**Name resolution is an important part of how the Internet works.**

Most of the time, your operating system handles all lookups for you.

🔥 Name resolution role and when to run queries  
  - **Name resolution** is fundamental to Internet operation; **operating systems** usually perform lookups automatically.  
  - As an **IT support specialist**, running lookups manually reveals how queries are resolved and helps troubleshoot issues.  
  - There are **many command-line tools** for DNS troubleshooting; this summary focuses on **nslookup**.

🍳 nslookup availability  
  - **nslookup** is available on **Linux, Mac, and Windows**, making it a cross-platform basic DNS tool.

💡 Basic nslookup usage and output  
  - Run **nslookup ** to query a name; output shows **which server** handled the request and the **resolution result**.  
  - Example: **nslookup twitter.com** returns the host’s **A record** (IPv4 address).

🛠 Interactive mode for repeated queries and configuration  
  - Start interactive mode by running **nslookup** with **no hostname**; an angle-bracket prompt appears for multiple queries.  
  - Interactive mode supports making many requests in sequence without re-invoking the tool.  
  - Use **server ** to direct all subsequent queries to a **specific DNS server** instead of the default—useful for testing particular resolvers or authoritative servers.

💡 Requesting different DNS record types  
  - Use **set type=** followed by a resource record type to request non-default records; default is **A**.  
  - Common examples: **AAAA** (IPv6), **MX** (mail exchange), **TXT** (text/SPF records).  
  - This lets you verify records relevant to IPv6, email routing, domain verification, etc.

🔎 Debug mode and data exposure risks  
  - **set debug** makes nslookup display the **full response packets**, including intermediary requests and detailed contents.  
  - Debug output can be extensive and shows metadata such as **TTL** (time-to-live) for cached responses and the **serial number of the zone file** the query was answered from.  
  - Implication: debug reveals helpful diagnostic detail but can expose sensitive operational info; use with caution.
