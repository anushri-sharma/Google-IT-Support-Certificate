
# [IPv6 Addressing and Subnetting](https://www.youtube.com/watch?v=xzn6AxDzkeI&t=6s)

🔥 **IPv4 exhaustion and why IPv6 exists**  
  - **IPv4 uses 32 bits**, yielding **2^32 = 4,294,967,296 (~4.29 billion)** addresses, which proved insufficient as the Internet grew.  
  - **IPv6 was created to solve address exhaustion** by expanding address space to **128 bits**.

🍳 **IPv5 trivia**  
  - **IPv5 was an experimental, connection-oriented protocol** that never saw wide adoption; its name was not reused for the successor protocol.

💡 **IPv6 address scale (128 bits)**  
  - **IPv6 = 128 bits → 2^128 addresses**, roughly **3.4 × 10^38** (a ~39-digit number).  
  - That space is astronomically large (often compared to counts like atoms on Earth) and practical for addressing devices for the foreseeable future.

🔢 **How IPv6 addresses are written and shortened**  
  - **Canonical form**: eight groups of 16 bits, each group shown as **4 hexadecimal digits** (e.g., 2001:0db8:0000:0000:0000:0000:0000:0001).  
  - **Shortening rules**:  
    -  Remove **leading zeros** in any 16-bit group.  
    -  Replace **one** run of consecutive all-zero groups with **::** (only once per address to avoid ambiguity).  
  - **Example extremes**:  
    - IPv6 loopback is **::1** (equivalently 0:0:0:0:0:0:0:1).  
    - IPv4 loopback is the **127.0.0.0/8** block (common single address **127.0.0.1**).


**The IPv6 address space has several other reserved address ranges, besides just the one reserved for documentation purposes or the loopback address.**

**For example, any address that begins with FF00:: is used for multicast**, which is a way of addressing groups of hosts all at once.

It's also good to know that addresses beginning with FE80:: are used for **Link-local unicast.**

**Link-local unicast addresses allow for local network segment communications and are configured based upon a host's MAC address.**

<img width="1280" height="296" alt="image" src="https://github.com/user-attachments/assets/1fa4e4e6-49e5-4d48-a50e-f961d9541838" />

The **link-local address** are used by an IPv6 host to receive their network configuration, which is a lot like how DHCP works.

The **host's MAC address** is **run through an algorithm** to turn it from a 48-bit number into a unique 64-bit number.

**It's then inserted into the addresses host ID.**

The IPv6 address space is so huge, there was never any need to think about splitting it up into address classes like we used to do with IPv4.

📚 **Reserved and special-purpose prefixes**  
  - **Documentation**: **2001:0db8::/32** is reserved for documentation and examples; that block contains **2^(128-32) = 2^96 ≈ 7.9 × 10^28** addresses (vastly larger than the entire IPv4 space).  
  - **Multicast**: addresses beginning with **FF00::** (RFC defines **FF00::/8**) are used for **multicast** (addressing groups of hosts).  
  - **Link-local**: addresses beginning with **FE80::** (allocated under **FE80::/10**) are **link-local unicast** for communication on a local network segment and for initial configuration tasks.



🔧 **Link-local addressing and autoconfiguration**  
  - **Link-local addresses** are typically derived from a host's MAC address using the **EUI-64** method: a 48-bit MAC is transformed into a 64-bit interface identifier and inserted into the host ID portion of the IPv6 address.  
  - Link-local addresses are used by hosts for **neighbor discovery and address configuration** (stateless autoconfiguration mechanisms analogous in purpose to DHCP but using NDP/SLAAC).

📐 **IPv6 structure, no address classes, and subnetting**  
  - IPv6 eliminated IPv4-style address classes; addresses have a straightforward split: **first 64 bits = network ID**, **last 64 bits = host ID** in the common default model.  
  - That yields **2^64 ≈ 18,446,744,073,709,551,616 (~1.84 × 10^19)** possible hosts per network (an extremely large host space).  
  - **Subnetting** in IPv6 uses **CIDR notation** (e.g., /64) to define how many bits of the address are the network prefix vs. host identifier; network engineers still subnet for administrative and organizational needs.

From the very beginning, an **IPv6 address had** a very simple line between **network ID and host ID.**

The **first 64 bits of any IPv6 address is the network ID**, and the **second 64 bits of any IPv6 address is the host ID.**

This means that any given IPv6 network has space for over **nine quintillion hosts.**

Still, sometimes network engineers might want to split up their network for administrative purposes.

IPv6 subnetting uses the same cider notation that you're already familiar with.

This is used to define a subnet mask against the network ID portion of an IPv6 address.
