
**Computer networking** can be an incredibly complicated business.

There are so many **layers, protocols**, and **devices at play**, and sometimes this means that things just don't work properly.

Many of the **protocols and devices** , have **built in functionalities** to **help protect against** some of these **issues.**

**These functionalities are known as error detection and error recovery.**

For example, **cyclical redundancy checks** are used by **multiple layers** to make sure that the correct data was received by the receiving end.

<img width="1269" height="404" alt="image" src="https://github.com/user-attachments/assets/ef6d1e41-b044-461b-8e43-cbb7c47cdf39" />

If a **CRC value doesn't match the data payload, the data is discarded.**

At that point, the **transport layer** will decide if the data needs to be resent, but even with all
of these safeguards in place, errors still **pop up, misconfigurations occur, hardware breaks down, and system incompatibilities come to light.**

In this module,  learn about the **most common techniques and tools** you use troubleshooting network issues.

By the end of this module, will be able to detect and fix a lot of the common
network connectivity problems by using tools available on the three most common operating systems, Microsoft Windows, MacOS, and Linux.
