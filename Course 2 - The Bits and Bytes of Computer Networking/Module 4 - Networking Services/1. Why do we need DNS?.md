
# [Why do we need DNS?](https://www.youtube.com/watch?v=EkbDGXPj8h4&t=6s)

🔥 Computers use numeric addresses, not words  
  – **Computers communicate in binary (ones and zeros)** and, at the network layer, use numeric IP addresses composed of **four octets**.  
  – Memorizing raw IP numbers for every site is impractical for humans.

💡 DNS translates human-friendly names into IP addresses  
  – **Domain Name System (DNS)** is a global, highly distributed service that resolves strings of letters (domain names) into IP addresses.  
  – Example: **www.weather.com** can resolve to an IP such as **184.29.131.121** so users type words instead of numbers.

🍳 Domain names are abstractions that can map to changing IPs  
  – A **domain name** is the label DNS resolves; the underlying **IP can change** for many reasons.  
  – Implication: users keep the same URL even when servers move or IPs update.

🔁 DNS enables seamless administrative changes  
  – If a site moves data centers or changes hosting, the organization updates DNS mappings so end users do **not** need to change behavior.  
  – This separates human-facing names from operational infrastructure details.

🌍 DNS supports geographic routing and performance optimization  
  – Because DNS is global, organizations can resolve the same domain name to **different IPs based on the user’s region**.  
  – Example implication: a visitor in New York can be directed to a New York–nearby server while a visitor in New Delhi is directed to a local server, reducing latency.

⚡ Latency and proximity matter for user experience  
  – Although signals travel at near the **speed of light**, longer routing paths increase delay; serving users from nearby servers is typically faster.  
  – DNS-based regional resolution helps provide a better, faster experience for geographically distributed users.

🛠️ DNS is essential knowledge for IT support and troubleshooting  
  – DNS serves many practical purposes in networking and is a key technology to understand when diagnosing connectivity or routing issues.  
  – Knowing DNS behavior (name-to-IP mapping, region-based resolution, and that IPs can change) is necessary for effective troubleshooting.
