
# [Proxy Services](https://www.youtube.com/watch?v=DaMH3nZZbo4&t=4s)

💡 **Proxy service**: a server that acts on behalf of a client to access another server, sitting between client and server to provide benefits like **anonymity**, **security**, **content filtering**, and **performance**.  
  – Proxies are an **abstraction**, not a specific implementation; they exist at almost every layer of networking.  
  – Common infrastructure (for example, **gateway routers**) meets the definition of a proxy even if not labeled that way.

🕸️ **Web proxy (common usage)**: a proxy specialized for HTTP/HTTPS traffic that intermediates all web requests for an organization or client.  
  – Historically used for **caching**: the proxy fetched webpages from the Internet, stored them, and returned cached copies to subsequent requesters to improve performance.  
  – Caching is less useful today because connections are generally faster and the web is much more **dynamic/personalized** (e.g., sites like Twitter display different content per user).  
  – Contemporary use is often **content filtering**: proxies inspect requested URLs and **allow or deny** access (for example, blocking social media during work hours).

🔁 **Reverse proxy**: a front-end server that appears as a single server to external clients but forwards requests to many backend servers.  
  – Enables **load balancing** across multiple servers (conceptually similar to DNS round-robin) so high-traffic sites can scale beyond a single web server.  
  – Commonly used for **TLS/HTTPS decryption offload**: because **more than half** of web traffic is encrypted, reverse proxies can use specialized cryptographic hardware to handle encryption/decryption so backend servers focus on serving content.

📚 **Variety and operational implications**: there are dozens of proxy types and deployment patterns; the term covers many practical roles across systems.  
  – Key operational trade-offs include where to place caching or filtering, how to distribute load, and whether to offload CPU-intensive tasks like cryptography to proxy hardware.  
  – Core definition to apply when evaluating solutions: **any intermediary server that acts between a client and another server**.
