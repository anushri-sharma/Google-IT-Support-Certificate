```
A reverse proxy sits between clients and servers and helps with:

- IP address translation → Hides backend server IPs
- Encryption/Decryption (SSL/TLS termination) → Handles HTTPS traffic
- Load balancing → Distributes requests across multiple servers

```

```
A proxy acts as an intermediary:

- It sends requests on behalf of a client to a server
- It can also return responses back to the client

❌ Why others are incorrect:

- encrypts traffic sent across the Internet ✖️ → That’s encryption (not the core definition of a proxy)
- allows for many devices to speak to one other device ✖️ → That’s more like NAT or load balancing
- sends data across a single network segment ✖️ → Not related to proxy functionality

```
