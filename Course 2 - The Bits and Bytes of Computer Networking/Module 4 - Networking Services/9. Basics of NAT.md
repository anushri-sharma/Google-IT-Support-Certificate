

# [Basics of NAT](https://www.youtube.com/watch?v=XfnCVPFzYy8&t=3s)

🔁 Network Address Translation (NAT) is a **technique**, not a formal protocol standard.  
  – Different operating systems and network vendors implement NAT details differently, so explanations are often high-level.  
  – Primary motivations include **security safeguards** and **preserving limited IPv4 address space**.

🛠️ Core NAT behavior: a gateway rewrites IP addresses on passing datagrams while preserving mapping for replies.  
  – For outgoing traffic the gateway (router/firewall) **rewrites the source IP** to another address.  
  – The gateway retains the original source in a mapping so it can **rewrite the destination** back on the return packet.  
  – Normal router steps (decrement **TTL**, recalculate **checksum**) still occur; NAT adds the address translation step.

🧭 Concrete example (corrected interface addresses for clarity):  
  – Network A: **10.1.1.0/24**; Network B: **192.168.1.0/24**.  
  – Router interfaces: **10.1.1.1** (Network A) and **192.168.1.1** (Network B).  
  – Computer 1: **10.1.1.100** (on Network A) wants to reach Computer 2: **192.168.1.100** (on Network B).  
  – Computer 1 sends a packet to its gateway; the router rewrites the packet’s **source IP → 192.168.1.1** and forwards it.  
  – Computer 2 receives the packet appearing to come from **192.168.1.1**, replies to the router, and the router uses its mapping to **rewrite the destination back to 10.1.1.100** before forwarding.

🔐 Security and visibility effects (IP masquerading):  
  – NAT can **hide internal IP addresses** from external hosts; internal hosts appear to external networks as the gateway’s address.  
  – This “IP masquerading” means external parties cannot initiate connections to an internal host if they do not know its address as presented externally.  
  – A single external IP can represent **hundreds of internal hosts**, making the internal address space **protected and invisible** to the outside.

🔁 Common deployment pattern and terminology:

  – The described behavior is known as **one-to-many NAT** (many internal addresses mapped to one external address).  
  – **One-to-many NAT/IP masquerading** is widely used in LANs today to provide address conservation and basic network boundary protection.

- **IP masquerading** is an important security concept.

- The most basic concept at play here is that no one can establish a connection to your computer if they don't know what IP address it has.

- By using NAT in the way we've just described, we could actually have hundreds of computers on Network A, all of their IPs being translated by the router to its own.

- To the outside world, the entire address space of Network A is protected and invisible.
