🔥 NAT behavior at the transport layer vs. network layer  
  – **Network-layer NAT**: translates one IP address to another (simple address replacement).  
  – **Transport-layer complications**: multiple hosts share one external IP, so routers must track per-connection transport identifiers (ports) to route return traffic correctly.

💡 Port preservation (outbound NAT)  
  – **Definition**: the router keeps the **source port chosen by the client** when rewriting the source IP to the router’s external IP.  
  – **How it works**: the router records a mapping (external IP + source port → internal IP + source port) in a table and uses that to forward returning packets.  
  – **Example**: host **10.1.1.100** picks source port **51,300**; router replaces the source IP with its external IP and stores a mapping so replies to port **51,300** get forwarded to **10.1.1.100**.

📌 Ephemeral port range and implications  
  – **Ephemeral ports** are in the range **49,152–65,535** (used for outbound connections).  
  – **Implication**: large but finite pool; collisions can occur when multiple internal hosts choose the same source port.

⚠️ Port collisions and port translation  
  – **Collision handling**: if two internal hosts use the same source port simultaneously, the router **chooses an unused external port** and rewrites the source port for one of the flows (updates the NAT mapping accordingly).  
  – **Result**: external observers see different source-port values even if internal hosts used the same port.

🔀 Port forwarding (destination NAT)  
  – **Definition**: configure the router so **specific external destination ports** are always forwarded to predetermined internal hosts and ports.  
  – **Use case**: allows **complete IP masquerading** while still exposing internal services to the public Internet.  
  – **Example**: external IP **(router)** e.g., **192.168.11.1**, receives traffic on **TCP port 80** → router forwards that traffic to **10.1.1.5** (internal web server).  
  – **Response handling**: the router rewrites the source IP of reply packets to its external IP so external clients see responses coming from the external address.

🧩 Multiple services on one external IP  
  – **Single external IP, multiple services**: different destination ports map to different internal servers (same DNS/name for external clients).  
  – **Example**: web server at **10.1.1.5** (port 80) and mail server at **10.1.1.6** (mail ports) both reachable via the same external IP; router forwards based on destination port.

🔎 Operational implications and edge cases  
  – **Unsolicited inbound traffic**: without port forwarding or specific mappings, unsolicited incoming connections will not reach internal hosts.  
  – **NAT table state**: routers must maintain per-connection mapping tables (address+port tuples) to support return traffic and forwarding decisions.  
  – **Port scarcity risk**: extremely large numbers of concurrent sessions can exhaust available external port mappings, requiring additional NAT strategies or multiple external IPs.
