
# [Interior Gateway Protocols](https://www.youtube.com/watch?v=0BU-UA_g6vc&t=6s)

🔁 **Routing protocols** let routers exchange information so routing tables are continuously updated with the quickest paths to destination networks.  
  - Routers "speak" to share what they know so a router on one side of the planet can learn the best path to a remote network.  
  - Purpose: discover reachable networks and update routing tables with path metrics.

🧭 **Protocol categories: Interior vs Exterior**  
  - **Interior Gateway Protocols (IGPs)** operate within a single **autonomous system (AS)** — a set of networks under one operator (examples: a large corporation's internal networks, an ISP's national network).

🔗 **Two main IGP types: link-state and distance-vector**  
  - Both aim to find best paths but differ in what data routers share and how they compute routes.

📏 **Distance-vector protocols — mechanics and limits**  

<img width="1277" height="213" alt="image" src="https://github.com/user-attachments/assets/545b0a10-ae6e-4909-88f4-c7fe272954bd" />

  - Each router sends its entire routing table (a list, i.e., a "vector") of known networks and **distances in hops** to every directly connected neighbor.  
  - Routers only have detailed knowledge of their immediate neighbors, so global state is limited.  
  - Convergence to topology changes can be slow for distant events because updates propagate hop-by-hop.  
  - Example: Router A initially routes to **10.1.1.0/24** (network X) via Interface 2 through Router C at **4 hops**; Router B reports network X is **2 hops** from B; A learns that forwarding to B via its Interface 1 yields **3 hops**, so A updates its table to use Router B.  
  - Simplicity is an advantage; limited visibility and slower reaction are drawbacks.

🧩 **Link-state protocols — mechanics and requirements**  
  - Each router advertises the **state of each interface** (links to routers or networks); these link-state advertisements are flooded so every router in the AS receives them.  
  - Every router builds a complete topology map of the AS (full knowledge of routers and links).  
  - Each router runs a shortest-path algorithm (e.g., **Dijkstra**) against the global topology to compute optimal paths and update its routing table.  
  - Requires more **memory** to store topology data and more **CPU** to run path computation.  
  - Provides a faster, more accurate reaction to network changes due to global visibility.

⚖️ **Trade-offs and historical trend**  
  - **Distance-vector**: older, simpler, lower resource needs, less timely convergence for distant changes.  
  - **Link-state**: more resource-intensive but offers complete topology knowledge and faster convergence.  
  - With cheaper, faster hardware over time, **link-state protocols have largely supplanted distance-vector protocols** in many environments.
