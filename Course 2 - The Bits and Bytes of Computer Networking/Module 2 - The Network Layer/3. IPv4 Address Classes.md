
# [IPv4 Address Classes](https://www.youtube.com/watch?v=vbW1CfKKcPs&t=5s)

- IP addresses can be split into two sections; the **network ID and the host ID**.
- **IBM** owns all **IP addresses** that have a **nine as the value of the first octet in IP address**.
- If we take an example IP address of **9.100.100.100**, the **network ID will be the first octet** and the **host ID** would be the **second, third, and fourth octets**.
- The **address class system** is a way of defining how the global IP address space is split up.

- There are **three primary types of address classes: Class A, class B, and class C**.

🔥 Network vs Host split (concept and example)  
  - **IP addresses are split into a network ID and a host ID**.  
  - Example: **9.100.100.100** — network ID = **first octet** (9), host ID = **second–fourth octets** (100.100.100).  
  - Note: **9.0.0.0/8 was historically assigned to IBM** (often cited as “IBM’s block”), not a current ownership claim.

🧮 How to identify class from leading bits and dotted-decimal ranges  
  - **Leading bit patterns:**  
    - **Class A:** starts with **0**.  
    - **Class B:** starts with **10**.  
    - **Class C:** starts with **110**.  
  - **Decimal first-octet ranges:**  
    - **Class A:** **0–127** (first octet).  
    - **Class B:** **128–191**.  
    - **Class C:** **192–223**.  
  - Reminder: each octet = **8 bits**, values **0–255**.

🧮 Address counts and scale differences (per network)  
  - **Class A host space:** **24 host bits** → **2^24 = 16,777,216** addresses per Class A network.
  - Class A address, the possible values for the first octet are 0-127.
  - **Class B host space:** **16 host bits** → **2^16 = 65,536** addresses per Class B network.
  - Class B addresses are restricted to those that begin with the first octet value of 128 through 191  
  - **Class C host space:** **8 host bits** → **2^8 = 256** addresses per Class C network.
  - Class C addresses begin with the first octet value of 192 through 223.
You might notice that this doesn't cover every possible IP address.
That's because there are two other IP address classes, but they're not quite as important to understand.
  - Implication: **Class A networks are vastly larger** than Class C networks; class choice historically determined scale of networks.

🌐 Classes D and E (other classes and purposes)  
  - **Class D:** starts with bits **1110**, decimal **224–239**, used for **multicasting** (one-to-many delivery).
  - Class D addresses always begin with the bits 1110 and are used for multicasting, which is how a single IP datagram can be send to an entire network at once.
These addresses begin with decimal values between 224 and 239.  
  - **Class E:** decimal **240–255**, **reserved/experimental/test use**, not for general assignment.
  - **Class E** addresses makeup all of the **remaining IP addresses**, but they are **unassigned** and **only used for testing purposes**.

⚠️ Special/reserved addresses and caveats within classful ranges  
  - **0.x.x.x** is special (unspecified/“this network” use) and **127.x.x.x** is reserved for **loopback** (e.g., **127.0.0.1**).  
  - Some first-octet values within the raw 0–127 Class A range are therefore **not usable as ordinary Class A networks**.

📌 Practical status and relevance  
  - The **classful addressing system has largely been replaced by CIDR (Classless Inter-Domain Routing)** for flexible allocation and route aggregation.  
  - Despite being largely obsolete operationally, **address classes remain important conceptually** for foundational networking understanding.

But the address class system is still in place in many ways and is important to

understand for anyone looking for a well-rounded networking education, And you know we're all about that.
