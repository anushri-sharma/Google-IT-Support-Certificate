
# [Basic Routing Concepts](https://www.youtube.com/watch?v=W_c0gPpeWZE&t=9s)

🧭 A router’s role and definition  
  - Forwards traffic based on **destination IP** and must connect at least two networks (has **≥2 network interfaces**).  
  - Routing is the process that allows communication between separate networks across the Internet.

🔁 Core packet-forwarding steps (applied repeatedly at each hop)  
  - Router receives a packet on one of its interfaces.  
  - Router strips data-link encapsulation and inspects the **IP datagram** for the destination IP.  
  - Router looks up the destination **network** in its **routing table**.  
  - Router forwards the packet out the interface chosen by the routing table (closest/next-hop path).

🖧 Two-network example (directly connected route)  
  - Network A: **192.168.1.0/24**, router IP **192.168.1.1** on A.  
  - Network B: **10.0.0.0/24**, router IP **10.0.0.254** on B.  
  - Host **192.168.1.100** → destination **10.0.0.10**: host sends frame to gateway MAC (router).  
  - Router inspects destination **10.0.0.10**, finds **10.0.0.0/24** in routing table, and has the host MAC via **ARP**.  
  - Router creates a new frame: decrements **TTL** by 1, recalculates checksum, sets source MAC to its B-interface and destination MAC to host, then forwards on Network B.

🔗 Three-network example (next-hop routing)  
  - Network C: **172.16.1.0/23**, second router with B-side IP **10.0.0.1** and C-side IP **172.16.1.1**.  
  - Host **192.168.1.100** → **172.16.1.100**: first router’s routing table points to **next-hop 10.0.0.1** (not directly connected).  
  - Packet is forwarded to the next-hop (TTL decremented); the next router sees destination as **directly connected** and delivers to final host.  
  - Demonstrates **next-hop vs directly connected** route handling.

⚙️ Router behavior and protocols used in forwarding  
  - Routers rebuild data-link frames for each outgoing interface while preserving the IP payload aside from **TTL** and checksum updates.  
  - **ARP** is used to discover layer‑2 (MAC) addresses for IPs on directly connected networks.  
  - Routing decisions are based solely on **routing table** entries (destination network and next-hop/interface).

🌐 Internet-scale considerations and redundancy  
  - Internet routing differs mainly by **scale**: routers often connect to many networks and traffic may traverse a dozen or more routers.  
  - Core routers form a **mesh topology**, offering multiple paths and resilience against link/router failures.  
  - Despite complexity and scale, the basic per-hop process (inspect IP → routing lookup → forward) remains the same for every packet.

- Finally, in order to protect against breakages, core Internet routers are typically connected in a mesh, meaning that there might be many different paths for a packet to take.

- Still, the concepts are all the same.

- Routers inspect the destination IP, look at their routing table to determine which path is the quickest, and forward the packet along the path.

- This happens over and over, every single packet making up every single bit of traffic all over the Internet at all times.
