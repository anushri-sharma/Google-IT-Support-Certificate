
# [Address Resolution Protocol](https://www.youtube.com/watch?v=B9c366BYk3U&t=5s)

You now understand how both **MAC addresses are used at the data link layer and how IP addresses are used at the network layer.**

How these two separate address types relate to each other.

This is where **address resolution protocol, or ARP**, comes into play.

<img width="1264" height="391" alt="image" src="https://github.com/user-attachments/assets/492a2bfc-3f7b-4014-8b75-fb64e190298c" />

**ARP is a protocol used to discover the hardware address of a node with a certain IP address.**

Once an **IP datagram** has been fully formed, it needs to be **encapsulated inside an Ethernet frame**.

This means that the transmitting device needs a destination MAC address to complete the Ethernet frame header.

Almost all network connected devices will retain a local ARP table.

**An ARP table is just a list of IP addresses and the MAC address is associated with them.**

<img width="1273" height="349" alt="image" src="https://github.com/user-attachments/assets/7d328ee6-b259-452a-8027-c3896c354e66" />

Let's say we want to send some data to the IP address 10.20.30.40.

<img width="1280" height="613" alt="image" src="https://github.com/user-attachments/assets/6726f111-fe8a-48d4-be69-03e2abbd4940" />

It might be the case that this destination doesn't have an entry in the ARP table.

When this happens, the node that wants to send data sends a broadcast ARP message to the MAC Broadcast address, 
which is all **Fs.**

<img width="1275" height="596" alt="image" src="https://github.com/user-attachments/assets/087e4d03-e814-4ada-b157-2fde9fea7930" />

These kinds of broadcast ARP messages are delivered to all computers on the local network.

When the network interface that's been assigned an IP of 10.20.30.40 receives this ARP broadcast, it sends back what's known as an ARP response.

<img width="1271" height="589" alt="image" src="https://github.com/user-attachments/assets/3128e15f-251a-45cc-9251-46406fb31ba9" />

**This response message will contain the MAC address for the network interface in question.**

Now the transmitting computer knows what MAC address to put in the destination hardware address field and the Ethernet frame is ready for delivery.

<img width="1275" height="612" alt="image" src="https://github.com/user-attachments/assets/34a12cee-b81e-455a-bb60-569531fa0f50" />

It will also likely store this IP address in its local ARP table so that it
won't have to send an ARP broadcast the next time it needs to communicate with this IP.


<img width="1273" height="279" alt="image" src="https://github.com/user-attachments/assets/79e956ed-2a63-4754-94a8-7e24ccf2c6eb" />


ARP table entries generally expire after a short amount of time to ensure changes in the network are accounted for.
