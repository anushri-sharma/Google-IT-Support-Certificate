🔥 Classful addressing limitations  
  – **Network ID sizes were fixed** at **/8 (Class A)**, **/16 (Class B)**, and **/24 (Class C)**, producing mismatched network sizes for many organizations.  
  – Fixed sizes led to routing-table bloat when organizations used multiple contiguous Class C blocks routed to the same place.  
  – Fixed sizes also caused inefficient allocations (e.g., a single Class B often far larger than needed; a Class C often too small).

Many companies ended up with various adjoining class C networks to meet their needs.

That meant that routing tables ended up with a bunch of entries for a bunch of class C networks that were all actually being routed to the same place.

This is where CIDR or classless inter-domain routing comes into play.

CIDR is an even more flexible approach to describing blocks of IP addresses.

It expands on the concept of subnetting by using subnet masks to demarcate networks.

To **demarcate something means to set something off.**

When discussing computer networking, you'll often hear the term demarcation point to describe where one network or system ends and another one begins.

In our previous model, we relied on a network ID, subnet ID, and host ID to deliver an IP datagram to the correct location.

**With CIDR, the network ID and subnet ID are combined into one.**

🍳 What CIDR (Classless Inter-Domain Routing) does  
  – **Abandons address classes** and treats addresses with arbitrary prefix lengths expressed in **slash notation** (CIDR notation).  
  – **Combines network ID and subnet ID** into a single prefix; an address is defined by a **network prefix** and a **host ID** only.  
  – Uses **subnet masks/prefix lengths** to demarcate networks (demarcation = where one network ends and another begins).

💡 CIDR notation example and interpretation  
  – Address **9.100.100.100/24** uses a **/24 mask (255.255.255.0)**, so the **network ID = 9.100.100** and the **host ID = .100**.  
  – Routers use the prefix length to determine the network portion rather than an implicit class.

🧩 Aggregation (supernetting) benefit and example  
  – Multiple contiguous smaller blocks can be combined into a single larger prefix to reduce routing entries.  
  – Example: two **/24** networks combined as a **/23 (255.255.254.0)** yields one routing entry instead of two.  
  – **Routing implication:** routers only need the aggregated **/23** route to reach all addresses in both former **/24**s.

🔢 Address-count math and usable hosts (practical implication)  

- Before, network sizes were static.

- Think only class A, class B, or class A, and only subnets could be of different sizes.

- CIDR allows for networks themselves to be differing sizes.

- Before this, if a company needed more addresses than a single class C could provide, they'd need an entire second class
C. With CIDR, they could combine that address space into one contiguous chunk with a net mask of slash 23 or 255.255.254.0.

- This means that routers now only need to know one entry in their routing table to deliver traffic to these addresses instead of two.

- It's also important to call out that you get additional available host IDs out of this practice.

- Remember that you always lose two host IDs per network.

  – A **/24** contains **2^8 = 256** addresses; **usable = 256 − 2 = 254** (network + broadcast reserved).  
  – Two separate **/24**s provide **254 + 254 = 508** usable addresses.  
  – A single **/23** contains **2^9 = 512** addresses; **usable = 512 − 2 = 510** — **2 more usable hosts** than two separate /24s and one fewer routing entry.

⚠️ Operational rule and takeaway  
  – **Always subtract two** addresses per IPv4 network for **network** and **broadcast** when calculating usable hosts.  
  – CIDR enables **arbitrary-sized networks**, improved address utilization, and reduced routing-table entries through aggregation.
