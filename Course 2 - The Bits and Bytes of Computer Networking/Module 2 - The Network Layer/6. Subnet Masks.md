
# [Subnet Masks](https://www.youtube.com/watch?v=_UECZ03Ok0E&t=5s)



🔥 **Three ID components in one IP address: network ID, subnet ID, host ID**  
  – **Network ID** identifies the broad network (used by Internet core routers).  
  – **Subnet ID** subdivides a network into smaller addressable segments (used by gateway routers).  
  – **Host ID** identifies the specific machine within the subnet (used by the final router to deliver datagrams).

💡 **IP addresses and binary structure**  
  – An IP address is a **32-bit number** written as four octets (decimal).  
  – Example: **9.100.100.100** in binary: **00001001.01100100.01100100.01100100** (each octet is 8 bits).

🔍 **What a subnet mask is and how it works**  
  – A subnet mask is a **32-bit number** with a contiguous run of **ones** followed by **zeros**.  
  – **Ones** indicate bits that belong to the **network/subnet IDs** (can be ignored when extracting the host ID).  
  – **Zeros** indicate bits that belong to the **host ID** (kept when extracting the host part).

🧭 **How routers use the IDs**  
  – **Core routers** route on the **network ID** only.  
  – **Gateway routers** (at the network edge) use subnet information to forward toward the correct subnet/next hop.  
  – The **final router** uses the **host ID** to deliver to the exact machine.

📐 **Subnet mask examples and boundaries**  
  – **255.255.255.0** = **/24** → **24 ones**, **8 zeros**; only the last octet is host space.  
  – **255.255.255.224** = **/27** → **27 ones**, **5 zeros**; **5 host bits** → **32 total addresses**.

🔢 **Address counts and usable hosts**  
  – An n-bit host field yields **2^n total addresses** (e.g., 8 bits → **256** addresses).  
  – Typically **usable host addresses = 2^n − 2**, because **all-zeros (network address)** and **all-ones (broadcast)** are reserved.  
  – Examples: **/24** → **256 total**, **254 usable**; **/27** → **32 total**, **30 usable**.

Let's say we're dealing with our old friend 9.100.100.100 with a subnet mask of 255.255.255.224.

Since that subnet mask represents 27 ones followed by five zeros, a quicker way of referencing this is with the notation /27.

✂️ **How subnet IDs are derived (example with classful illustration)**  
  – Using a **class A** example IP **9.100.100.100** (classful assumption: network = first octet), the remaining three octets are split by the subnet mask.  
  – Bits in those remaining octets with corresponding **mask ones** form the **subnet ID**; bits with **mask zeros** form the **host ID**.

🔤 **CIDR shorthand and dual notation**  
  – A mask can be written in dotted decimal (**255.255.255.224**) or prefix length shorthand (**/27**).  
  – Example full notation: **9.100.100.100/27**. Both notations are commonly used.

The entire IP and subnet mask could be written out as 9.100.100.100/27.

📚 **Practical note on learning masks**  
  – People often memorize common masks, but understanding that masks are binary strings of ones then zeros clarifies how subnets and host ranges are determined.  
  – The mask alone **defines the subnet size**, independent of classful network boundaries used for illustration.

⚠️ **Edge case and modern context**  
  – The classful example (network = first octet for class A) is a legacy teaching device; modern routing typically uses **CIDR** (prefix-based) addressing where the prefix length defines network/subnet boundaries.
