
# [Wireless Security](https://www.youtube.com/watch?v=97Htm4yFRbE&t=1s)

🔥 Wired vs. wireless privacy contrast  
  – Wired links provide inherent privacy because only the two endpoint devices on the cable normally see the traffic.  
  – Wireless transmits radio signals through the air, so any device within range can potentially intercept transmissions whether intended for them or not.

🔒 WEP: “Wired Equivalent Privacy” — legacy, weak encryption  
  – **WEP** was created to provide privacy for wireless links but offers only low-level protection.  
  – The common figure cited as **40-bit** refers to a **40-bit secret key plus a 24-bit IV**, often labeled as 64-bit WEP; later variants used larger keys but the algorithm remained weak.  
  – Practical implication: WEP can typically be **cracked in minutes** with modern tools and is considered deprecated.

💡 Encryption key length and security principle  
  – **Key length (bits)** correlates with difficulty to brute-force: more bits → exponentially harder to crack.  
  – Examples from wireless history: roughly **40-bit** (WEP, weak) → **128-bit** (WPA/TKIP-era) → stronger AES-based schemes used in later standards.

🔐 WPA and WPA2: stronger successors, implementation nuances  
  – **WPA** replaced WEP and introduced per-packet keying (TKIP); it is significantly stronger than WEP and commonly described with **128-bit** security components.  
  – **WPA2** upgraded to **AES-based CCMP**, providing substantially stronger protection than WEP; common implementations use AES-based ciphers (typically **128-bit AES** in CCMP).  
  – Operational nuance: security also depends on deployment mode—Pre-Shared Key (PSK) security is limited by passphrase strength; weak passphrases enable offline dictionary attacks even with WPA/WPA2.

🧩 MAC filtering: access control, not encryption  
  – **MAC filtering** restricts which device MAC addresses an access point will accept, providing an additional barrier to joining the network.  
  – It does **not** encrypt wireless traffic or protect data in flight.  
  – Practical limitation: MAC addresses can be **spoofed**, so MAC filtering is easily bypassed by determined attackers.

⚙️ Practical implications and best practices (technical facts)  
  – Treat **WEP as unsafe and obsolete**; do not rely on it for protection.  
  – Use **WPA2 (AES/CCMP)** or newer standards (e.g., WPA3 where available) and enforce **strong passphrases** for PSK deployments.  
  – Combine robust encryption with network access controls, but do not rely on MAC filtering alone because it can be bypassed.
