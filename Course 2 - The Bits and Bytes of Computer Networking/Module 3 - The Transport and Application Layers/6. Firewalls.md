
# [Firewalls](https://www.youtube.com/watch?v=st4sqPkeCT0&t=8s)

A **firewall** is just a device that blocks traffic that meets certain criteria.

**Firewalls** are a critical concept to keeping a network secure, since it's the primary way you can stop traffic you don't want from entering a network.

#### Firewalls can actually operate at lots of different layers of the network.

There are firewalls that can perform inspection of application layer traffic and firewalls that primarily deal with blocking ranges of IP addresses.

The reason we cover firewalls here is that they're most commonly used at the transportation layer.

**Firewalls that operate at the transportation layer will generally have a configuration that enables them to block traffic to certain ports while allowing traffic to other ports.**
🔥 **Definition and purpose of a firewall**  
  – A **firewall** is a device or program that **blocks traffic meeting specified criteria**, used to stop unwanted traffic from entering a network and to keep the network secure.

💡 **Layers and inspection capabilities**  
  – Firewalls can operate at **multiple layers** of the network stack: examples include **application-layer** inspection (examining application payloads) and **network-layer** actions like blocking IP ranges.  
  – The content emphasizes the **transport layer** as the most common use case for firewalls, where rules are typically based on **port numbers**.

🧩 **Transport-layer behavior and limits**  
  – **Transport-layer firewalls** generally allow or block traffic based on **ports**, enabling simple access control for services.  
  – Implication: port-based filtering controls which services are reachable but does not inspect application payloads the way application-layer firewalls do.

🏢 **Illustrative small-business example**  
  – A single server may host multiple services (e.g., a public **web server** and an internal **file server** containing confidential documents).  
  – A perimeter firewall can be configured to **allow incoming traffic to port 80** (web access) while **blocking all other ports from external IPs**, preventing outside access to the file server.  
  – This demonstrates how selective port rules expose public services while protecting internal resources.

```
Let's imagine a simple small business network.

This small business might have one server, which hosts multiple network services.

This server might have a web server that hosts the company's website while also serving as the file server for a confidential internal document.

A firewall placed at the perimeter of the network could be configured to allow anyone to send traffic to port 80 in order to view the webpage.

At the same time, it could block all access for external IPs to any other
port so that no one outside of the local area network could access the file server.
```

🔌 **Forms and placements of firewalls**  
  – Firewalls can be **independent network devices** or **software programs** that run on different platforms.  
  – For many companies and most home users, a single device often performs both **router + firewall** roles.  
  – Firewalls can also run on individual machines as **host-based firewalls**, providing per-host control.

🖥️ **Host-level firewall capability**  
  – **All major modern operating systems** include built-in firewall functionality.  
  – Host-based rules let administrators block or allow traffic to specific **ports** and therefore to particular services on that host.

**Firewalls are sometimes independent network devices**, but it's really better to think of them as a program that can run anywhere.

For many companies and almost all home users, **the functionality of a router and a firewall is
performed by the same device, and firewalls can run on individual hosts instead of being a network device.**

All major modern operating systems have firewall functionality built in.

That way, blocking or allowing traffic to various ports and therefore to specific services can be performed at the host level as well.
