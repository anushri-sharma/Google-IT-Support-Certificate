🔥 **Transport layer responsibilities**
  
  – **Multiplexing and demultiplexing** traffic between multiple services on a node.  
  – **Establishing long-running connections** between endpoints.  
  – **Ensuring data integrity** via error checking and data verification.
  – The ability to multiplex/demultiplex **sets the transport layer** apart from lower layers.

🎯 **Learning goals covered**
  
  – Be able to **describe multiplexing/demultiplexing** and how they work.  
  – Be able to **identify differences between TCP and UDP**, and **explain the TCP three‑way handshake** and **TCP flags** (introduced as objectives).  
  – Be able to **describe basic firewall functions** for keeping networks safe (introduced as an objective).

🔁 **What multiplexing and demultiplexing mean**
  
  – **Multiplexing:** a node directs inbound network traffic to many different receiving services (multiple services sharing one host).  
  – **Demultiplexing:** the receiving node takes traffic addressed to the same IP and delivers each flow to the correct local service.  
  – This ability **distinguishes the transport layer** from lower layers.

📍 **Ports and socket addresses**

<img width="1280" height="443" alt="image" src="https://github.com/user-attachments/assets/44c132d9-341c-47d1-97a0-03c9d5022845" />

  
  – A **port** is a **16‑bit number** used to route traffic to a specific service on a host (**range 0–65535**).  
  – Ports are written after an IP with a colon; the combination is a **socket address/socket number** (for example, **10.1.1.100:80**).  
  – The **colon notation** (IP:port) is the standard way to denote the destination service endpoint.

🧩 **Server/client model and examples**
  
  – **Server/service:** a program waiting for incoming requests on a specific port.  
  – **Client:** a program initiating requests to a server’s IP and port.  
  – Common examples: **HTTP (unencrypted web traffic) → port 80**, **FTP → port 21**. 
  to reach FTP on the same host use **10.1.1.100:21.**
  – A single machine can host many services (web, mail, file sharing, print, etc.) simultaneously because each service
  listens on a different **port**, enabled by multiplexing/demultiplexing.
