
# [Unicast, Multicast, and Broadcast](https://www.youtube.com/watch?v=f3XxaoLCccE&t=4s)

🔥 **Unicast — one-to-one Ethernet transmission**  
  - **Identified by the I/G bit = 0** (the least significant bit of the first octet of the destination MAC).  
  - **Frame is addressed to a single MAC address**, so only that device will process it; on a shared medium it is still physically seen by all devices in the collision domain but only the intended recipient accepts it.  
  - **Use case:** standard one-to-one communications between two hosts.

- At the ethernet level, this is done by looking at a special bit in the destination MAC address.

- If the least significant bit in the first octet of a destination address is set to zero, it means that ethernet frame is
  intended for only the destination address.

- This means it would be sent to all devices on the collision domain, but only actually received and processed by the intended destination.

- If the least significant bit in the first octet of a destination address is set to one, it means you're dealing with a multicast frame.

- **A multicast frame is similarly set to all devices on the local Network segment**.

- What's different is that it will be accepted or discarded by each device depending on criteria aside from their own hardware MAC address.

- Network interfaces can be configured to accept lists of configured multicast addresses for these sorts of communications.

💡 **Multicast — one-to-many to a selected group**  
  - **Identified by the I/G bit = 1** (least significant bit of the first octet set to one).  
  - **Delivered to the local network segment** but **each device accepts or discards the frame based on membership/filters**, not simply its unicast MAC.  
  - **Network interfaces can be configured with lists of multicast addresses** (hardware or driver filters) so only interested NICs process matching multicast frames.  
  - **Implication:** more efficient than broadcast when targeting a group, but requires group membership/filtering to limit processing.

📣 **Broadcast — one-to-all on the LAN**  
  - **Destination MAC = all F's (FF:FF:FF:FF:FF:FF)**, explicitly targeting every device on the LAN.  
  - **Every device on the local broadcast domain receives and processes the frame**, used where every host must see the packet.  
  - **Common use:** device discovery and address resolution (for example, ARP-style discovery) so devices can learn about each other.

- An ethernet broadcast is sent to every single device on a LAN.

- This is accomplished by using a special destination known as a broadcast address.
  
- The **ethernet broadcast address is all F's.**

- Ethernet broadcasts are used so that devices can learn more about each other.

⚙️ **Practical notes, edge cases and behavior**  
  - **Collision domain note:** on shared-media segments a unicast frame is still visible to all nodes at the physical layer but only the addressed node processes it.  
  - **NIC modes:** in normal mode NICs filter by unicast/multicast/broadcast criteria; in promiscuous mode a NIC can accept all frames regardless of those bits.  
  - **Filtering and scaling:** multicast requires membership/filtering to avoid unnecessary processing; broadcasts reach all hosts and can increase load on a LAN.
