

# [DHCP - Dynamic host configuration protocol](https://www.youtube.com/watch?v=2pBrEVO0GXY&t=8s)


🔥 **What DHCP is (Dynamic Host Configuration Protocol)**  
  - **DHCP** automatically assigns IP addresses to devices on a network by leasing addresses from a DHCP server.  
  - The server issues a **lease** rather than a permanent assignment, removing the need to manually configure clients.

💡 **Static IP vs DHCP-assigned IP (trade-offs and implications)**  
  - **Static IP**: you manually set an IP on each device and must **keep track of every assigned address** to avoid conflicts.  
  - **DHCP-assigned IP**: devices receive addresses automatically, reducing manual effort and human error.  
  - If you **expand the IP address range**, DHCP clients update automatically without changing settings on each machine.

🛠️ **Core DHCP configuration items you must decide**  
  - **IP range** (scope) from which the DHCP server will assign addresses; choose a range that fits network size and avoids conflicts with reserved/static addresses.  
  - **Subnet mask** to define the network portion of addresses.  
  - **Default gateway** to provide clients with the router IP for external network access.  
  - **DNS server addresses** if you want DHCP to tell clients which DNS servers to use.  

🔗 **DHCP and DNS integration (behavior and benefit)**  
  - You can configure DHCP to provide **local DNS server addresses** to clients.  
  - When DHCP leases out new addresses and is integrated with DNS, **DNS records update automatically** to reflect current IP-to-hostname mappings, keeping name resolution in sync with leases.

⚙️ **DHCP server software and deployment details**  
  - Different DHCP server implementations expose settings in different ways; **vendor-specific configuration layouts vary**, so consult product documentation.  
  - **Windows Server** includes a DHCP service that can be enabled and configured natively.  
  - After enabling a DHCP server and setting clients to accept DHCP, clients should begin receiving addresses per the configured settings.

📝 **Operational notes and practical cautions**  
  - Ensure the DHCP **scope does not overlap static addresses** or another DHCP server’s scope to prevent IP conflicts.  
  - Verify DNS integration settings if automatic hostname-to-IP updates are required; the DHCP and DNS services must be correctly configured to synchronize.
