
# [Configuring Services in Windows](https://www.youtube.com/watch?v=iKz5J94X2U4&t=1s)


## Notes

**Summary of Windows Service Management Introduction**  
The video begins by introducing the process of managing Windows services, specifically focusing on Internet Information Services (IIS), which is Windows’ built-in web server feature. The presenter contrasts this with previously demonstrated Linux service management tasks such as starting, stopping, restarting, modifying, and reloading service configurations. The first step in Windows involves enabling IIS via the “Turn Windows features on and off” option in the Control Panel, which leads to the Server Manager interface.

Key points:  
- IIS is the Windows equivalent to Linux web server services.  
- The feature is enabled through Control Panel’s feature management interface, opening Server Manager.  

 
**Enabling Internet Information Services (IIS) via Server Manager**  
The presenter details the step-by-step procedure to enable IIS by navigating through Server Manager dialogs:  
- Clicking "Next" three times to move through initial setup screens.  
- Scrolling to locate and select “Web Server (IIS)”.  
- Adding the feature to the installation queue.  
- Proceeding with multiple “Next” confirmations before clicking “Install”.  

This installation enables the web server component on the Windows machine. The process installs all necessary dependencies for IIS to function.

Highlights:  
- The installation requires confirming multiple steps (“Next” clicks).  
- Web Server (IIS) is explicitly chosen as the feature to enable.  
- The system installs all associated components for IIS.  


**Confirmation of IIS Installation and Service Manager Integration**  
After installation completes, the video shows that a new service entry labelled “IIS” appears in the Windows Service Manager. This confirms that IIS is now installed and running as a service. Configuration access is available by right-clicking the IIS entry and selecting “Internet Information Services Manager.”

Key insights:  
- IIS service appears in Services Manager post-installation.  
- Configuration is performed via IIS Manager accessible from the service list.  

  
**Viewing and Managing Websites within IIS Manager**  
Within IIS Manager, the presenter expands the server node and navigates to the “Sites” section, which lists websites managed by IIS. Initially, there is only one default website named “Default Website.” The presenter demonstrates verifying this site’s functionality by opening Internet Explorer and navigating to `localhost`, confirming that IIS serves the default webpage correctly.

Summary:  
- IIS Manager categorises websites under “Sites”.  
- Default Website is pre-configured and accessible on localhost.  
- Verification is done by browsing to `http://localhost`.  


**Adding a New Website to IIS**  
The video proceeds to demonstrate adding a custom website:  
- The presenter copies a prepared example website folder from the Documents directory to the IIS default web directory, `C:\inetpub`.  
- The copy operation requires administrator permission, indicated by a security prompt.  
- After copying, the IIS Manager console is used to add a new website entry via right-clicking the “Sites” node and selecting “Add Website.”  

During the add website dialog, the following fields are configured:  
- **Site Name:** example  
- **Physical Path:** the newly copied folder inside `inetpub`  
- **Port:** 8080 (to avoid conflict with the default website on port 80)  

After submission, IIS confirms the new site is running.

Key details:  

| Field           | Value                   | Purpose                           |
|-----------------|-------------------------|---------------------------------|
| Site Name       | example                 | Identifier for the new website   |
| Physical Path   | C:\inetpub\example      | Location of website files        |
| Port            | 8080                    | Runs site on non-default port    |


**Testing the Newly Added Website on Port 8080**  
To verify the new website’s operation, the presenter opens Internet Explorer and navigates to `http://localhost:8080`. The successful loading of the new website confirms proper configuration and functionality.

Conclusions:  
- Multiple websites can run simultaneously on different ports (default port 80 and custom port 8080).  
- IIS supports easy website addition and management through its GUI.  


**Final Summary and Preview of Next Steps**  
The video closes by summarising the core learning points:  
- Installation, management, and configuration of both Linux and Windows services have been demonstrated.  
- The specific example was IIS for Windows, showing enabling the feature, configuring services, and adding websites.  
- A preview is given that upcoming lab exercises will allow hands-on practice of these tasks.

**Key Takeaways:**  
- Windows service management involves enabling features via Control Panel and Server Manager.  
- IIS is the default Windows service for hosting websites, easily managed through IIS Manager.  
- Websites can be hosted on different ports to allow multiple simultaneous sites.  
- The video prepares viewers for practical application through lab exercises.

### Summary Table: IIS Website Setup Process

| Step                       | Description                                     | Tools/Locations                     |
|----------------------------|------------------------------------------------|-----------------------------------|
| Enable IIS                 | Turn Windows features on → Server Manager → Select Web Server (IIS) → Install | Control Panel, Server Manager      |
| Confirm IIS service        | Check for IIS in Services Manager               | Services Manager                   |
| Verify default website     | Access `http://localhost` in browser            | IIS Manager, Internet Explorer    |
| Copy website files         | Copy example site folder to `C:\inetpub`        | File Explorer                     |
| Add new website in IIS     | Right-click Sites → Add Website → Set name, path, port | IIS Manager                      |
| Test new website           | Access `http://localhost:8080`                   | Internet Explorer                 |

### Core Concepts Highlighted  
- **Windows feature management** for enabling/disabling services.  
- **IIS as a Windows web server service**, analogous to Linux web services.  
- **IIS Manager** as a GUI tool for configuring and managing websites.  
- **Port-based website hosting**, allowing multiple sites on one server.  
- **File system structure for IIS (`C:\inetpub`)** as the default web content directory.
