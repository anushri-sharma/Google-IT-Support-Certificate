
# [Types of IT Infrastructure Services](https://www.youtube.com/watch?v=OhBA3WPQkDM&t=7s)


🏗️ **On-premises physical infrastructure (traditional model)**  
  – **Buy or rent hardware** for servers and **manage end-to-end** (on-site or colocated).  
  – Responsibilities include hardware maintenance, physical storage, and lifecycle management.  
  – Use cases: dedicated file servers, custom hardware requirements, full control over environment.

- **Traditional IT Infrastructure** involves owning or renting physical servers (e.g., for file storage), managing hardware, maintaining operating systems, applying security patches, and configuring networks internally.

☁️ **Infrastructure as a Service (IaaS)**  
  – **IaaS = pre-configured virtual machines** that behave like physical servers (removes physical hardware management).  
  – You still typically **manage the guest OS** (installing apps, applying security patches, updates) unless you layer on managed services.  
  – Examples: **AWS EC2**, **Linode**, **Microsoft Azure** (formerly Windows Azure), **Google Compute Engine**.

**Infrastructure as a Service (IaaS):**  
  - Cloud providers offer pre-configured **virtual machines** that operate like physical servers.  
  - Users do not need to manage hardware directly but are responsible for the operating system and software on the virtual servers.  
  - Examples: Amazon Web Services EC2, Linode, Windows Azure, Google Compute Engine.

🌐 **Networking and Networking as a Service (NaaS)**  
  – On-prem network tasks: assign **IP addresses** (static or DHCP), configure **subnets**, DNS, wireless access, and routing.  
  – In large companies a dedicated networking team often handles these; in smaller companies one person may do it.  
  – **NaaS** lets organizations outsource networking infrastructure to the cloud to avoid buying expensive hardware and to offload **network security, routing, WANs and private networking (VPNs)**.  
  – Implication: reduces capital expense and on-site hardware work but creates reliance on provider-managed networking features.

**Networking as a Service (NaaS):**  
  - A cloud-based alternative to managing corporate networks internally.  
  - Offloads the management of networking hardware, security, routing, and virtual private networks to the provider.  
  - Useful for companies that want to reduce costs and operational complexity in networking.

📄 **Software as a Service (SaaS)**  
  – **SaaS = software delivered over the web**, eliminating per-machine installation and many local maintenance tasks.  
  – Examples: **Microsoft Office 365**, **Google Workspace** (formerly G Suite) for document editing, email, spreadsheets, presentations via browser.  
  – Considerations: licensing, centralized updates, and fewer local admin tasks but recurring subscription costs and provider dependency.

**Software as a Service (SaaS):**  
  - Cloud-hosted software applications accessible via web browsers.  
  - Eliminates the need to install and maintain software on individual machines.  
  - Examples: Microsoft Office 365, Google G Suite.

🚀 **Platform as a Service (PaaS)**  
  – **PaaS = integrated platform** to build, store, and serve applications (handles runtime, scaling, and many infra concerns).  
  – Developer needs addressed: programming environments, code build/deploy pipeline, databases, and web hosting without managing OS or infrastructure.  
  – Examples: **Heroku**, **Google App Engine**, **Microsoft Azure App Service**.  
  – Implication: accelerates development and deployment but may constrain control over underlying environment and incur vendor lock-in.

**Platform as a Service (PaaS):**  
  - Provides an integrated platform to develop, build, deploy, and host web applications.  
  - Includes development tools, databases, and web servers in a single solution.  
  - Examples: Heroku, Windows Azure, Google App Engine.

🗂️ **Directory services and Directory-as-a-Service (DaaS)**  
  – **Directory services** centralize user and computer accounts for access, authorization, onboarding/offboarding (examples: **Windows Active Directory**, **OpenLDAP**).  
  – **DaaS** offers cloud-hosted directory functionality; note: **DaaS** can also mean Desktop-as-a-Service in other contexts—here it refers to **Directory as a Service**.  
  – Benefit: centralized identity management and easier administration; trade-offs include provider dependency and recurring cost.

- **Directory as a Service (DaaS):**  
  - Cloud-based directory services for centralised management of users and computers.  
  - Facilitates user access control, authentication, and authorisation.  
  - Examples: Windows Active Directory (on-premises), OpenLDAP, DaaS cloud providers.
 
  
⚖️ **Cloud vs. on-premise trade-offs and cautions**  
  – **Benefits of cloud**: reduces physical hardware management, faster provisioning, managed services for networking, directories, platforms, and software.  
  – **Cons of cloud**: **recurring costs**, **dependence on provider availability and features**, potential vendor lock-in, and sometimes less direct control over OS/runtime.  
  – Operational caution: **understand how a service works and how to maintain it** before adopting—cloud reduces some responsibilities but does not eliminate the need for system knowledge or security practices.



  ### Key Insights

- **Cloud services reduce the need for in-house management** of physical infrastructure, networking, software deployment, and user access, allowing organisations to focus on core business activities.

- **IaaS offers flexibility** by providing virtual servers that behave like physical ones but without hardware maintenance.

- **NaaS offloads complex networking tasks**, including routing and security, reducing capital expenditure on expensive network hardware.

- **SaaS simplifies software usage** by providing applications accessible via browsers, eliminating installation and update overheads.

- **PaaS supports streamlined application development**, integrating coding environments, databases, and hosting into one service.

- **DaaS centralises user management** and improves security by managing authentication and authorisation in the cloud.

- Despite the advantages, **cloud services come with trade-offs**, such as recurring costs and dependency on providers’ service availability and policies.
