# Notes

**Summary of DNS Fundamentals and Importance**

- **DNS (Domain Name System)** is a critical network service that **maps human-readable domain names to IP addresses**.
- Proper DNS setup is essential for **ensuring users can access websites by their names** rather than remembering complex numerical IP addresses.
- On personal computers, DNS configuration is typically automatic and invisible to the user: when a new device connects to the Internet, it automatically receives the DNS server address from the connected router.
- This DNS server address is usually provided by the **Internet Service Provider (ISP)**, allowing seamless access to websites without user intervention.

**Key Functions of DNS:**

1. **Internet Website Access:**  
   DNS enables the Internet to locate the correct IP address associated with a website’s domain name, allowing users to reach web services easily.

2. **Internal Network Usability:**  
   - For businesses or organisations running web services or servers, DNS is necessary to **advertise the IP address of their services to the Internet**.
   - DNS also facilitates **remote access to servers or user machines via memorable hostnames rather than numeric IP addresses**, simplifying management and use.

**Critical Insight:**  
Without correctly configured DNS, **users cannot reach websites by their domain names**, and network management becomes cumbersome, especially in enterprise environments.

---

**How DNS Works During Network Connection**

- When a device connects to a network, the router typically provides the device with a **DNS server address**.
- This address is configured in the device’s network settings and is often the ISP’s DNS server.
- The DNS server handles the translation of domain names into IP addresses behind the scenes, so users can simply enter web addresses without knowing or entering IP addresses.

**Summary of Process:**

| Step | Description                                   |
|-------|----------------------------------------------|
| 1     | Device connects to a network                  |
| 2     | Router assigns DNS server address to device  |
| 3     | Device uses DNS server to resolve domain names into IP addresses |
| 4     | User accesses websites by domain names seamlessly |

---

**Reasons for Setting Up Your Own DNS Servers**

- Although DNS “just works” by default using ISP DNS servers, there are **two main reasons organisations set up their own DNS servers:**

1. **Hosting Web Services:**
   - Organisations running websites or other Internet-accessible services must **publish DNS records that point their domain names to the correct IP addresses**.
   - Without control over DNS records, the Internet cannot resolve the service’s domain name to the correct server.

2. **Remote Access and Internal Network Management:**
   - Remote access to servers or user machines is simpler and more user-friendly by using hostnames rather than IP addresses.
   - Setting up DNS ensures that **hostnames are mapped correctly to internal or external IP addresses**, enabling easy connection and management.

**Key Conclusion:**  
Setting up DNS is fundamental not only for public-facing services but also for **internal network usability and ease of administration**.

---

### Overall Key Insights

- **DNS is foundational** for both public Internet access and private network management.
- The **automatic DNS configuration on personal devices masks DNS complexity** but relies on network infrastructure providing correct DNS server information.
- Organisations must manage their own DNS servers or DNS records for:
  - Publishing Internet-accessible services.
  - Simplifying remote access through hostnames.
- Failure to configure DNS properly results in **inaccessibility of services by domain names**, which impacts usability and service availability.
