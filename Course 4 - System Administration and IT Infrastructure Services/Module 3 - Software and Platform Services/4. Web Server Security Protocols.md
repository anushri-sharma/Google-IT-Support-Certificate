## Smart Summary
**Introduction to Security Services in IT Infrastructure**  
Security services are critical components integrated throughout all aspects of IT infrastructure. They encompass various protocols and measures designed to protect data, ensure authentication, and maintain secure communication channels. Although this overview is brief, a more detailed exploration is reserved for a dedicated IT security course. The key takeaway is that security underpins all IT operations and must be considered by everyone involved in managing systems, not only specialised security engineers.

**The Importance of Communicating Security to Users**  
When managing web servers that serve content to users, it is essential to communicate clearly that interactions are secure. For example, when a user logs into an online banking site, the URL commonly begins with **HTTPS**, signalling a secure connection.  

- **HTTP (Hypertext Transfer Protocol):** The foundational protocol used to format and transfer web content across the Internet, shown at the start of most URLs.  
- **HTTPS (Hypertext Transfer Protocol Secure):** The secure version of HTTP that ensures encrypted communication between the user’s browser and the website, enhancing privacy and data integrity.

**Protocols Enabling HTTPS: TLS and SSL**  
HTTPS security relies on two main protocols:

| Protocol           | Description                                                                                      | Status                        | Notes                              |
|--------------------|--------------------------------------------------------------------------------------------------|-------------------------------|-----------------------------------|
| **Transport Layer Security (TLS)** | The current standard protocol used to secure communications over a network via encryption. | Widely used, actively maintained | Supports many applications beyond web browsing. |
| **Secure Sockets Layer (SSL)**      | An older protocol designed to secure communication between client and server.            | Deprecated in favour of TLS    | SSL 3.0 is essentially TLS 1.0; often referenced interchangeably with TLS. |

- TLS has superseded SSL due to enhanced security features and ongoing updates.
- Although SSL is deprecated, its name persists in common usage (e.g., **SSL/TLS**) due to historical context and legacy systems.

**Technical Evolution: SSL to TLS**  
- SSL version 3.0 and TLS version 1.0 are effectively the same protocol under different names, but TLS introduced improvements that address vulnerabilities found in SSL.  
- TLS continues to evolve with new versions to maintain robust security standards, making it the preferred protocol for secure Internet communication.

**Enabling TLS on a Web Server: The Role of Digital Certificates**  
To activate HTTPS on a server, one must obtain a **digital certificate** from a trusted third party known as a **Certificate Authority (CA)**. The process involves:

- The **Certificate Authority** verifies the identity of the website owner and asserts control over the web server.  
- Once verified, the CA issues a digital certificate that can be installed on the server.  
- This certificate enables the server to initiate encrypted communication sessions with clients using TLS, resulting in the browser displaying HTTPS in the URL bar.

**Key points about digital certificates:**  
- They function as a **trust mechanism**, assuring users that the website they access is authentic and secure.  
- This trust is foundational for secure online transactions, data privacy, and overall confidence in web services.

**Security as a Shared Responsibility Across IT**  
Security is not solely the domain of specialised security engineers; it is a fundamental concern for **everyone involved in IT infrastructure**. Every layer of infrastructure should incorporate security measures to create a comprehensive defence strategy. This holistic approach ensures that vulnerabilities are minimised across:

- Network layers  
- Servers and applications  
- User access and authentication  
- Data storage and transmission

**In summary:** Security protocols like TLS enabled via trusted digital certificates are essential to secure online communications. Awareness and proactive security management across all roles in IT infrastructure significantly enhance organisational security posture.

### Key Terms and Definitions

| Term                   | Definition                                                                                   |
|------------------------|----------------------------------------------------------------------------------------------|
| **HTTPS**              | Secure version of HTTP using encryption to protect data transferred between browser and server. |
| **HTTP**               | Protocol for transferring web content without encryption.                                   |
| **TLS (Transport Layer Security)** | Protocol providing encryption and security over networks; replaces SSL.               |
| **SSL (Secure Sockets Layer)** | Older protocol for securing communications, now deprecated in favour of TLS.             |
| **Certificate Authority (CA)** | Trusted third party that issues digital certificates to verify website authenticity.       |
| **Digital Certificate** | An electronic document confirming the identity of a website and enabling encrypted connections.|
