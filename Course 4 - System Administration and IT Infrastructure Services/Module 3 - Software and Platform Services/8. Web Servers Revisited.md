 **Introduction to Platform Services and Web Servers**  
- **Platform services** allow developers to build and deploy software applications without managing underlying infrastructure such as operating systems, server hardware, or networking.  
- Examples of platform services include:  
  - Web servers where web applications are deployed.  
  - Development software used to write application code.  
- In the contemporary digital landscape, most businesses maintain an online presence, either via a promotional website or a website that constitutes their business itself.  
- These businesses host their services on **web servers**.

**Role and Functionality of Web Servers**  
- A **web server** stores and serves content to clients over the Internet.  
- Web servers are accessed by domain names, e.g., **google.com**.  
- A web server consists of two main components:  
  - The physical server that stores web files (e.g., HTML, CSS, images).  
  - The **HTTP server software** that processes HTTP requests.  
- **HTTP (Hypertext Transfer Protocol)** is the protocol used to format and transfer web pages across the Internet.  
- When a web browser requests a web page via a URL, it sends an HTTP request to the server.  
- The HTTP server then responds with the requested content via an HTTP response.

**Popular HTTP Server Software: Apache**  
- Several HTTP server software options exist, but the most widely used is the **Apache HTTP Server** (commonly called **Apache**).  
- Apache is:  
  - Free and open-source software.  
  - Responsible for serving a significant portion of the web pages on the Internet.  
-  installing Apache to illustrate how straightforward it is to run a web service.

**Demonstration: Installing Apache Web Server on Linux**  
- The installation command used is:  
  $$ \text{sudo apt-get install apache2} $$  
- After installation, the Apache service runs on the local machine, enabling it to host web content.  
- The host machine's IP address is $127.0.0.1$, referred to as **localhost**.  
- **Localhost** is a reserved hostname that points to the local computer itself and cannot be registered as a domain name on the Internet.

**Accessing Local Web Server Content**  
- By entering **localhost** in a web browser, the default web server content served by Apache can be viewed.  
- This default content comes pre-installed with Apache.  
- To serve custom web content, one must:  
  - Navigate to the directory where Apache stores web files.  
  - Replace the default files with their own web pages or resources.  
- Since this content is hosted locally, **DNS (Domain Name System)** is required to make the web server accessible globally by associating a domain name with the server’s public IP address.

**Summary of Web Server Management Roles**  
- **System administrators** are not typically responsible for creating the web content but are responsible for ensuring the availability and proper function of the web server and its services.  
- IT support specialists managing web services should have a solid understanding of how web servers operate in order to maintain and troubleshoot the services effectively.

### Key Insights  
- **Platform services abstract infrastructure management**, enabling developers to focus on application development.  
- A **web server combines physical storage and HTTP server software** to deliver web content.  
- **Apache HTTP Server** is a dominant, free, and open-source HTTP server widely used across the Internet.  
- Localhost ($127.0.0.1$) is a reserved IP address for local machines and cannot be used as an Internet domain.  
- DNS plays a crucial role in making locally hosted web servers accessible globally.  
- Effective web server management requires understanding the underlying mechanisms but not necessarily content creation.
