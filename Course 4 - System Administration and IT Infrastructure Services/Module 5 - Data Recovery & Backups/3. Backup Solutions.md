# Notes
 
**Introduction to Backup Solutions: DIY vs Cloud Providers**  
The video begins by addressing the decision-making process organisations face when choosing a backup solution: whether to implement a **DIY (Do-It-Yourself) backup system** or to use one of the many **cloud backup providers**.  

- **DIY backups** typically involve purchasing a commercial **NAS (Network Attached Storage)** device, filling it with hard drives, and transferring data over the network. This approach is straightforward but raises several challenges:  
  - **Scalability:** How to increase disk capacity when more storage is needed?  
  - **Reliability:** How to handle inevitable hard disk failures?  

- It is emphasised that these options are **not mutually exclusive**; combining **on-site and off-site backups** is often recommended when budget allows, offering enhanced data protection.

- Another critical factor in backup strategy is the **retention period**: how long backups must be kept. This affects the **long-term storage requirements** and the **cost** of maintaining the backup system.

  
**Backup Retention and Archival Storage: Cost and Accessibility Trade-offs**  
The discussion continues about managing the retention of backups and introducing archival storage methods:  

- A common approach to balance cost and convenience is to **archive older data** using **slower but cheaper storage media**.  
- The **standard medium for archival backup** is **data tapes**:  
  - Similar to audio cassette tapes, they use pools of magnetic tape for writing and reading data.  
  - Tape storage is **cost-effective** but **slower and less accessible** compared to hard drives or solid-state drives (SSDs).  

- Tape storage is typically reserved for **long-term archival purposes**, where the likelihood of needing the data is low, and some **delay in data retrieval is acceptable**.

  
**Overview of Backup Tools and Examples**  
The video then briefly surveys some common tools and examples of backup solutions, without delving into exhaustive detail due to the large number available:  

- **Rsync:**  
  - A **command line file transfer utility** often used for backups.  
  - Designed to **efficiently transfer and synchronise files** between locations or computers.  
  - Supports **compression** and uses **SSH (Secure Shell)** to securely transfer data over a network.  
  - Can synchronise files between remote machines, making it useful for **simple automated backups**.

- **Apple Time Machine:**  
  - A **first-party backup solution** for Mac operating systems.  
  - Uses an **incremental backup model**, which means only changes since the last backup are saved.  
  - Supports restoring the **entire system**, **individual files**, or **older versions** of files.

- **Microsoft Backup and Restore:**  
  - A first-party solution from Microsoft with two modes:  
    - **File-based backup:** saves files to a zip archive; supports both **complete** and **incremental backups**.  
    - **System image backup:** saves the entire disk block-by-block to a file; supports **differential backup mode**, which backs up only disk blocks that changed since the last backup.  

| Backup Solution       | Type                  | Backup Model        | Key Features                         | Supported Platforms        |
|----------------------|-----------------------|---------------------|------------------------------------|---------------------------|
| Rsync                | File transfer utility | Not explicitly backup, used as backup | Compression, SSH support for secure transfer, synchronisation | Linux, macOS, Windows (via Cygwin or ports) |
| Apple Time Machine   | Incremental backup tool | Incremental         | Full system restore, individual file restore, file versioning | macOS                     |
| Microsoft Backup & Restore | File and system image backup | Complete, Incremental (file); Differential (system image) | Zip archive, block-level system backup | Windows                   |


**Key Insights and Considerations for Backup Strategy**  
- **Hybrid backup approaches** combining on-site and cloud or off-site solutions provide **better resilience** and **disaster recovery capabilities**.  
- The choice of backup media (hard disk, SSD, tape) impacts **cost, speed, and accessibility** of data.  
- Backup retention policies have a **direct influence on storage costs and infrastructure planning**.  
- **Incremental and differential backup models** optimise storage usage and backup time by saving only changed data, rather than full copies each time.  
- The suitability of a backup tool depends on the **platform ecosystem** and the **specific backup needs** (e.g., full system recovery vs file-level restore).  

**Conclusions:**  
- On-site DIY backups offer **control and potential cost savings** but require careful planning for scalability and fault tolerance.  
- Cloud or third-party solutions may offer **ease of management and scalability** but often at a recurring cost.  
- Combining both strategies is often the most **robust and recommended** approach for organisations with the budget.
