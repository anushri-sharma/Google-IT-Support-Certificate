# Notes


**Summary of Data Backup and Recovery Planning**

- **Primary objective:** Protect an organisation from critical data loss by designing an effective data backup and recovery plan.
- **Step 1: Identify essential data for backup.**  
  - Only back up data that is **absolutely necessary for operations** and not reproducible from other sources.  
  - Typical critical data includes:  
    - Emails  
    - Spreadsheets (financial and operational)  
    - Databases  
    - Server configurations  
  - Non-essential data, such as personal downloads or non-critical files (e.g., cat pictures), should be excluded to avoid unnecessary storage and cost increases.

- **Cost consideration:**  
  - Every additional file backed up consumes disk space and raises overall backup solution costs.  
  - This necessitates careful selection of data to be backed up.

**Assessment of Backup Size and Future Growth**

- After deciding which data to back up, determine the **current total volume of data** to be backed up.
- Consideration must extend beyond current needs:  
  - Organisations typically experience **data growth over time**.  
  - Backup solutions must be **scalable and flexible** to accommodate increasing data volumes without significant reconfiguration or cost spikes.

- **Key point:** Backup planning is a continuous process that evolves alongside organisational growth.

**Local (On-site) vs Off-site Backup Solutions**

- **On-site backups:**  
  - Store data physically close to the operational environment.  
  - Advantages:  
    - Faster access and data restoration due to proximity.  
    - No dependency on external network bandwidth for backup or restore operations.  
  - Disadvantages:  
    - Vulnerable to local disasters (e.g., fire, flood). If the physical site is compromised, both primary data and backups can be lost simultaneously.

- **Off-site backups:**  
  - Backup data is stored remotely, in a physically separate location.  
  - Can be managed via:  
    - Another company office  
    - Cloud-hosted backup services  
  - Advantages:  
    - Protection against catastrophic events at the primary site ensures data availability even if the original location is destroyed.  
  - Disadvantages:  
    - Requires transmitting data over the Internet or WAN, which implicates:  
      - Bandwidth consumption and potential bottlenecks.  
      - Longer backup times depending on connection speed and data volume.  
      - Security considerations during transmission.

**Security Considerations for Backup Data**

- Backup data often contains **sensitive and confidential information**, making security paramount.
- **Encryption requirements:**  
  - Data must be encrypted **during transmission** (preferably using **TLS**) to prevent interception or unauthorized access when sent off-site.  
  - Data must also be encrypted **at rest** on backup storage devices to protect against theft or unauthorized physical access.  
- These encryption measures are considered **best security practices** and must be integrated into any backup strategy to ensure data confidentiality and integrity.

### Summary Table: Backup Solution Characteristics

| Backup Type | Advantages                              | Disadvantages                          | Security Considerations                     |
|-------------|--------------------------------------|--------------------------------------|---------------------------------------------|
| On-site     | Fast access and restoration          | Vulnerable to site-wide disasters    | Local controls; physical security important |
| Off-site    | Protection from catastrophic events  | Requires bandwidth; longer transfer times | Must encrypt data in transit and at rest    |

### Key Insights

- **Data selection for backup is critical** to balance operational resilience and cost-efficiency.
- **Backup solutions must scale** with organisational data growth.
- **Combining on-site and off-site backups** is recommended to mitigate different types of risks.
- **Encryption in transit and at rest is mandatory** for protecting confidential business data.
- Organisations should plan for **future-proof backup strategies** that integrate security, scalability, and reliability.
