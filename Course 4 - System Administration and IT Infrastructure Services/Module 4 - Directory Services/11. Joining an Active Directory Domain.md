
# [Joining an Active Directory Domain](https://www.youtube.com/watch?v=kxb_5AG55_0&t=6s)



## Summary
  
**Introduction to Joining Computers to Active Directory (AD)**  
- Computers can be **joined (or bound) to an Active Directory domain**, which involves two key aspects:  
  1. **Active Directory has a computer account provisioned for that machine.**  
  2. **The computer knows about the AD domain and authenticates with it.**  
- The presenter demonstrates a Windows machine **not joined to a domain**, described as a **workgroup computer**.

 
**Workgroup Computers vs Domain-Joined Computers**  
- **Workgroup computers** are collections of **stand-alone computers that work together** without central administration.  
- Workgroups **lack centralised management**, becoming difficult to manage as network size increases.  
- The goal is **central administration and authentication** via Active Directory.  
- The process to join a workgroup computer to a domain begins by accessing the **System Properties** and making domain changes.


**Manual Domain Join Process via GUI**  
- The computer is initially part of a **workgroup**.  
- In the **System Properties > Change Settings > Change**, you can choose to make the computer a member of either:  
  - A **workgroup** (default)  
  - Or a **domain** (cannot be both simultaneously).  
- The user enters the domain name (e.g., **example.com**) to connect to the Active Directory domain.

 
**Authentication and Domain Controller Interaction**  
- After entering the domain name and clicking **OK**, the computer:  
  - Searches the network for a **domain controller (DC)**.  
  - Requests **authorization credentials** (domain admin username and password).  
- Upon successful authentication, the domain controller:  
  - **Creates a computer account** in the domain for this machine.  
  - The computer **reconfigures itself to use AD authentication services**.  
- This process **requires a reboot** to complete.

  
**Viewing the Joined Computer in Active Directory Administrative Center**  
- The presenter demonstrates opening the **Active Directory Administrative Center**.  
- Within the **Computers container**, the newly joined computer appears.  
- Once joined, the machine:  
  - Uses the **Active Directory domain for authentication**.  
  - Can be managed via **Group Policy** for centralised control.

  
**Joining Computers to the Domain Using PowerShell**  
- The presenter also shows how to join a computer to the domain using the **PowerShell CLI** with the command:  

  $$ \text{Add-Computer -DomainName example.com -Server dc1} $$  

- After execution, the user is prompted for **credentials**.  
- By default, the command does **not reboot the machine automatically**; however, adding the **`-Restart` parameter** will reboot the machine after joining.  
- This method provides a **simple and scriptable** alternative to GUI-based domain joining.


**Active Directory Functional Levels**  
- Over time, **Active Directory has evolved through multiple versions**, known as **functional levels**.  
- A domain or forest’s **functional level determines the AD features and capabilities supported**.  
- Administrators must be aware of their domain and forest functional levels to:  
  - Understand feature availability.  
  - Plan upgrades to support new AD features.  
- The presenter shows how to view the domain’s functional level in the domain properties, noting the example domain is at **Windows Server 2016 functional level**.

 
**Retrieving Functional Level Information via PowerShell**  
- Functional levels can also be checked from PowerShell using:  

  $$ \text{Get-ADForest} $$  
  $$ \text{Get-ADDomain} $$  

- These commands reveal:  
  - The **forest functional level** (forest mode).  
  - The **domain functional level** (domain mode).  
- Knowing these levels helps correlate with supported AD features and informs upgrade decisions.

### Summary Table: Key Commands and Concepts

| Task                             | Method                       | Command / Action                          | Notes                        |
|---------------------------------|------------------------------|------------------------------------------|------------------------------|
| Join computer via GUI            | System Properties            | Change domain/workgroup -> enter domain  | Requires domain admin creds   |
| Join computer via PowerShell     | CLI                          | `Add-Computer -DomainName example.com -Server dc1 [-Restart]` | Prompts for credentials; optional reboot |
| View domain functional level     | Domain Properties (GUI)       | N/A                                      | Example: Windows Server 2016  |
| View domain/forest functional level | PowerShell                  | `Get-ADForest`; `Get-ADDomain`           | Shows forest mode and domain mode |

### Key Insights  
- **Joining a computer to Active Directory involves provisioning a computer account on the domain controller and configuring the machine to authenticate via AD.**  
- **Workgroup computers are suitable only for small or unmanaged networks; domains provide centralised management and authentication.**  
- **PowerShell provides an efficient scripting alternative for domain join operations, supporting automation and remote management.**  
- **Active Directory functional levels dictate the available features and capabilities; administrators must monitor and upgrade these levels appropriately.**

### Core Concepts  
- **Active Directory Domain:** A network structure enabling centralised authentication and management.  
- **Domain Controller (DC):** A server that responds to authentication requests and enforces security policies within the domain.  
- **Computer Account:** A representation of the joined machine within AD that facilitates authentication.  
- **Group Policy:** A mechanism to centrally manage and configure operating system and application settings on domain-joined machines.  
- **Functional Level:** The version or capability set of an AD domain or forest, influencing what AD features are available.

This summary captures all the key points and procedural details related to joining Windows computers to an Active Directory domain, both via the GUI and PowerShell, along with important AD management concepts such as functional levels.
