
# [What is LDAP?](https://www.youtube.com/watch?v=qXebEsB2ZFI&t=4s)

- **LDAP (Lightweight Directory Access Protocol)** is the underlying protocol used for accessing information in directory services over a network.

<img width="1272" height="418" alt="image" src="https://github.com/user-attachments/assets/05ec2429-c26c-47e5-89c5-6a53271b9eca" />


- **Active Directory** and **OpenLDAP** are two popular directory services that utilize LDAP.

<img width="1280" height="716" alt="image" src="https://github.com/user-attachments/assets/bdf16f07-61b2-45ec-ac54-c7376f4fc357" />


- LDAP supports various operations, including adding, deleting, and modifying entries in a directory server database.
- An **LDAP entry** is a collection of information used to describe an object within the directory service.

<img width="1278" height="221" alt="image" src="https://github.com/user-attachments/assets/a1c8a12d-0912-48ed-8ec3-69d5f83dfb55" />

- LDAP entries are structured with a unique **Distinguished Name (dn)**, which is a unique entry name.

- Each entry also consists of **attributes** and their corresponding **values**.
- Common attributes include:
  - **CN (Common Name):** Represents the name of the object (e.g., a person's name).
  - **OU (Organizational Unit):** Denotes a group or organizational division (e.g., Sysadmin).
  - **DC (Domain Component):** Represents parts of a domain name (e.g., "example" and "com" from "example.com").

- The core concept of LDAP notation is to use attributes and values to describe entries within directory services.
