
# [What is a directory server?](https://www.youtube.com/watch?v=Y0sQ4qokrIo&t=7s)

### Introduction to Directory Servers
- **Directory servers** function like a phone directory, providing a lookup service for network resources and their addresses.
- They organize and manage **organizational objects** like user accounts, groups, telephone numbers, and network shares.
- Instead of local management on each machine, information is stored on a directory server for easy access and management.
- An ideal enterprise-quality directory server should support **replication**, allowing data to be distributed across multiple servers while appearing as a unified store.
- **Replication** is crucial for:
  - Redundancy: Multiple services available reduce disruption.
  - Decreased latency: Faster query responses from the directory service.

### Benefits of Directory Servers
- Having **replicas** in each office allows for quicker responses to directory service queries.
- The directory service should be **flexible**, allowing new object types to be created as needs evolve.
- Access to information must be feasible from various OS types and designated areas of the corporate network.
- **Hierarchical model** of objects and containers enhances organization and searchability.
  - **Organizational Units (OUs)** function like folders, containing objects or other OUs.
  - This structure helps maintain organization, similar to subfolders in a file system.

### Hierarchical Structure of Directory Services
- The directory structure may include an OU called **users**, containing user accounts.
- Additional OUs can represent teams, such as **sales**, **engineering**, and **marketing**.
- This hierarchical structure allows for specific policies:
  - Stricter password requirements can be enforced for engineering members without affecting others.
  - Sub OUs inherit characteristics from their parent OU, affecting all subvalues uniformly.
- **System administrators** are responsible for the setup, configuration, and maintenance of the directory server, including:
  - OS management tasks (e.g., updates and service configurations).
  - Installation and configuration of the directory service itself and any related services.
  - Designing and implementing the hierarchy and structure of the directory.
