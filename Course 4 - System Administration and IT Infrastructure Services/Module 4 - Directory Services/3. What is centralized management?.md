**Summary of Systems Administrator Role and Challenges**  
- The core responsibility of a **systems administrator (sysadmin)** is to manage and maintain IT systems to ensure they are operational and serve their organisational functions.  
- Tasks include applying **security patches**, updating applications, and managing **user accounts** across many computers and devices.  
- Performing these tasks manually on each server or end-user device (e.g., logging into each server or visiting each employee’s desk) is highly inefficient and prone to inconsistency.  
- This highlights the need for **centralised management** systems to streamline administration and improve consistency and efficiency.  

 
**Introduction to Centralised Management and Directory Services**  
- Instead of manual, individual device management, sysadmins use a **central service** to send instructions to all parts of the IT infrastructure.  
- **Directory services** are key examples of such centralised management tools.  
- Directory services provide **centralised authentication, authorisation, and accounting (AAA)**, which are critical to managing access and permissions across all networked resources.  
- This centralisation allows user accounts to be created once, and immediately be available organisation-wide, even in environments with **dozens, hundreds, or thousands of users and computers**.

**AAA Services and Directory Services in Large Networks**  
- Directory services handle:  
  - **Authentication:** verifying user identities.  
  - **Authorisation:** deciding what resources users can access.  
  - **Accounting:** tracking user activities and access for audit purposes.  
- Using directory services for AAA means access decisions are centralised rather than handled individually by each computer or application.  
- This approach simplifies user account management across an extensive network.  

  
**Role-Based Access Control (RBAC) and Group Management**  
- Access control can be managed more efficiently by assigning permissions based on a user’s **role** rather than individual identity.  
- Example: granting access to a network file system for all members of the IT department via a group, rather than individual users.  
- This method is crucial when employees join or leave departments, or change roles.  
- **User groups** enable sysadmins to organise accounts based on various criteria such as:  
  - Physical location (e.g., building).  
  - Job function or role.  
  - Any logical classification that supports management needs.  
- This organisation simplifies access control and reduces administrative overhead.

**Practical Example of RBAC for Sysadmins**  
- A systems administrator’s permissions—such as creating user accounts and resetting passwords—are granted because of their **role**.  
- Rather than assigning permissions individually to each sysadmin, a **"sysadmins" group** is created.  
- All sysadmins are added to this group, which is granted the necessary permissions collectively.  
- When a person changes roles, their group membership is updated, which automatically adjusts their access rights without changing individual permissions.  
- This mechanism is called **Role-Based Access Control (RBAC)** and is essential for scalable, manageable security administration.  


**Centralised Configuration Management Beyond Access Control**  
- Centralised management extends beyond access control to **configuration management**.  
- Examples of configuration tasks include:  
  - Setting up printers.  
  - Configuring software.  
  - Mounting network file systems.  
- Individually configuring these for every computer is inefficient and impractical in large organisations.  
- Centralised configuration management allows administrators to define rules or policies that automatically apply configurations across many systems.  
- One simple starting point for centralised configuration is the use of **logon scripts** that execute whenever a user logs into a computer, applying configurations or policies dynamically.  

### Key Concepts and Terms  
| Term                       | Definition                                                                                  | Notes                                                |
|----------------------------|---------------------------------------------------------------------------------------------|------------------------------------------------------|
| **Systems Administrator**  | IT professional responsible for managing and maintaining computer systems.                   | Role includes system updates, user account management|
| **Directory Services**      | Centralised services that manage AAA (authentication, authorisation, accounting).           | Enables network-wide user account and permission management|
| **AAA**                    | Authentication, Authorisation, Accounting - core functions for access control and auditing. | Central to network security and management            |

### Summary of Advantages of Centralised Management  
- **Efficiency:** Reduces manual work by centralising tasks that would otherwise require individual attention.  
- **Consistency:** Ensures uniform application of security policies, configurations, and access control.  
- **Scalability:** Supports large organisations with many users and devices without exponential increase in management effort.  
- **Flexibility:** Role and group-based controls adapt easily to organisational changes such as staff turnover or role changes.
