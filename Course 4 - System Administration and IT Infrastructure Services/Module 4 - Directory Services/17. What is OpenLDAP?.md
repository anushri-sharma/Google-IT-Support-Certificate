
# [What is OpenLDAP?](https://www.youtube.com/watch?v=JKTm2nIQ6Ao&t=11s)

This lesson provides an overview of OpenLDAP, a free and open-source directory service that operates similarly to Active Directory. OpenLDAP uses LDAP notation or LDIF for authentication and management of users, groups, and computers across various operating systems like Linux, macOS, and Windows. While Active Directory is recommended for Windows environments due to it being Microsoft's proprietary software, OpenLDAP's open-source nature makes it a versatile option.

The lesson demonstrates how to set up and configure an OpenLDAP server. The process begins with installing the necessary packages, `slapd` and `ldap-utils`, using the command `sudo apt-get install slapd ldap-utils`. During installation, users are prompted to set an administrator password for LDAP.

Following the initial installation, the `slapd` package is reconfigured using `sudo dpkg-reconfigure slapd`. This command initiates a series of configuration questions, including:

- **Omit OpenLDAP server configuration:** Set to 'No'.
- **DNS domain name:** Configured as 'example.com'.
- **Organization name:** Set to 'example'.
- **Administrator password:** Re-entered for confirmation.
- **Database:** MDB is chosen.
- **Remove database on purge:** Set to 'No'.
- **Move old database:** Set to 'Yes'.
- **Allow LDAP Version 2 protocol:** Set to 'No'.

Upon completion of these steps, an OpenLDAP server is successfully running.
