
# [What is LDAP Authentication?](https://www.youtube.com/watch?v=_TL4nw_mGbo&t=1s)

LDAP (Lightweight Directory Access Protocol) provides different authentication levels to control access to directory information, similar to how public phone books differ from private mobile contact lists. These levels allow directories to be made public or kept private.

**LDAP Operations and Authentication:**

- **Bind Operation:** This operation authenticates clients to the directory server. For example, when logging into a website using a directory service, your login information is sent to the server via LDAP to verify your account and password, granting access if valid.
- **Data Protection:** It is important that data is protected and encrypted during this authentication process.

**Common Authentication Methods:**

1. **Anonymous Binding:** No actual authentication occurs. Depending on configuration, anyone may access the directory, akin to a public phone book.
2. **Simple Authentication:** Requires only a directory entry name and password, which are typically sent in plain text, making it insecure.
3. **SASL (Simple Authentication and Security Layer):** This method can utilize security protocols like TLS and Kerberos. SASL authentication requires mutual authentication between the client and the directory server.
   - **Kerberos:** A common authentication method used with SASL. Kerberos is a network authentication protocol that verifies user identity and secures credential transfers. It is a complex topic that will be revisited.

**Authorization:**

Once a client successfully authenticates with the LDAP server, the user is authorized to access resources based on their assigned access levels.
