
# [Implementing Directory Services](https://www.youtube.com/watch?v=oIggpfIua1I&t=2s)

### Directory Services Overview
- **Directory services** became an **open network standard** for interoperability among different software vendors.
- In **1988**, the **X.500 directory standard** was approved, which included several protocols:
  - **Directory Access Protocol (DAP)**
  - **Directory System Protocol (DSP)**
  - **Directory Information Shadowing Protocol (DISP)**
  - **Directory Operational Bindings Management Protocol (DOP)**
- Alternatives to DAP were created to allow clients to access the X.500 directory, with the most popular being **Lightweight Directory Access Protocol (LDAP)**.
- Numerous **implementations** of directory services emerged, including offerings from:
  - **Apache**
  - **Oracle**
  - **IBM**
  - **Red Hat**
- Detailed focus will be on two main implementations:
  - **Microsoft's Active Directory (AD)**: Customized for the Windows platform with additional features.
  - **OpenLDAP**: An open-source implementation supporting various platforms like Windows, UNIX, and Linux.
 
### Client Tools and Applications
- Alongside service software, there are **client tools** for accessing and managing a directory server.
- **Microsoft Active Directory Users and Computers (ADUC)** is a notable client tool that works effectively with Microsoft's Active Directory server.
- Other **open tools** can interface with various directory server implementations.
- Client applications are also available to **query** a directory server, allowing integration across major operating systems for login and authentication.
- This setup provides the advantage of **centralized management** of user accounts.
