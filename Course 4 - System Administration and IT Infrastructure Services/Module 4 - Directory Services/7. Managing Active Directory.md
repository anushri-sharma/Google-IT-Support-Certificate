
# [Managing Active Directory](https://www.youtube.com/watch?v=oK-nGhy47NE&t=7s)


Managing Active Directory (AD) is a complex task, often handled by dedicated system administrators. Key to AD management are default user groups created when a domain is set up.

### Core Active Directory Groups:

- **Domain Admins:** This group holds ultimate power over the AD domain. Members can make any changes to the domain and, by extension, to all computers joined to it, effectively becoming local administrators on those machines. Due to this extensive control, accounts should not be added to this group lightly.
- **Enterprise Admins:** These administrators have permissions to modify the AD domain in ways that impact other domains within a multi-domain forest. Membership is typically limited to the administrator account in a new domain and is primarily used for tasks like upgrading the AD forest.
- **Domain Users:** This group includes every user account in the domain. It's a convenient way to grant access to network resources for all users without individually assigning permissions to each account.
- **Domain Computers:** This group contains all computers joined to the domain, excluding domain controllers.
- **Domain Controllers:** This group encompasses all domain controllers within the domain.

### Best Practices for Administrators:

System administrators, IT support specialists, and those with domain or enterprise admin roles must exercise caution. A Domain Admin account should never be used for daily tasks due to the high risk of accidental, widespread changes. Instead, use a standard user account with restricted permissions for everyday activities. If frequent administrative tasks require broad AD access, consider using delegation. Similar to file system permissions, Access Control Lists (ACLs) can be applied to AD objects to grant specific permissions for delegated tasks.
